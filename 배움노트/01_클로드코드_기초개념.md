# 📘 배움노트 1편 — 클로드 코드, 진짜 쉽게 이해하기

> 작성일: 2026-07-13
> 목표: 어려운 용어 없이, 나올 때마다 옆에 바로 설명 붙이기.
> 비유 하나로 끝까지 갑니다: **"클로드 = 우리 회사에 새로 온 만능 직원"**

---

## 1. 클로드 코드가 뭔가요?

**클로드 코드**(Claude Code)는 그냥 "말로 시키면 컴퓨터 작업을 대신 해주는 AI 직원"이에요.

- 일반 클로드(채팅)는 **말만 해주는 상담사**
- 클로드 코드는 **직접 손으로 일하는 직원** — 파일을 만들고, 고치고, 저장하고, 인터넷도 찾아봐요.

우리 회사로 치면:
> "○○ 클라이언트 견적서 초안 만들어줘" 하면
> 말로만 설명하는 게 아니라 **실제로 파일을 만들어서 폴더에 넣어주는** 직원.

---

## 2. 헷갈리는 3형제: MD, 스킬, 에이전트

이 셋이 제일 헷갈리는데, **"직원한테 주는 문서 3종류"**라고 생각하면 끝이에요.

### ① CLAUDE.md = 사원 수첩 📓

- **md**(마크다운, markdown): 그냥 **글자로 된 문서 파일**이에요. 한글(hwp)이나 워드(docx)처럼 파일 종류 이름일 뿐. `#`을 붙이면 제목이 되는 등 간단한 규칙이 있는 메모장 파일.
- **CLAUDE.md**: 클로드가 **출근할 때마다(= 대화 시작할 때마다) 자동으로 읽는 수첩**이에요.

여기에 이렇게 적어두는 거예요:
> "우리 사장님은 디자인 회사 실장님이다. 한국어로 답해라. 프로젝트는 이 폴더에 정리해라."

그래서 매번 다시 설명 안 해도 클로드가 나를 아는 거예요. **딱 하나만 기억하세요: CLAUDE.md = 클로드가 매번 읽는 나에 대한 메모.**

### ② 스킬(Skill) = 업무 매뉴얼 📋

**스킬**은 "이런 일은 이런 순서로 해라"라고 적어둔 **작업 설명서**예요.

예: 우리한테 있는 `list-deck-design` 스킬
> "리포트 만들어줘" 하면 → 미리 정해둔 스타일(주황색 포인트, 프리텐다드 글꼴...)대로 만들어라

라고 적힌 매뉴얼이에요. 매번 "주황색으로, 이 글꼴로..." 설명할 필요 없이 **매뉴얼 이름만 부르면** 되는 거죠.

- 저장 위치: `.claude/skills/스킬이름/SKILL.md`
- 쓰는 법: 그냥 말로 시키거나 `/스킬이름` 이라고 치면 됨

### ③ 에이전트(Agent) = 전문 분야가 있는 부하 직원 👤

**에이전트**는 **역할을 정해둔 가상의 직원**이에요. 우리 회사엔 지금 24명 있죠.

- `quote-accountant` = 견적 담당 경리
- `design-critique` = 시안 까칠하게 봐주는 검수자
- `market-research` = 시장조사 담당

"견적 뽑아줘"라고 하면 클로드(팀장)가 견적 담당 직원한테 일을 넘기는 구조예요.

- 저장 위치: `.claude/agents/에이전트이름.md`

### 세 줄 정리

| 이름 | 비유 | 한 줄 설명 |
|---|---|---|
| CLAUDE.md | 사원 수첩 | 클로드가 매번 읽는 "나에 대한 정보" |
| 스킬 | 업무 매뉴얼 | "이 작업은 이렇게 해라" 순서 설명서 |
| 에이전트 | 부하 직원 | 역할이 정해진 전문 담당자 |

**공통점: 셋 다 그냥 글로 쓴 md 파일**이에요. 코딩이 아니라 한국어로 쓴 지시문이에요. 무섭지 않아요.

## 3. 깃허브(GitHub)는 왜 쓰나요?

### 깃허브 = 회사 공용 서류 캐비닛 🗄️ (인터넷에 있는)

**깃허브**(GitHub)는 **파일을 인터넷에 보관해주는 창고**예요. 네이버 클라우드나 구글 드라이브랑 비슷한데, 두 가지가 특별해요:

1. **모든 변경 기록이 남아요** — "누가, 언제, 뭘, 왜 바꿨는지" 전부. 시안 파일명 `최종_진짜최종_v3.psd` 이런 짓 안 해도 됨.
2. **클로드가 직접 읽고 쓸 수 있어요** — 그래서 컴퓨터를 꺼도, 다른 데서 접속해도 이어서 작업 가능.

### 꼭 알아야 할 깃허브 용어 (이것만!)

| 용어 | 쉬운 뜻 | 비유 |
|---|---|---|
| **저장소**(repository, 레포) | 프로젝트 하나를 담는 폴더 | 캐비닛 서랍 하나 |
| **커밋**(commit) | "여기까지 작업함" 하고 저장 도장 찍기 | 서류에 날짜 도장 쾅 |
| **푸시**(push) | 내 컴퓨터 작업물을 인터넷 창고로 올리기 | 서류를 캐비닛에 갖다 넣기 |
| **풀**(pull) | 인터넷 창고 최신본을 내 컴퓨터로 가져오기 | 캐비닛에서 서류 꺼내오기 |
| **브랜치**(branch) | 원본 안 건드리고 만드는 **작업용 복사본** | 원본 시안은 두고 복사본 위에 수정하기 |
| **메인**(main) | 진짜 원본이 있는 기본 브랜치 | "확정본" 서랍 |
| **PR**(pull request, 풀 리퀘스트) | "복사본 작업 끝났어요, 원본에 합쳐도 될까요?" 하는 **합치기 요청서** | 팀원이 실장님한테 컨펌 요청 올리는 것 |
| **머지**(merge) | PR을 승인해서 원본에 합치기 | 실장님이 "컨펌!" 하고 확정본에 반영 |

### 브랜치, 조금만 더

디자인으로 치면 이래요:

```
main (확정 시안 원본)
 └── 작업용 브랜치 (복사본 떠서 수정 작업)
      → 작업 끝 → PR (컨펌 요청) → 머지 (원본에 반영)
```

**왜 이렇게 하냐면**: 복사본에서 아무리 망쳐도 원본은 멀쩡하니까요. 우리가 원본 PSD 놔두고 사본으로 작업하는 거랑 똑같은 이유예요.

지금 이 문서도 `claude/beginner-claude-concepts-ph30f8`라는 브랜치(작업용 복사본)에 올라가 있어요.

## 4. 그럼 전체 흐름은 이렇게 돌아가요

내가 클로드 코드로 일하는 하루:

1. **말로 시킨다** — "○○ 프로젝트 견적 정리해줘"
2. 클로드가 **CLAUDE.md**(사원 수첩)를 읽고 나를 파악
3. 필요하면 **에이전트**(담당 직원)나 **스킬**(매뉴얼)을 꺼내서 작업
4. 결과물을 파일로 만들고 **커밋**(저장 도장)
5. **푸시**(인터넷 창고에 올리기) → 어디서든 다시 볼 수 있음
6. 필요하면 **PR**(컨펌 요청) → 확인하고 **머지**(확정)

**내가 외울 건 1번뿐이에요. 2~6번은 클로드가 알아서 합니다.**

## 5. 보너스: "스키마"도 봤는데 그건 뭔가요?

**스키마**(schema)는 **"양식(서식)"**이에요. 그게 다예요.

견적서에 비유하면:
> "견적서에는 반드시 ①클라이언트명 ②항목 ③단가 ④합계가 들어가야 한다"

이렇게 **빈칸의 형태를 미리 정해둔 틀**이 스키마예요. 클로드나 프로그램들이 자료를 주고받을 때 "이 양식대로 채워서 줘"라고 약속하는 용도로 써요. 우리가 직접 만질 일은 거의 없으니 "아, 양식이구나" 하고 넘어가면 됩니다.

## 6. 용어 사전 (모르는 말 나오면 여기서 찾기)

| 용어 | 읽는 법 | 뜻 |
|---|---|---|
| Claude Code | 클로드 코드 | 말로 시키면 파일 작업까지 해주는 AI 직원 |
| md / markdown | 엠디 / 마크다운 | 간단한 규칙이 있는 텍스트 문서 파일 (hwp, docx 같은 파일 종류) |
| CLAUDE.md | 클로드 엠디 | 클로드가 대화 시작마다 읽는 나에 대한 메모 |
| skill | 스킬 | 특정 작업의 순서를 적어둔 업무 매뉴얼 |
| agent | 에이전트 | 역할이 정해진 가상의 전문 직원 |
| GitHub | 깃허브 | 파일을 기록과 함께 보관하는 인터넷 창고 |
| repository | 리포지토리 (레포) | 프로젝트 하나를 담는 저장 공간 |
| commit | 커밋 | "여기까지 작업" 저장 도장 찍기 |
| push | 푸시 | 내 작업물을 인터넷 창고에 올리기 |
| pull | 풀 | 인터넷 창고의 최신본을 가져오기 |
| branch | 브랜치 | 원본을 안 건드리는 작업용 복사본 |
| main | 메인 | 확정본(원본)이 있는 기본 브랜치 |
| PR (pull request) | 피알 (풀 리퀘스트) | "복사본을 원본에 합쳐주세요" 컨펌 요청 |
| merge | 머지 | 컨펌 요청을 승인해서 원본에 합치기 |
| schema | 스키마 | 자료를 담는 양식(서식) 틀 |
| MCP | 엠씨피 | 클로드를 다른 서비스(노션, 피그마, 지메일…)에 연결해주는 어댑터(멀티탭) |
| prompt | 프롬프트 | 클로드에게 하는 지시·요청 말 |
| session | 세션 | 클로드와의 대화 한 판 (한 번의 근무 시간) |
