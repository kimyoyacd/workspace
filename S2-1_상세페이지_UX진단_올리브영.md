|---------|----------|------|

| 컬러명 표시 | 별도 텍스트 이미지로 나열 | 팔레트 위에 직접 오버레이되지 않음 |
| 텍스처 구분 | 미구분 (매트/시머/글리터/베이크드 혼재) | 리뷰어들이 자체적으로 텍스처 분류하여 리뷰 |

| AI 활용 | 없음 | 현행 상세페이지에 AI 생성 콘텐츠 미적용 |

### B-2. 문제별 상세 분석 + 실증 근거

#### P1. 도입부(ATF) 후킹 부재

```
현상:
  상세페이지 첫 화면 = 제품 패키지 풀컷 (화이트 배경)
  브랜드 무드·감각이 전달되지 않는 "카탈로그" 시작

근거:
  ① "고객이 3초 안에 핵심을 파악하지 못하면 이탈"
     — 2026 상세페이지 트렌드 (draph.art)
  ② "뷰티 리테일 모바일 트래픽 비중 78~84% (연령대별 차이)"
     — 복수 이커머스 리서치 (Salsify, PowerReviews 등)
  ③ "스크롤 한 번에 하나의 핵심 메시지만 전달하는 구조"
     — 숏폼 형식 강화 트렌드
  ④ "극단적 여백 + 대형 타이포" = 감각 전달의 기본
     — 3CE Mood Recipe 사례 (S5 레퍼런스)

경쟁사 대비:
  3CE   → 풀블리드 화보 + 캠페인 카피로 진입
  롬앤   → 제품 무드 컬러 배경 + GIF 텍스처로 진입
  NARS  → 모델 화보 풀블리드 → 브랜드 세계관 즉각 전달
  ───
  올리브영 PB → 화이트 배경 제품 컷으로 진입

영향:
  첫 3초에 이탈 → 하단 콘텐츠 도달률 급락
  브랜드 무드 인지 실패 → 가격 비교로 전환
```

#### P2. 발색·텍스처 정적 표현

```
현상:
  16색 팔레트의 발색이 팔 스와치 사진 1~2장으로만 표현
  매트/시머/글리터/베이크드 4종 텍스처 차이가 시각적으로 구분 불가

근거:
  ① "화장품: 발림성 표현"에 동영상/GIF 필수
     — 2026 상세페이지 트렌드 7번째 (draph.art)
  ② 올리브영 발색 비교 기능은 "립 제품"에 최적화
     → 아이섀도우 팔레트에는 텍스처 비교가 필요하나 미지원
     — mobiinside UX 분석 (2022)
  ③ "After Feel이 구매 결정 1위 요인"
     — K-뷰티 2026 Sensory Textures 트렌드
  ④ 리뷰어들의 자체 보완: "매트 3개, 시머 5개, 글리터 4개"
     → 공식 페이지가 하지 않는 텍스처 분류를 리뷰어가 대행
     — maedeia.com 웨이크메이크 리뷰

경쟁사 대비:
  롬앤   → GIF 애니메이션으로 질감·발색 직접 표현
  Fenty → 50+ 쉐이드를 피부 톤 필터로 시각화
  NARS  → 컬러 클릭 시 발색 미리보기 전환
  ───
  올리브영 PB → 정적 스와치 이미지 1~2장

영향:
  "실물이 어떤 느낌인지 손으로 느껴질 것 같다" 경험 불가
  → 리뷰 의존도 상승, 첫 구매 전환 저하
```

#### P3. 퍼스널컬러 매핑 부재

```
현상:
  웨이크메이크의 핵심 가치 = "퍼스널 컬러 기반 맞춤형 메이크업"
```
