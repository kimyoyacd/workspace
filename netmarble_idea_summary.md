## 왜 이 제안인가

HIVELAB은 넷마블과 함께 배너·영상·캠페인 소재를 제작해왔습니다.
그 과정에서 쌓인 **6,700개 이상의 제작 자산**, **IP별 성과 데이터**, **크리에이터 네트워크**가 있습니다.

지금까지는 "의뢰 → 제작 → 납품" 구조였습니다.
이 제안은 그 구조 위에 **AI를 얹어서**, 같은 팀이 같은 비용으로 **더 많이, 더 빠르게, 더 정확하게** 만들자는 것입니다.

새로운 파트너를 찾는 게 아닙니다.
**이미 함께하고 있는 HIVELAB이, AI로 더 잘 도와드릴 수 있습니다.**

## 3. 제안: 기존 제작에 AI 얹기 (추가 계약 없이 시작)

### 기본 아이디어

넷마블에 별도 제안서를 보내는 것이 아닙니다.
**지금 진행 중인 다음 납품물에 AI를 적용해서 보여드리겠습니다.**

| 기존 납품 | + AI 적용 | 넷마블이 체감하는 변화 |
|----------|----------|---------------------|
| 배너 10개 제작·납품 | 원본 10개 + AI 변형 400개 함께 납품 | "같은 비용으로 40배?" |
| 완성본 전달 후 피드백 대기 | 기획+스토리보드+영상 동시 제출 | "A/B/C 중에 하나만 고르면 끝?" |
| 엑셀 성과 리포트 | AI 인사이트 + 다음 달 추천 1장 요약 | "다음에 뭘 만들지 답이 나오네" |
| 과거 소재 요청 → 하드디스크 뒤짐 | 자연어 검색: "붉은 머리 남성 전사" → 즉시 | "10년 자산을 바로 찾을 수 있어?" |

### 실행 방식

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│   현재: 의뢰 → 제작 → 납품 → 끝                                  │
│                                                                 │
│   제안: 의뢰 → 제작 → AI 확장 → 납품 + 인사이트 → 다음 의뢰 반영   │
│                    ↑                          │                  │
│                    └──────────────────────────┘                  │
│                      데이터가 쌓일수록 정확도 ↑                     │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

**비용 증가 없이** 기존 납품 품질과 볼륨을 높이면,
넷마블 담당자가 자연스럽게 "이걸 더 크게 하면 어떨까?"라고 묻게 됩니다.
그것이 업세일즈입니다.
