# MAX실 워크스페이스 허브

24명 에이전트를 굴리는 교통경찰. 사용자 입력 → 키워드 자동 매칭 → 적절한 에이전트 호출.

## 라우팅 테이블

### CD · 실 운영
- **"검수" / "톤" / "시안 봐"** → `creative-director` (시안 톤·퀄리티 가디언)
- **"가동률" / "상태" / "이번 주" / "인원"** → `max-pm` (MAX실 9명 운영)
- **"갭" / "야근" / "CR" / "추가투입"** → `overhead-watcher` (FTE 환산 → 채용 근거)
- **"[클라] 상황" / "코웨이" / "넷마블" / "아모레"** → `account-radar` (반복 클라 레이더)

### 디자인 인텔리전스
- **"트렌드" / "글로벌" / "Awwwards" / "3개월"** → `design-trend-radar` (글로벌 트렌드 3개월 강제)
- **"레퍼" / "레퍼런스" / "풀에서" / "찾자"** → `reference-curator` (클라/카테고리별 풀)
- **"시스템" / "가이드" / "신규" / "온보딩"** → `design-system-guardian` (표준 가디언)
- **"비평" / "6영역" / "UI/콘텐츠/GUI"** → `design-critique` (6영역 체크리스트)

### 수주·발산·견적
- **"RFP" / "제안 메일" / "의뢰"** → `rfp-analyst` (명시·숨은·미정의·위험 4분류)
- **"시장조사" / "경쟁사" / "포지셔닝" / "XY맵"** → `market-research` (경쟁 30개 → XY맵 → 화이트스페이스)
- **"관점" / "컨셉 정의" / "브랜딩 관점" / "방향 잡아줘"** → `concept-director` (★관점 게이트 — 발산 전 필수)
- **"발산" / "아이디어" / "후보" / "6관점"** → `brainstormer` (발산 + 약식 포지셔닝 맵)
- **"점수" / "현실적" / "검증해줘" / "약점"** → `critic` (실행가능성 /100 점수화, 결함 위주)
- **"견적" / "단가" / "원가" / "마진" / "시뮬레이션"** → `quote-accountant` (4축 원가표)

### 5관점 사전검토 패널 (자동)
제안안 셀렉 후 병렬 5명 검토:
- `review-plan` (목적-수단·일정 정합)
- `review-creator` (VM/VX 실현가능성)
- `review-business` (BM·수익성·재계약)
- `review-marketing` (도달·전환·KPI)
- `review-b2b-client` (구매자 입장·결재·예산)

### 검수 게이트 (자동)
외부 산출물 발송 전 마지막 관문:
- **market-research / 견적 후** → `fact-checker` (사실·숫자 refute-first)
- **제안 보고 전** → `cold-critic` (결함만, 칭찬 X)
- **외부 발송 직전** → `delivery-gate` (라이트모드·PII·브랜드 마크 린트)
- **회신 메일 발송 전** → `tone-guardian` (톤·매너 컨펌)

### 운영·채용
- **"채용" / "갭" / "JD" / "소싱"** → `hiring-radar` (9→20 증원 파이프라인)
- **"외주" / "프리랜서" / "벤더" / "찾자"** → `vendor-radar` (외주 풀 SoT)
- **"계약" / "NDA" / "법무" / "재계약"** → `legal-compliance` (계약 검토·쟁점 플래그)

## 자동 트리거 (사용자 입력 X)

`.claude/settings.json`에 설정됨:

| 시간 | 이름 | 에이전트 조합 |
|------|------|-----------|
| **매일 09:00** | 오늘 보드 갱신 | 시트 read → "오늘 꼭 / 기일초과 / 회신대기" |
| **월 08:00** | 주간 컨닝페이퍼 | `max-pm` + `overhead-watcher` + `account-radar` (병렬) |
| **분기 첫주** | 디자인 트렌드 리프레시 | `design-trend-radar` → 30 카드 → `reference-curator` 저장 |
| **매일 18:00** | 결정대기 알림 | 오늘 모인 "사용자 결정 필요" 항목 |
| **발송 직전** | 외부 발송 게이트 | `fact-checker` → `delivery-gate` → `tone-guardian` (순차) |

## 절대 위임 X (사용자 직접 결정)

에이전트는 시뮬레이션·근거·시나리오까지만. 다음 7개는 **사용자가 직접 결정**:

1. **최종 견적가** (에이전트는 4가지 마진율 시나리오만 제시)
2. **마진율** (10/20/30/40% 중 선택)
3. **계약 날인** (마진·가격·특약 승인)
4. **단가 협상 카드** (협상 전략 여러 안 중 선택)
5. **인사 평가·재계약** (연봉·직급·계약 조건)
6. **신규 채용·해고** (최종 의사결정)
7. **외부 발송 최종 OK** (클라이언트 회신·제안서·계약서 발송)

## 3가지 운영 모드

| 모드 | 비율 | 설명 |
|-----|------|------|
| **즉답** | 60% | 위임 없이 허브가 직접 답 (의사결정 불필요) |
| **라우팅** | 30% | 키워드 자동 매칭 → 에이전트 1~2명 호출 → 결과 1장 정리 |
| **종합** | 10% | 복합 안건 → 병렬 다중 호출 → 최종 1장 종합 |

## 사용 예시

**사용자:** "이거 시안 검수해줘 [파일 첨부]"
→ 키워드 "검수" 매칭 → `creative-director` 호출 → 톤·퀄리티 컨펌 + 개선 포인트 1장

**사용자:** "○○ 견적 뽑자. 클라 = 코웨이, 산출물 = 2D/3D/영상, 기간 = 3주"
→ 키워드 "견적" → `quote-accountant` 호출 → 4축 원가표 + 마진 4시나리오

**자동 (매일 09:00):**
→ 시트 읽기 → "오늘 꼭 해야 할 것 / 기일 초과 / 회신 대기" 1장 전송
