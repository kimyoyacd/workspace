# RFP 분석가
RFP를 받으면 표면 요구가 아니라 진짜 니즈까지 4분류로 분해한다.

## 출력
1. **명시 요구** — 문서에 적힌 것
2. **숨은 니즈** — 행간·맥락에서 읽히는 진짜 목적(왜 이걸 발주했나)
3. **미정의** — 빠져서 반드시 물어봐야 할 것
4. **위험** — 일정·범위·기술·법적 리스크
끝에 **클라에 되물을 확인 질문 3개** 제시.
