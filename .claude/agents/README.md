# 에이전트 운영 가이드

당신의 24가지 커스텀 에이전트는 6개 그룹으로 분류되며, **Logical Mode ↔ Creative Mode**의 이원화 사고방식을 실현합니다.

---

## ① 전략·수주 (Logical Mode)

RFP 수주부터 포지셔닝까지, 데이터와 논리 기반의 의사결정을 지원합니다.

### `rfp-analyst` (RFP 분석가)
- **역할**: RFP 또는 제안요청서를 4가지로 분류
- **출력**: 명시 요구사항 → 숨은 니즈 → 미정의 영역 → 위험 요소
- **트리거**: "RFP 분석", "과업 분석", "제안요청서 봐줘"

### `market-research` (시장조사)
- **역할**: 경쟁사 N개를 XY 포지셔닝 맵으로 정리
- **출력**: 포지셔닝 맵 + 빈자리(화이트스페이스) + 출처 명시
- **트리거**: "시장조사", "경쟁사 분석", "포지셔닝 맵", "화이트스페이스"

### `concept-director` (컨셉 디렉터) ⭐
- **역할**: 사전 관점 게이트 — 핵심 제품/타이틀 → 브랜드 관점 한 문장
- **출력**: 전체 매체(평면·영상·UI 등)가 상속할 관점 정의
- **트리거**: "관점 잡아줘", "컨셉 정의해줘", "핵심가치 한 문장"
- **중요**: 모든 디자인 작업 **이전에** 호출

### `critic` (냉정한 비평가)
- **역할**: 아이디어를 비용·기간·역량·실현가능·설득력 관점에서 점수화
- **출력**: 각 항목별 점수(1-5) + 리스크 지적 + 우선순위
- **트리거**: "이거 현실적으로 돼?", "평가해줘", "리스크 짚어줘"

---

## ② 크리에이티브 발상 (Creative Mode)

전략 위에서 감성적, 예술적 아이디어를 발산하고 시각화합니다.

### `brainstormer` (브레인스토머)
- **역할**: 자유로운 발산 + 방향별 키워드·약식 포지셔닝 축 제시
- **출력**: 다양한 방향성의 아이디어 군집
- **트리거**: "아이디어 줘", "브레인스토밍", "발산해줘", "방향 좀 뽑아줘"

### `moodboard-builder` (무드보드 빌더)
- **역할**: 컨셉별 무드보드를 레퍼런스 + 생성 이미지로 조합
- **출력**: 비주얼 무드 + 톤 정의 + 색감·타이포그래피 가이드
- **트리거**: "무드보드", "분위기 잡아줘", "톤 보드 만들어"

### `visual-generator` (비주얼 제너레이터)
- **역할**: 키비주얼, 목업, 스타일프레임 이미지 생성 (Higgsfield 사용)
- **출력**: 생성된 이미지 + 사용된 프롬프트
- **트리거**: "키비주얼 뽑아", "이미지 생성", "목업 만들어", "스타일프레임"

### `storyboard-maker` (스토리보드 메이커) ⭐ 영상 전담
- **역할**: 영상·모션 스토리보드 + 시나리오 작성 (Higgsfield 통합)
- **출력**: 컷별 구성 + 모션 의도 + 음향 가이드
- **트리거**: "스토리보드", "영상 콘티", "모션 기획", "콘티 짜줘"

---

## ③ 디자인 인텔리전스 (Research & Critique)

시장과 내부 기준을 기반으로 창의물의 품질을 검증합니다.

### `design-trend-radar` (트렌드 레이더)
- **역할**: 최근 3개월 글로벌 디자인 트렌드 추출
- **출력**: 트렌드 3-5개 + 출처 URL + 우리 작업과의 관련성
- **트리거**: "트렌드", "요즘 뭐가 유행", "레퍼런스 트렌드"

### `reference-curator` (레퍼런스 큐레이터)
- **역할**: 클라이언트·카테고리별 레퍼런스 모아 큐레이션
- **출력**: 시안별 참고 사례 5-10개 + 출처 URL + 선정 이유
- **트리거**: "레퍼런스 찾아줘", "참고 사례", "벤치마크 모아줘"

### `design-system-guardian` (시스템 가디언)
- **역할**: 시안이 사내 디자인 시스템(컬러·타입·컴포넌트·그리드) 준수하는지 점검
- **출력**: 준수율 % + 위반 항목 + 교정 제안
- **트리거**: "시스템 점검", "가이드 맞나", "토큰 확인"

### `design-critique` (디자인 비평가)
- **역할**: 시안을 UI·콘텐츠·GUI·UA·상세·브랜딩 6영역 검수
- **출력**: 각 영역별 🟢(완성) 🟡(보완) 🔴(재작업) 판정 + 상세 지적
- **트리거**: "시안 검수", "이 시안 봐줘", "디자인 점검", "QC 봐줘"

---

## ④ 디자인 구현 (Production)

크리에이티브 결정을 실제 산출물로 변환합니다.

### `figma-bridge` (피그마 브릿지)
- **역할**: 코드 ↔ Figma 양방향 동기화
- **출력**: Figma 파일 또는 코드 컴포넌트
- **트리거**: "Figma 보내", "Figma 가져와", "피그마 동기화"

### `creative-director` (크리에이티브 디렉터)
- **역할**: 최종 시안의 톤·퀄리티 점검 + 온보딩 팩 생성
- **출력**: 톤 검수 결과 + 온보딩 표준 1장
- **트리거**: "CD 검수", "톤 최종 점검", "온보딩 만들어줘"

---

## ⑤ 검수 게이트 (Quality Gate)

발송 직전 다단계 검증으로 오류와 톤 편차를 방지합니다.

### `fact-checker` (팩트 체커)
- **역할**: 사실·숫자·인용을 반증 먼저(refute-first)로 검증
- **출력**: 사실성 판정 (✅ 정확 / ⚠️ 유보 / ❌ 오류) + 근거
- **트리거**: "팩트체크", "이 숫자 맞아", "출처 확인"

### `review-panel` (5관점 패널)
- **역할**: 제안안을 기획·크리에이터·사업·마케팅·B2B고객 5관점으로 병렬 검토
- **출력**: 관점별 의견 + 종합 점수 + 통과 여부
- **트리거**: "5관점", "패널 검토", "다각도 검토"

### `delivery-gate` (딜리버리 게이트)
- **역할**: 라이트모드·PII·텍스트 잘림·마크다운·링크 결정론적 린트 (발송 직전 자동)
- **출력**: 통과/불통과 + 위반 항목 명시
- **트리거**: "발송 전 점검", "딜리버리 체크"

### `tone-guardian` (톤 가디언)
- **역할**: 회신·메일 초안의 톤·매너 컨펌·교정
- **출력**: 톤 판정 + 수정본
- **트리거**: "톤 봐줘", "이 메일 괜찮아", "회신 다듬어줘"

## ⑥ 운영·매출·조직 (Operations)

조직 운영, 재정, 인사를 통합 관리합니다.

### `project-manager` (프로젝트 매니저)
- **역할**: 진행 현황·매출 달성율·가동률·클라이언트 히스토리 정리 (노션·시트 read-only)
- **출력**: 대시보드 형식 요약 + 위험 신호
- **트리거**: "이번 주 현황", "가동률", "매출 달성율"

### `quote-accountant` (견적 회계사)
- **역할**: 노임단가표 기반 직접인건비 원가표 + 마진 시나리오
- **출력**: 원가표 + 마진율별 시나리오 3-5개 (최종 선택은 사용자)
- **트리거**: "견적", "견적 뽑자", "단가 계산"
- **특기**: UI Design만 우리실 매출로 분리 표기

### `account-radar` (어카운트 레이더)
- **역할**: 반복 클라이언트의 히스토리·거래 패턴·리스크 요약 (미팅 브리핑용)
- **출력**: 클라 프로필 + 거래 히스토리 타임라인 + 주의 사항
- **트리거**: "클라 상황", "미팅 브리핑", "클라 히스토리"

### `hiring-radar` (채용 레이더)
- **역할**: 9→20명 증원 채용 파이프라인(JD·소싱·면접·오퍼) 정리
- **출력**: 단계별 진행 현황 + 병목 포인트 + 다음 액션
- **트리거**: "채용 상황", "JD 써줘", "채용 현황"

### `vendor-radar` (벤더 레이더)
- **역할**: 외주·프리랜서 풀 관리 + 견적 중 외주비 산정
- **출력**: 벤더별 역량·단가·평판 + 프로젝트별 추천
- **트리거**: "외주 찾자", "프리랜서 풀", "외주비 계산"

### `legal-compliance` (법무)
- **역할**: 클라/외주/근로 계약·NDA의 쟁점·리스크 플래그 (자문 대체 아님)
- **출력**: 위험 항목 + 권장 조치 + 법무 자문 필요 여부
- **트리거**: "계약서 검토", "NDA 봐줘", "독소조항"

## 에이전트 선택 흐름도

```
프로젝트 시작
  ↓
1️⃣ RFP 분석? → rfp-analyst
  ↓
2️⃣ 시장 리서치? → market-research
  ↓
3️⃣ 브랜드 관점 정의 ⭐ → concept-director
  ↓
4️⃣ 크리에이티브 발산? → brainstormer
  ↓
5️⃣ 무드·톤 정의? → moodboard-builder
  ↓
6️⃣ 영상 필요? → storyboard-maker ⭐
  ↓
7️⃣ 이미지 생성? → visual-generator
  ↓
8️⃣ 트렌드/레퍼런스? → design-trend-radar, reference-curator
  ↓
9️⃣ 시안 검수? → design-critique, design-system-guardian
  ↓
🔟 최종 톤 점검? → creative-director
  ↓
1️⃣1️⃣ 발송 전 검증? → fact-checker, delivery-gate, tone-guardian
  ↓
완료
```

---

## 에이전트 간 협업

### Logical → Creative 흐름
```
rfp-analyst → market-research → concept-director → brainstormer
     (데이터 수집)     (시장 분석)    (전략 고정)    (감성 발산)
```

### Creative → Production 흐름
```
brainstormer → moodboard-builder → storyboard-maker → visual-generator
   (발상)        (톤 정의)         (영상 계획)        (자산 생성)
```

### Quality Gate 흐름
```
creative-director → design-critique → review-panel → delivery-gate → tone-guardian
  (톤 최종 확정)    (6영역 검수)      (다각도 검토)   (최종 린트)      (소통 톤 점검)
```
