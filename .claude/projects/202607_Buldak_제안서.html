Executive Summary

불닭은 단순한 '매운 라면'에서 글로벌 K-푸드 아이콘으로 격상했습니다. 글로벌 누적 200억 뷰의 "Spicy or Nothing" 정신을 한국 자사몰에 전면 이식하는 것이 2차 리뉴얼의 핵심입니다.

1차 리뉴얼의 성과: 시각적 현대화 + 기본 커머스 구조 ✅
2차 리뉴얼의 목표: 강렬함·성격·도전 정신을 UX로 번역 + 커뮤니티 중심화
200억+ 글로벌 콘텐츠 뷰
50%+ 재방문율 목표

1. 브랜드 컨텍스트 (Buldak)

브랜드 정의

글로벌 K-푸드 커머스 플랫폼에서 강렬한 도전정신을 추구하는 Gen Z·밀레니얼 전용 'Buldak' 자사몰

핵심 철학: "Spicy or Nothing"

중간이 없다. 도전할 것인가, 아예 안 할 것인가.
매움으로 표현하는 것은 용기, 한계 돌파, 강렬한 정체성.
당신이 평범하기를 원하지 않는다면, Buldak이 당신의 선택지다.

타겟층 특성

  • 강렬함과 도전에 진심: 매운맛 자체가 아니라 "용기·도전·한계 돌파"의 신호
  • 글로벌 한류 소비자: K-pop·한류 문화에 개방적, TikTok에서 챌린지 영상 자생 생성
  • 경험과 기록의 가치: SNS 공유 욕구 높음, 친구들에게 증명하고 싶음
  • 커뮤니티 소속감: 같은 열정의 사람들과의 연결, 게이미피케이션에 반응
  • 콘텐츠 소비자 겸 제작자: 자신도 브랜드의 일부가 되고 싶음

브랜드 기능 3층 구조

계층 역할 UX 요소
커머스 "쇼핑" — 빠르고 직관적인 구매 PLP·PDP·결제 효율
경험·스토리 "빠져드는 것" — 라이프스타일 제안 Recipe·News·About·콘텐츠 허브
커뮤니티 "함께하는 것" — 정체성 표현 Scoville Club·리뷰·게이미피케이션

2. Wide Research — 카테고리 30개 브랜드 지도

매운맛·강렬함(Spicy or Nothing) 카테고리 국내 15 + 해외 15 브랜드를 8속성(이름·로고·소재지·오픈시기·스토리·타겟·기능·목적)으로 조사하고, 인사이트에 따라 3개 타입으로 분류했다.

📋 명시된 요구사항 (명확함)

✅ 명시

6개 범위의 디자인 리뉴얼:

  1. 브랜드 BI 방향성에 맞는 사이트 디자인 + 컨셉 도출
  2. UX/UI 분석 및 개선
  3. Home / PLP / PDP 디자인 리뉴얼
  4. Recipe 페이지
  5. News / Blog 페이지 개선
  6. About / My Page 개선

예산: 1억 원 (범위 미상)

💡 숨은 니즈 (읽어낼 것)

🔍 숨은
  • 1차와의 경계 불분명: "2차"라는 표현 = 1차 산출물 인수인계 가능성 높음. 하지만 수주처·시스템 미명시.
  • 브랜드 강화 ≠ 단순 비주얼: "아이덴티티 강화"는 BI 가이드·권한·글로벌 톤 적용을 의미. 자유도 높음 vs 제약 있음의 양극.
  • 커머스 + 콘텐츠 허브화: Recipe·News·Blog·My Page 변경은 단순 시각이 아니라 기능·운영 구조 변경. CMS 템플릿·에디터 제약 존재 가능.
  • 글로벌 아이콘화의 한국 지현: "Spicy or Nothing"을 KR 자사몰에서 어떻게 풀 것인가 = 전략·관점의 영역. 클라이언트도 답이 없을 수 있음.
  • 커뮤니티 기반 재구매: "My Page 개선"은 게이미피케이션·레벨·뱃지 같은 새로운 기능 추가를 암시.

❓ 미정의 항목 (확인 필수)

⚠️ 미정의
  • 1차 리뉴얼의 주체: 누가 만들었나? (삼양 내부 / 외부 에이전시 / 우리?) → 산출물 인수인계 가능성
  • 1억의 범위: 디자인만? 퍼블·개발 포함? VAT별도/포함?
  • BI 가이드 존재 여부: 있으면 권한(준수 vs 재해석)은? 없으면 우리가 정의?
  • PC / Mobile: 둘 다? 반응형 1벌?
  • 산출물 정의: 피그마 파일만? HTML/CSS 퍼블? 개발 연동?
  • 초기 콘텐츠: Recipe/News/Blog 실콘텐츠 누가 준비? (우리 vs 클라이언트)
  • 자사몰 플랫폼: 자사 개발? Cafe24? Shopify? → 디자인 제약 직결

🚩 위험 요소 (프로젝트 영향)

🔴 위험
  • 1차와의 연속성 붕괴: 1차 시스템 미인수 → 2차에서 컴포넌트 재제작 → 숨은 공수 발생. 영향: 일정·비용 +20~30%
  • BI 권한 모호: "가이드 준수만" vs "웹 시스템 신규 구축" → 디자인 자유도·공수 극단적 차이. 영향: 제안 범위 결정 불가
  • 1억에 개발 포함: 총액은 크나 UI Design 직군만 우리실 매출 → 실적 얇아짐. 영향: 채산성 악화
  • 커머스 결제/회원 제약: 플랫폼 솔루션의 UI 제약이 크면 PDP·My Page 디자인 한계. 영향: 제안 설득력 ↓
  • 7.30 오픈 직후 2차 착수: 1차 버그·피드백과 2차 리소스 충돌 가능. 영향: 일정 압박

경험 설계 (페이지별)

HOME — "당신은 몇 도(℃)인가?"

히어로: 불꽃 비주얼 + "Spicy or Nothing" ↓
Scoville Club 심리테스트 (10개 문제) ↓
결과: 자신의 "Spicy Level" 뱃지 ↓
맞춤 제품 추천 + 신상 + 유저 챌린지 영상

PLP — "강렬한 선택의 현장"

필터: 불꽃 개수로 맵기 표현 + 자신의 테스트 결과 비교
카드: 이미지 > 맵기 > 제품명 순 위계
액션: 구매 + 하트 + SNS 공유

PDP — "도전의 증명"

히어로: 제품 사진 큼 + 불꽃 레벨
스펙: 매운맛을 감정언어로 ("용감함이 필요합니다")
공동구매: "300명이 이미 도전중"
결제 위: "구매 후 당신의 영상을 공유하면?" → Scoville Club 포인트

RECIPE — "불닭을 더 극단적으로"

필터: 당신의 Level별로 보기
카드: 이미지 + 비디오 + 재료(직구매링크)
UGC: "나는 이렇게 했어" 피드
상단 고정: 에디터 추천 조합 "한 번에 담기"

SCOVILLE CLUB — "내 정체성을 확인하는 곳"

프로필: Level + 뱃지 + 랭킹
대시보드: 도전 기록 + 다음 Level까지의 진행도
피드: 같은 Level 사람들의 리뷰·영상
미션: 신상 도전 + 영상 올리기 (상위 3개 선정)

ABOUT / NEWS — "우리가 왜 이렇게 강렬한가"

ABOUT: 불닭의 역사 + 철학 + 글로벌 커뮤니티
NEWS: 신제품 스토리 + 글로벌 뉴스 + 유저 영상 셀렉션

B. 1차 ↔ 2차 경계 (가장 중요)
