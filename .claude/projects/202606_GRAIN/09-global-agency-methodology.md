# GRAIN · 글로벌 톱티어 브랜딩 에이전시 방법론 리서치
> 2026-07-01 · design-trend-radar 에이전트 산출

## 조사한 에이전시 (10곳, 확인 가능한 것 위주)

1. **Porto Rocha × Google DeepMind (YOYOYO)** — 형태탐색→아이덴티티 도출(오브젝트 논리에서 직접 발생)→컨텍스트 확장→움직임. AI가 전 과정 생성엔진, 디자이너는 큐레이터 겸 프롬프트 디렉터. [출처](https://www.transformmagazine.net/articles/2026/porto-rocha-tests-limits-of-ai-with-yoyoyo-project/)
2. **Pentagram (Performance.gov)** — 일러스트레이터가 스타일 확립(페인트·테이프 콜라주) → Midjourney로 1500개 이상 스케일. "스타일은 인간, 스케일링은 AI". [출처](https://www.creativebloq.com/design/ive-never-seen-anything-like-this-before-paula-scher-on-the-backlash-to-pentagrams-ai-government-website-design)
3. **COLLINS** — 브랜드를 "기업가치 레버"로 재정의, 자체 AI 자산엔진 "Aspect OS" (기술 미공개)
4. **Wolff Olins (LG)** — AI 기능나열 대신 "삶이 어떻게 나아지는가" 인간 경험 중심. 브랜드 언어 단계엔 AI 미사용 — 역방향 전략
5. **Koto (Amazon 리브랜드)** — 진단(파편화 문제 명문화)→핵심자산 우선순위화(로고/타입/컬러 별도 트랙)→커스텀 타입페이스→글로벌 아키텍처. [출처](https://www.dezeen.com/2025/05/09/amazon-rebrand-2025-koto-xcm/)
6. **Bureau Borschë (On Athletics Club)** — 오브젝트/컨텍스트의 물리적 형태(트랙 곡선)를 로고 형태로 직접 전사. YOYOYO와 동일 원리
7. **Athletics (IBM)** — "기회 언어화→실사용 프레임워크" 원칙, 모션·사운드·3D를 처음부터 통합하는 멀티센서리 접근
8. Character, Base Design, Face37 — 최신 케이스 확인 안됨 (프로파일 제외)

## 공통 방법론 패턴 5가지

1. **물리적 논리를 먼저 확정한다 (Form/Logic First)** — 컨셉 카피보다 "제품이 물리적으로 어떻게 작동하는가"를 먼저 확정하고 거기서 아이덴티티 도출
2. **스타일 오너십은 인간, 스케일링은 AI (Style Anchor → AI Scale)** — AI에게 "스타일 발명"을 맡기지 않고, 인간이 만든 레퍼런스 1~수개를 "정답 스타일"로 고정 후 AI로 확장
3. **기능적 은유(Functional Metaphor)를 먼저 찾고 시각 시스템으로 번역** — "이 브랜드가 무엇을 하는가"를 한 문장 은유로 고정한 뒤에만 비주얼 발산 (concept-director의 "관점 한 문장" 게이트와 일치하는 업계 표준)
4. **파편화 진단 → 자산 단위 해체 → 규칙 기반 재조립** — 자산(로고/타입/컬러/모션)을 독립 트랙으로 분리해 각각 병렬 발전 후 조합
5. **검증은 실사용 시나리오로 (Context Stress-Test)** — 순수 비주얼 완성도가 아니라 다양한 환경·조명에서의 일관성, 실제 물리적 제약 통과 여부로 품질 검증

## 우리가 안 하고 있지만 이들이 공통으로 하는 것 (GRAIN 파이프라인 개선점)

1. **"기능적 은유 한 문장" 고정 단계 부재** — Manus 1과 Manus 2 사이에 "이 제품이 어떻게 작동하는가"(회전·리듬 등 물리적 은유)를 명문화하는 게이트가 없음
2. **"스타일 앵커 고정 후 AI 스케일" 워크플로우 미도입** — 지금은 바로 키비주얼을 생성하지만, 먼저 1-3개 "정답 스타일" 레퍼런스를 확정하고 그 값을 프롬프트에 못박은 뒤 대량 변주해야 일관성이 확보됨
3. **자산 트랙 분리 후 재조립 프로세스 부재** — 형태/타이포/컬러/모션을 독립 트랙으로 각각 병렬 생성 후 최적 조합을 나중에 조립하는 방식 미도입
4. **실사용 맥락 스트레스 테스트 단계 부재** — "다양한 배경·조명·구도에서도 브랜드 요소가 일관되게 인식되는가"를 정량 테스트하는 QC 단계 없음
