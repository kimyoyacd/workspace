## 주제 ① — 내가 진행한 AI 성과 리뷰

### 1-A. 개인 레벨 — "직무 경계 파괴 + 공수 200~400% 절감"은 이미 증명 완료 (~5월)

| 사례 | 해결한 업무 | AI 파이프라인 | 공수 절감 | 핵심 의미 |
|---|---|---|---|---|
| **코웨이 프리미엄 안마의자** | 7인치 터치 GUI 고도화용 브랜드 시각화 전략 100% 수립 + 시안 80% 직접 제작. 비전공 **3D 기기 해체 구조·인체 체형 맵핑**까지 단독 | NotebookLM(브랜드·인체공학·기술문서 분석) → Claude(UX 전략·기능 매핑) → Midjourney·Higgsfield(3D 컨셉·해체 비주얼) | **300%+** | 과거 3D 전문가 3~4인 TF가 수주 걸리던 융복합 작업을 디렉터 1인이 내재화. 대형 비딩에서 압도적 제안 경쟁력 |
| **배달의민족 아이콘 콘텐츠** | 아이콘 리뉴얼 기획 + 브랜드 카피 + **브랜드 영상 제작**까지 1인 100% 완수 (비전공 카피·모션 영역) | ChatGPT(카피·시나리오 구조화) → Midjourney(아이콘·에셋) → CapCut(타이포 모션·컷편집) | **400%+** | 기획-카피-디자인-영상 4~5인 TF가 한 달 매달릴 스케일을 1인 원스톱. 텍스트·시각·내러티브가 정렬된 최종 영상형 아웃풋 선제시 |
| **QQ Poker GUI** | 디자인 시스템 기반 GUI 단독 리드. 메인(A안) + 서브안 + **3D 오브제 80%** + 제작 가이드라인까지 | ChatGPT(서브 컨셉 논리·제작 가이드) → Midjourney(GUI 에셋 초안→고도화) | **200%+** | 담당자들이 비전문인 상황에서 디렉터 경험치 + AI로 메인·서브·가이드까지 단독 제어. 리소스 한계를 AI로 상쇄 |

**공통 패턴 (여기를 강조):**
- **비전공 영역(3D·영상·카피)을 AI 멀티 파이프라인으로 1인이 원스톱 커버.** "TF 3~5인 → 디렉터 1인"이 반복 재현됨.
- 절감의 본질은 인건비가 아니라 **제안 경쟁력**: 경쟁 PT에서 텍스트·정적 시안이 아니라 **완성형 3D/영상 아웃풋을 선제 제시**해 온도 차를 없앰.
- 단, 이 단계의 한계는 **"김효정 개인기"** — 재현·전파·품질 관리가 사람 한 명에 묶여 있음. (→ 1-B로 연결)

### 1-B. 6월 변곡점 — 개인기를 '시스템'으로 옮기기 시작 (올리브영 + MAX실 OS)

**(1) 올리브영 색조 PB 상세페이지 입찰 — 순발력이 아니라 재현 가능한 프로세스로**
- RFP·설명회 녹취·단가표를 **`rfp-analyst` 파이프라인**으로 분해 → 명시요구/숨은니즈/미정의/위험 4분류.
- 리스크 8건을 선제 도출(🔴 택1·택2 조건 모순, 색조 포폴 부재, 1주 내 4종 제출 / 🟡 최저단가 준용 함정, 촬영역량 필수, RA 법무검수, AI 경계선).
- **AI 역량을 차별화 카드로 전략화**: 발주 담당자가 "AI 포트폴리오 반갑게 검토"를 두 번 강조 → AI 연출컷·GIF를 포트폴리오 정식 섹션으로, 단 발색컷 AI 생성은 컴플라이언스 금지선으로 명확히 구분.
- **의미**: 코웨이·배민이 "손이 빠른 디렉터"였다면, 올리브영은 **"에이전트가 RFP를 읽고 리스크를 뽑는 프로세스"** — 나 없이도 굴러가는 방향의 첫 실전.

**(2) MAX실 OS 구축 — 개인 역량의 조직 자산화 착수**
- **에이전트 24종 / 6그룹**: 수주·전략 · 디자인 인텔리전스 · 디자인 생성 · 검수 게이트 · 운영·매출 · 조직·채용·법무.
- **자동화 트리거 맵**: 스케줄(매일 현황·주간 컨닝·분기 트렌드) / 이벤트(발송 직전 fact-check→delivery→tone, 시안 업로드 시 critique→guardian→CD) / 수동 호출. **리스크 티어(0/1/2)**로 검수 강도 차등.
- **데이터 연결(SoT)**: 노션 2026 Project DB(진행현황·조직구조·리소스로그) + 구글시트(가동률·실투입 시간) read-only 연결. **견적 SoT**(노임단가표·완료견적 학습).
- **브랜드 제안 풀파이프라인**: Manus 1(리서치·포지셔닝) → Manus 2(비주얼 아이덴티티) → Runable 3(시스템화·최종 덱), 게이트 통과제.
- **거버넌스 라인**: "절대 위임 금지"(최종 견적가·마진율·계약 날인·채용/해고·외부 발송 최종 OK는 항상 사람).

### 1-C. 솔직한 자기평가 — 여기서 신뢰를 얻는다

**잘된 것**
- 개인 레벨 ROI는 실증됨 (200~400% 절감, 다수 대형 비딩 경쟁력).
- 산발적 개인 툴 사용을 **"에이전트 + 트리거 + 데이터 + 게이트"의 뼈대**로 정리해 둠 → 재설계의 출발점이 이미 존재.

**아직 약한 것 (재설계가 필요한 진짜 이유)**
- ⚠️ **자동화가 세션 의존적** — 진짜 상시 자동화는 별도 러너 필요(아직 미연결).
- ⚠️ **피드백 루프 없음** — 에이전트 결과물에 팀이 레이팅·피드백을 남겨 공동 고도화하는 구조가 없음.
- ⚠️ **메모리가 텍스트로만 겉돎** — 클라이언트 미팅의 섬세한 브랜드 맥락(코웨이 Care 언어, 터치감각 등)이 문서로만 존재, AI가 검색·참조하는 자산이 아님.
- ⚠️ **개인 의존도 높음** — 여전히 상당 부분이 실장 1인에 묶여 있음. **이걸 푸는 게 주제 2.**

## 주제 ② — AI 시대에 적합한 향후 조직의 모습

### 2-A. 문제 정의 (넷마블 인사이트)

> **"넷마블에 다녀와서 확인한 건, 대형 조직도 결국 개인 툴 파편화를 끝내고 '조직 OS'로 간다는 것. 늦었지만, 우리는 9명이라 오히려 더 빠르게 구조를 재설계할 수 있다. 지금 이 로직 관리를 구체화하지 않으면 개인기의 천장에 계속 부딪힌다."**

- 개개인이 Claude·Midjourney 플러그인을 파편적으로 쓰면 → 1차 리뷰 기준이 모호해지고, 공동 고도화가 안 되고, 품질이 사람 편차에 좌우됨.
- 답은 도구를 더 사는 게 아니라 **공용 작업 기반(워크벤치) + 맥락 기억(메모리) + 실행·피드백 관제(파운드리) + 권한 가드레일(넥서스)**을 하나의 운영체계로 묶는 것.

### 2-B. 방향 — MAX실을 'AIOS(AI Operating System)'로 재설계 (Litmus 레퍼런스)

리트머스(김응진 대표) "AI 네이티브 회사" 모델의 4개 축에, 우리 현황·갭·액션을 매핑:

| Litmus AIOS 축 | 우리 현황 | 갭 | 대응 액션 |
|---|---|---|---|
| **① 넥서스** — 롤별 권한·데이터 격리, 격리 프로필·API 토큰 | 리스크 티어 0/1/2 + 절대위임금지 라인 有 | **롤(신규/주니어/시니어/리드)별 조회·생성·승인 권한 구조 없음** | **액션 4: 역할별 가드레일 + 시스템 온보딩** |
| **② 워크벤치 + 시멘틱 메모리** — 프로젝트별 맥락을 기억에 적재, 에이전트가 검색·참조 | `.claude/library` + `projects` 아카이브 有 | **맥락이 텍스트로만 겉돎 — 검색·참조되는 시멘틱 메모리 아님** | **액션 2: 프로젝트 '디자인 메모리' 구축(MCP 서빙)** |
| **③ 파운드리** — 실행 관제 + 결과물 레이팅·피드백 공동 고도화 | 트리거 맵·게이트 순서 정의됨 | **레이팅/피드백 루프 부재, 자동화 상시 러너 미연결** | **액션 1: Design Gatekeeper 공용 에이전트 격상 + 피드백 루프** |
| **④ 컨설턴트화** — 세일즈·PM·디자이너 경계 붕괴, 모두가 AI 레버리지로 메타 관점 솔루션 | 실장 개인이 이미 증명(코웨이·배민·QQ) | **팀 전파·표준화 안 됨** | **액션 3 + 목요일 AI 스터디 정례화** |

### 2-C. 실행 로드맵 — 4개 액션, 우선순위 순

**🎯 액션 1 (즉시 착수, 이번 주 목요일 AI 스터디) — Design Gatekeeper를 공용 워크벤치 에이전트로 격상**
- 개발 중인 Design Gatekeeper를 MAX실 공식 **공용 프로필 에이전트**로 등록.
- **밀러의 법칙·피츠의 법칙 등 BBD(행동 비즈니스 디자인) 가이드라인**을 컨텍스트로 중앙 주입(MD 모듈).
- **피드백 루프**: 주니어/시니어가 리뷰 결과물에 Good/Bad 레이팅 + 로그 → 매주 목요일 스터디에서 프롬프트 파이프라인 공동 튜닝.
- 효과: "1차 패스 디자인 리뷰 기준"이 사람마다 다르던 문제 해소 → **품질을 사람이 아니라 시스템이 보증.**

**🎯 액션 2 (차기 프로젝트 적용) — 프로젝트 '디자인 메모리' 자산화**
- 미팅 텍스트·카톡/슬랙 협의 중 **디자인 정책 변경 건을 1~2문장 시멘틱 요약**으로 프로젝트별 메모리에 적재.
- 에이전트가 Midjourney 프롬프트·Figma 초안 구조를 짤 때 이 메모리를 **MCP로 호출** → 클라이언트 최신 요구가 즉각 반영.
- 테스트 지표: 메모리 주입 전/후 **프롬프트·초안 퀄리티 정교화 정도**.

**🎯 액션 3 (수주율 직결) — 제안·세일즈 단계부터 BBD 시각화**
- 인바운드 문의 → AI가 해당 기업 최근 뉴스·재무·타깃 성향 자동 리서치 팩 생성(spawn).
- **질의서 자동 생성** 스킬로 기획 단계 놓칠 비즈 요구 선제 도출.
- 텍스트 제안·IA를 넘어 **BBD 원칙 적용 와이어프레임/초안 화면을 AI로 즉시 시각화 → 제안 미팅 데모.** 세일즈 단계에서부터 압도적 USP.

**🎯 액션 4 (증원과 직결) — 역할별 권한 가드레일 + 시스템 온보딩**
- 롤 정의: **신규/주니어 = 조회·기본 자산 생성** / **시니어·리드(게이트키퍼) = 최종 납품 에셋 승인 + 환경변수 제어.**
- 신규 합류 시 파편화된 SOP 교육 대신 **"디자인 워크벤치에 접속해 에이전트에게 물어보며 작업하라"** → 시스템 기반 초고속 온보딩.

**시간 축 (제안):**
- **0~30일**: 액션 1 가동(Gatekeeper 공용화 + BBD 모듈 + 목요일 튜닝 정례화). 상시 자동화 러너 1건 연결.
- **30~60일**: 액션 2를 진행 중 프로젝트 1건에 적용해 메모리 효과 검증. 액션 4 롤 권한 초안.
- **60~90일**: 액션 3을 신규 인바운드 1건에 실전 적용(리서치팩→질의서→와이어프레임 데모). 증원 온보딩에 시스템 적용.

### 2-D. 조직·인력 함의 (부대표가 가장 궁금해할 지점)

- **9→20 증원을 이 구조 위에 얹는다**: 온보딩 비용 급감 + **주니어도 시니어급 레버리지**를 즉시 확보 → 증원의 ROI가 선형이 아니라 계단식으로 올라감.
- **롤 재정의**: 세일즈/PM/디자이너 → **AI 레버리지 컨설턴트**(프로젝트 메타 관점에서 솔루션 제안). 채용 JD도 이 관점으로 재작성.
- **거버넌스는 오히려 강화**: 절대위임금지 라인(견적·계약·최종 OK는 사람) 유지 + 롤별 데이터 격리로 **보안과 생성 정확도를 동시에** 확보.
- **매출 연결**: UI Design 직군 기준 우리 실 매출 집계 구조는 그대로, 액션 3이 **수주율**을, 액션 1·2가 **제작 생산성/품질**을 끌어올림.

### 2-E. 예상 질문 대비 (부대표 반론 리허설)

| 예상 질문 | 답변 라인 |
|---|---|
| "이게 돈이 되나?" | 개인 레벨에서 이미 공수 200~400% 절감 실증. 조직화하면 이 효과가 9명 전체로 확산 + 액션 3으로 수주율 직접 상승. |
| "리스크·컴플라이언스는?" | 절대위임금지 라인 + 롤별 데이터 격리 + 게이트(fact/delivery/legal)로 관리. 올리브영 발색컷 AI 금지선처럼 경계 이미 운영 중. |
| "왜 지금인가?" | 넷마블서 확인했듯 대형사도 조직 OS로 이동 중. 늦을수록 개인기 천장·인력 편차가 누적. 9명일 때 재설계가 가장 싸다. |
| "증원이랑 상충 아닌가?" | 반대. 시스템 온보딩이 있어야 20명이 파편화 없이 흡수됨. 구조 없이 증원하면 관리비용만 폭증. |
| "당장 뭘 하면 되나?" | 이번 주 목요일 스터디에서 액션 1 착수. 승인 필요한 건 상시 자동화 러너 + 롤 권한 정책 두 가지. |
