## 주제 ① — 내가 진행한 AI 성과 리뷰

### 1-A. 개인 레벨 — "직무 경계 파괴 + 공수 200~400% 절감"은 이미 증명 완료 (~5월)

| 사례 | 해결한 업무 | AI 파이프라인 | 공수 절감 | 핵심 의미 |
|---|---|---|---|---|
| **코웨이 프리미엄 안마의자** | 7인치 터치 GUI 고도화용 브랜드 시각화 전략 100% 수립 + 시안 80% 직접 제작. 비전공 **3D 기기 해체 구조·인체 체형 맵핑**까지 단독 | NotebookLM(브랜드·인체공학·기술문서 분석) → Claude(UX 전략·기능 매핑) → Midjourney·Higgsfield(3D 컨셉·해체 비주얼) | **300%+** | 과거 3D 전문가 3~4인 TF가 수주 걸리던 융복합 작업을 디렉터 1인이 내재화. 대형 비딩에서 압도적 제안 경쟁력 |
| **배달의민족 아이콘 콘텐츠** | 아이콘 리뉴얼 기획 + 브랜드 카피 + **브랜드 영상 제작**까지 1인 100% 완수 (비전공 카피·모션 영역) | ChatGPT(카피·시나리오 구조화) → Midjourney(아이콘·에셋) → CapCut(타이포 모션·컷편집) | **400%+** | 기획-카피-디자인-영상 4~5인 TF가 한 달 매달릴 스케일을 1인 원스톱. 텍스트·시각·내러티브가 정렬된 최종 영상형 아웃풋 선제시 |
| **QQ Poker GUI** | 디자인 시스템 기반 GUI 단독 리드. 메인(A안) + 서브안 + **3D 오브제 80%** + 제작 가이드라인까지 | ChatGPT(서브 컨셉 논리·제작 가이드) → Midjourney(GUI 에셋 초안→고도화) | **200%+** | 담당자들이 비전문인 상황에서 디렉터 경험치 + AI로 메인·서브·가이드까지 단독 제어. 리소스 한계를 AI로 상쇄 |

**공통 패턴:** "TF 3~5인 → 디렉터 1인"이 반복 재현. 절감의 본질은 인건비가 아니라 **제안 경쟁력** — 완성형 3D/영상 아웃풋을 선제 제시해 경쟁 PT의 온도 차를 없앰. 단, 한계는 **"김효정 개인기"** — 재현·전파가 사람 1인에 묶여 있음.

### 1-C. 7월 10일 넷마블 실전 피드백 — 시스템화가 필요하다는 확증 (신규)

> 만족도·퀄리티·신뢰는 항상 최상위. 그런데 **AI 항목만 함께 쓰는 타 파트너사 대비 최하위.** 실력 문제가 아니라는 게 이번 미팅의 결론.

**넷마블이 짚은 4가지 원인**

| 원인 | 내용 | 실제 발언 |
|---|---|---|
| ① AI를 안 보이게 씀 | 배경·리소스에 AI를 이미 쓰지만 알리지 않아 "수작업"으로 보임 | "AI가 적용됐는지조차 알 수가 없다" |
| ② TO가 쪼개져 느림 (최다 반복) | 기획·이미지·영상 TO 분리 → 단계별 검수 핑퐁 → 급한 소재는 타사로 | "목적과 그림이 명확할 때만 여기 요청... 그 손길이 줄고 있다" |
| ③ 타사는 "키워드 → 완성물" | 키워드 한 줄이면 SB 없이 2~3일 내 영상 도착 | "한 줄 던졌더니 리소스도 안 줬는데 영상이 왔다" |
| ④ "AI=기획TO 필요"라는 심리적 장벽 | 실제론 SB에 "AI 활용해주세요" 한 줄이면 되는데 별도 TO가 필요하다는 인식 존재 | "AI를 쓰려면 뭔가 기획 TO가 필요할 것 같다" |

- ⚠️ **경고**: 넷마블 내부 제작팀도 AI로 키컷·키영상 제작을 이미 시작 — 넷마블을 가장 잘 아는 내부 경쟁자.
- **2단계 도약론(넷마블 본부장)**: 1단계 = "지금 쓰는 타 업체 수준"(AI 속도·원스톱)을 먼저 따라잡는 것. 우리의 알파(10년 눈높이·퀄리티)는 2단계인데, 1단계가 안 붙어서 **보여줄 무대 자체가 없는 상태.**
- **합의**: 하이브랩이 내부 구조(TO·프로세스·AI 대응)를 정리해 "콜사인"을 주면, 넷마블은 요청을 얼마든지 준다. 공동 목표 = 하반기 설문 AI 수치 개선.

**넷마블이 요구한 4가지 대응 (그대로 우리 실행안이 됨)**

| 무기 | 내용 |
|---|---|
| **A. AI 전담팀** ★대표지시 | 기획·이미지·영상 팀 안 거치고 키워드만 받으면 결과물까지 내는 별도 팀. 고난이도(시네마틱)는 숙련 인력 순차 진행 / 저난이도(UA·라이브 소재)는 당일~2일 출고 |
| **B. 통합 TO·원스톱** | 창구 하나 + PM 1명이 요청을 보고 "이건 바로 제작 / 이건 기획 먼저"를 내부 배분. SB 생략 가능(3~5초 단위 결과물 피드백 시대) |
| **C. AI를 "보이게"** | 납품 시 "이 부분은 AI로 생성/변형" 역으로 명시 태깅. 비용 거의 0, 즉효 |
| **D. 알파** | 1단계(경쟁사 수준 속도) 확보 후, 그 위에 넷마블 10년 눈높이를 얹는 것 |

### 1-D. 솔직한 자기평가

**잘된 것** — 개인 레벨 ROI는 실증됨(200~400% 절감). 개인 툴 사용을 "에이전트+트리거+데이터+게이트" 뼈대로 정리해 재설계 출발점을 이미 확보.

**아직 약한 것 — 넷마블이 정확히 짚었다**
- ⚠️ **TO 구조 자체가 문제였다** — 우리 견적 SoT(`quote-sot/to-templates.md`)를 열어보면 TO-01~05 전부 VM/VX/실장을 기능별 %로 쪼개는 구조. 넷마블이 "AI가 없던 시대의 관행"이라 지적한 게 **우리 자체 템플릿에 그대로 코드화되어 있음.**
- ⚠️ **단가 구조도 사람-시간 총합 기준** — `unit-rates.md`가 시간당 단가 × 배분율. "키워드→2~3일 완성물"을 파는 타사와는 애초에 다른 셈법으로 경쟁 중.
- ⚠️ AI를 이미 쓰면서도 안 보이게 씀, 자동화는 세션 의존적, 피드백 루프 부재, 메모리가 텍스트로만 겉돎.
- **결론**: 이건 조직도만 바꿔서 될 문제가 아니라 **프로세스(TO 템플릿)와 단가표(SoT)를 다시 설계**해야 풀리는 문제. → 주제 2의 핵심.

## 주제 ② — AI 시대에 적합한 향후 조직의 모습

### 2-A. 문제 정의 — 넷마블은 신호탄이다

> **"넷마블에서 확인한 건, 대형 조직도 결국 개인 툴 파편화를 끝내고 조직 OS로 간다는 것. 그리고 이건 게임만의 이야기가 아닙니다. 올리브영·현대·코웨이도 같은 니즈(시즌 물량, 채널별 베리에이션, 빠른 출고)를 갖고 있고, 타 대행사는 이미 AI 역량을 앞세워 그쪽에도 영업 중입니다. 지금 이 로직 관리를 구체화하지 않으면 개인기의 천장에 계속 부딪힙니다."**

- 넷마블의 "AI 점수 최하위" 원인 4가지 중 3가지(TO 분절·타사 속도·심리적 장벽)는 **일반 업계 고객사에도 동일하게 반복될 문제.**
- 우리가 정말 바꿔야 할 것은 에이전트 개수가 아니라 **① 프로젝트 프로세스 자체 ② 단가 경쟁력(가격표) 자체.** 이 둘이 오늘 미팅의 핵심 안건.

### 2-B. 일반 업계 기회 지도 — 같은 구조, 다른 제약

넷마블 대응(AI 전담팀·패스트트랙)은 한 번 만들면 게임 밖 고객사에도 그대로 쓸 수 있는 인프라. 업종만 바뀌면 "건드리면 안 되는 것"이 바뀔 뿐, 공식은 동일:

| | 게임 (넷마블) | 일반 기업 (올리브영·현대·코웨이) |
|---|---|---|
| 니즈 | 키워드→영상, UA 속도전 | **동일** — 시즌 물량, 채널별 베리에이션, 빠른 출고 |
| 제약 | IP 캐릭터 변형 금지 | 모델 얼굴(초상권)·제품 정확성·브랜드 가이드 |
| 심의 | 게임 등급·확률 표기 | 업종별 표시광고법(화장품 효능·가전 성능·자동차 제원) |
| AI 쓰는 곳 | 배경·이펙트·연출(캐릭터 외) | 배경·공간·무드(제품·모델 외) — **구조가 똑같다** |

**대표 3사 요약** (`hivelab_ai_general_industry_plan.html` 상세)
- **올리브영**: 시즌 세일 물량전 → AI 배경·연출·사이즈 자동변환 최적. 단, 제품 실물 왜곡 금지 + 표시광고법 심의.
- **현대**: 로케이션 촬영을 AI 배경으로 대체(실차 촬영비 절감). 단, 차량 형상 왜곡 절대 금지 + 그룹 가이드 사전 협의.
- **코웨이**: 라이프스타일 컷 AI 배경 무한 확장. 단, 제품 형태·패널 표기 정확성 + 모델 초상권.
- **+ 사이트·UI 트랙**: 이벤트 페이지·상세페이지·기업 홈페이지도 AI 대상 — **VX팀이 UI 경험 많아 홈그라운드.** "배너 세트 + 이벤트 페이지" 묶음 제안이 타사 대비 차별화 포인트.

### 2-C. 프로젝트 프로세스 재설계 (신규 핵심)

**Before — 우리 자체 TO 템플릿이 넷마블이 지적한 그 구조**

`quote-sot/to-templates.md`를 그대로 인용하면:

```
TO-01 UI/UX 웹사이트: VM시니어 80% + VM주니어 60% + VX시니어 40% + 실장 20%
TO-02 모션/영상:      VM시니어 100% + VM주니어 80% + 실장 10%
TO-04 브랜드 아이덴티티: VM시니어 80% + VM주니어 60% + VX시니어 40% + 실장 30%
```

→ 전부 **기능(직군)별로 쪼개 % 배분하고 순차 진행**하는 구조. 넷마블이 말한 "이미지·영상·기획 TO 분리는 AI가 없던 시대의 관행"이 정확히 이것.

**After — 난이도 이원화 + 통합 창구**

| 트랙 | 대상 | 처리 방식 | 담당 |
|---|---|---|---|
| **패스트트랙 (신규 TO-06)** | UA·라이브 소재, 시즌 배너 베리에이션, 반복 상세페이지 | 창구 1개 + PM 1명이 즉시 라우팅 → SB 생략 → AI 전담이 당일~2일 출고 → 결과물 단위 피드백(고르기) | AI 전담팀 |
| **정교트랙 (기존 TO-01~05 유지)** | 시네마틱, 브랜드 아이덴티티, 하이엔드 캠페인 | 기존 순차 TF 방식 유지 — 여기가 "알파"를 파는 자리 | VM/VX/실장 TF |

- **창구 통합**: "어느 TO에 넣을지" 고민을 클라이언트가 하지 않게 — PM 1명이 받아서 고/저난이도로 즉시 분류.
- **SB 생략 조건 명문화**: 반복성 높고 결과물 단위로 3~5초씩 보며 피드백 가능한 건은 스토리보드 없이 바로 제작 착수.
- **AI 보이게 태깅**: 납품물에 "이 부분 AI 생성/변형" 명시 — 비용 거의 0, 즉시 시행 가능.
- 이 라우팅 로직이 **일반 업계(올리브영 이벤트 페이지 등)에도 그대로 이식** — 한 번 설계해서 두 시장에서 사용.

### 2-D. 단가 경쟁력 재설계 (신규 핵심)

**Before — `unit-rates.md`는 사람-시간 총합 기준**

```
VM 시니어 ○○,000원/h · VM 주니어 ○○,000원/h · VX 시니어 ○○,000원/h · 실장 ○○,000원/h
→ 견적 = TO 템플릿의 %배분 × 시간당 단가 × 기간
```

`client-rate-policy.md`의 등급(Standard/Preferred/Strategic)도 **"관계 연차"**로만 우대율이 갈릴 뿐, **속도·AI 활용도와는 무관.** 반면 넷마블이 만난 타사는 "키워드 한 줄 → 2~3일" — 애초에 셈법이 다른 상품을 판다.

**After — 속도를 상품화한다**

| 항목 | 방향 |
|---|---|
| **패스트트랙 건단가 신설** | 시간당 단가가 아니라 **결과물 단위 건단가**(예: UA 소재 1건, 배너 세트 1건)로 별도 상품화. `unit-rates.md` "건단가 참고" 섹션 확장. |
| **등급에 SLA(속도) 얹기** | Standard/Preferred/Strategic 우대율에 **AI 대응 속도**를 추가 축으로: Strategic=당일 출고 포함, Preferred=2일, Standard=표준 기간. 관계뿐 아니라 속도로도 등급을 정당화. |
| **볼륨/구독형 옵션** | 반복 물량이 많은 고객(올리브영 시즌 배너, 넷마블 UA 소재)엔 건별 견적보다 **패키지·구독형**이 우리도 예측 가능하고 고객도 유리. |
| **알파는 여전히 시간단가로 방어** | 정교트랙(시네마틱·브랜드)은 기존 시간당 단가 유지 — 여기가 프리미엄이 정당화되는 자리. 패스트트랙 저가 경쟁에 전체 단가를 끌려가지 않도록 **두 상품을 명확히 분리해서 파는 것이 핵심.** |
| **올리브영 사례가 이미 경고** | 최저단가 준용 구조 + 라이선스 무기한 포함 조건 — 단가 하한선을 미리 정하지 않으면 협상에서 끌려감. 패스트트랙 상품화가 이 협상의 방어선이 될 수 있음. |

### 2-E. 참고 프레임 — Litmus AIOS 4축 매핑

리트머스(김응진 대표) "AI 네이티브 회사" 모델. 프로세스·단가 재설계(2-C, 2-D)를 뒷받침하는 인프라 축으로 참고:

| 축 | 우리 현황 | 갭 | 연결 |
|---|---|---|---|
| ① 넥서스(권한·데이터 격리) | 리스크 티어 0/1/2 有 | 롤별 조회/생성/승인 권한 없음 | 패스트트랙 vs 정교트랙 담당 권한 분리에 활용 |
| ② 워크벤치+시멘틱 메모리 | library+projects 아카이브 有 | 검색·참조되는 메모리 아님 | 클라이언트별 AI 가능/금지 지도를 메모리화 |
| ③ 파운드리(실행관제+피드백) | 트리거 맵 정의됨 | 레이팅 루프·상시 러너 부재 | AI 전담팀 결과물 관제에 직결 |
| ④ 컨설턴트화 | 실장 개인이 이미 증명 | 팀 전파 안 됨 | 목요일 스터디로 전파 |

### 2-F. 통합 실행 로드맵

**즉시 · 이번 주**
- AI 보이게 태깅 시행 (다음 납품물부터, 비용 0)
- AI 전담팀 신설안 초안 — 고/저난이도 라인 분리, 인원 구성
- 타이틀·클라이언트별 **AI 허용 범위 표** 작성 (넷마블 확인 요청 + 올리브영·현대·코웨이 착수)
- **TO-06(패스트트랙) 템플릿 초안** — `to-templates.md`에 신설 제안

**단기 · 2주**
- 창구 통합 설계 — PM 1명 라우팅 규칙 + 영업과 정산 방식 협의
- 원스톱 파일럿 준비 — 넷마블 타이틀 1개, 키워드→영상 2~3일, 4주 시범
- **패스트트랙 건단가 초안** — `unit-rates.md` 확장 + 등급별 SLA 설계
- 일반 업계 심의·권리 체크리스트(화장품·가전·자동차 표시광고, 초상권)

**30~60일**
- 패스트트랙을 진행 중 프로젝트 1건에 실전 적용해 속도·품질 검증
- 일반 업계 업종별 데모 1건 제작(가상 브랜드로 권리 이슈 없이)
- 등급별 SLA를 실제 계약서 조항으로 시범 반영

**60~90일**
- 기존 고객 대상 AI 제안서(데모+파일럿) 발송
- 정교트랙 알파(고퀄리티)를 실제로 보여줄 무대 확보 — 넷마블 2단계 도약론 실현
- 증원 온보딩에 패스트트랙/정교트랙 구조 그대로 적용

### 2-G. 조직·인력 함의 (9→20)

- **AI 전담팀이 곧 증원의 형태다** — 신규 인력을 기존 TF에 얹는 게 아니라, 패스트트랙 전담 라인으로 우선 배치 → 온보딩 즉시 실전 투입 가능.
- **롤 재정의**: 세일즈/PM/디자이너 → AI 레버리지 컨설턴트. PM은 이제 "라우팅 판단자"(고/저난이도 분류)라는 명확한 역할을 가짐.
- **거버넌스는 강화**: 절대위임금지 라인(견적·계약·최종 OK는 사람) 유지. 패스트트랙이라도 심의·권리 체크는 사람이 최종 확인.
- **매출 연결**: 패스트트랙(볼륨) + 정교트랙(알파 프리미엄) 두 상품 구조가 매출 다각화 + 가동률 안정(게임 비수기에 일반업계 물량으로 채움)으로 직결.

### 2-H. 내부 결정 필요 안건 (부대표와 이 자리에서 정할 것)

| 안건 | 선택지 | 추천 |
|---|---|---|
| 패스트트랙 첫 적용처 | (a) 넷마블 (b) 올리브영 (c) 동시 | (a) 먼저 — 방어가 급함. 검증 후 (b)로 확장 |
| AI 전담팀 리소스 배분 | (a) 게임 우선 (b) 비율 고정 (c) 시즌 유동 | (a)로 시작 → 일반업계 첫 계약 나오면 비율 전환 |
| 패스트트랙 과금 방식 | (a) TO형 (b) 건별 (c) 구독형 | 파일럿은 (b) → 반복 물량 확인되면 (c) 제안, 영업과 협의 |
| 정교트랙 단가 방어선 | (a) 유지 (b) 인하 | (a) — 패스트트랙 저가 경쟁에 알파 단가를 끌어내리지 않는다 |

### 2-I. 예상 질문 대비

| 예상 질문 | 답변 라인 |
|---|---|
| "이게 돈이 되나?" | 개인 레벨 200~400% 절감 실증 + 넷마블이 직접 요청한 구조. 패스트트랙(볼륨)·정교트랙(알파 프리미엄) 두 상품으로 매출 다각화. |
| "단가 낮추자는 거야?" | 아니다. 패스트트랙은 **새 상품**을 만드는 것이고, 정교트랙 단가는 그대로 방어한다. 오히려 알파를 보여줄 무대가 생겨 프리미엄이 강화된다. |
| "리스크는?" | 절대위임금지 + 심의 체크리스트는 패스트트랙에도 그대로 적용. 올리브영 발색컷 AI 금지선처럼 경계는 이미 운영 중. |
| "왜 지금인가?" | 넷마블이 이미 요구했고, 일반 업계도 같은 니즈를 보이는 중. 늦을수록 우리 자체 TO·단가 SoT의 구조적 문제가 누적된다. |
| "증원이랑 상충 아닌가?" | 반대. AI 전담팀이 증원의 형태이므로, 패스트트랙 구조가 먼저 있어야 신규 인력이 바로 투입된다. |
| "당장 뭘 하면 되나?" | 이번 주 AI 보이게 태깅 + TO-06 템플릿 초안 착수. 승인 필요한 건 AI 전담팀 신설 + 패스트트랙 건단가 신설 두 가지. |
