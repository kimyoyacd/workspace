## 1. 브랜드 정의

**TEMS Intelligence는 실무 판단을 학습·측정·검증하며 축적하는 Decision Intelligence System이다.**
문제은행도, 사람을 평가·선별하는 어세스먼트 툴도 아니다. 리스크 판단·우선순위·이해관계자 조정·트레이드오프·커뮤니케이션이라는 5가지 판단축 위에 실제 케이스를 쌓아, 전문가 검증과 사용자 수행 데이터로 계속 보정되는 '판단 데이터 네트워크'다.

### 관점 한 문장 (전 매체 상속)
> **"판단은 재능이 아니라, 축적되는 자산이다." (Judgment Compounds.)**

- 제품 코어(데이터 플라이휠)·참여 구조(성장 사다리)·클라 요구(신뢰+성장하는 기업)를 한 문장으로 관통
- 평가·선별 진영(SHL·HireVue·AI역량검사)이 절대 말할 수 없는 문장 = 차별화 그 자체
- 예비안: A "우리는 평가하지 않는다. 판단을 길러준다." (카피용 — 격식 매체엔 공격적) / B "모든 판단이 네트워크가 된다." (심볼 모티브용)

### 브랜드 키워드 5
| 키워드 | 역할 |
|---|---|
| **Compound (축적)** | 코어 — 데이터도, 사용자 커리어도, 회사도 쌓여서 커진다 |
| **Warm Trust (따뜻한 신뢰)** | 톤 — 기관 격식 × 휴먼 온기, 시장 비주얼 공백 그 자체 |
| **Ladder (성장 사다리)** | 구조 — 입문→성장→검증 전문가 참여 설계 |
| **Verified (검증된)** | 신뢰 근거 — 따뜻함이 물러지지 않게 잡는 균형추 |
| **Structure (케이스×판단축)** | 형태 언어 — 심볼·그리드·명함 레이아웃의 조형 원천 |

## 2. 타겟 정의 (이중 구조 — BI가 둘 다 감당해야 함)

### ① 결재권자 = 1차 영업 타겟: 대학 취업지원처·진로교육 담당 교직원
- 도입 명분이 필요한 사람들: 공신력, 데이터 근거, 기관 격식, '검증 안 된 업체' 리스크 회피
- 이들의 시각 언어: 파랑 계열(고등교육 관례색), 시리프=학술 권위, '거버넌스·투명성·인간 감독' 어휘
- 비비드 캐주얼(잡다·캐치류)은 "학생 서비스"로, 차가운 어세스먼트(HireVue류)는 "학생을 재단하는 AI"로 읽음 → 둘 다 회피
- **명함·제안서가 만나는 타겟은 이쪽** — BI의 격식 축이 여기를 담당

### ② 사용자: 학생 (확장 시 기업 실무자)
- 경험 없음이 불안한 취준생 — "여기서는 틀려도 자산이 된다"는 심리적 안전 필요
- 평가받는 두려움이 아니라 성장의 온기 — BI의 웜 액센트가 여기를 담당

### 브랜드 브리프 (템플릿 5필드)
1. **목표**: '평가받는 두려움'의 시장에서 '판단이 자산으로 쌓이는 안심'의 브랜드가 된다. 결재권자에겐 검증된 시스템의 신뢰를, 학생에겐 성장의 온기를 하나의 BI로 동시 전달.
2. **타겟층**: 위 이중 구조.
3. **브랜드 기능 + 모티브**: 기능 = 실무 판단의 학습·측정·검증·축적. 모티브 = **케이스×판단축 격자 위에 판단 노드가 연결·증식하는 구조** (HR·교육 시장 미선점 심볼 영역).
4. **이름 의미**: 공식 어원 클라 확인 필요(★질문 1건). 해석 레이어 제안 — 'Task·Experience·Measurement·System' 백로님 부여 가능.
5. **스토리 씨앗**: "일 잘하는 사람의 판단은 어디에도 기록되지 않고 사라져 왔다. TEMS는 그 판단을 처음으로 데이터로 남기고, 검증하고, 다음 사람의 성장 자산으로 되돌려준다."

## 3. 포지셔닝 (리서치 근거 요약)

- 시장 축: '기관용 평가'(SHL·HireVue·Eightfold)와 '개인용 취준 캐주얼'(잡다·캐치·링커리어) 양쪽 과밀 → **공백 = "기관(대학)에 공급되는 성장·판단 시스템"** (유일한 유사 선례 Forage는 국내 부재 + 판단 데이터 축적 아님)
- 비주얼 톤 축: '차가운 신뢰'(네이비 엔터프라이즈) vs '따뜻한 캐주얼'(비비드 B2C) 양극화 → **공백 = "따뜻한 신뢰"** (HireVue·뤼이드 리브랜딩의 이동 방향이 수요 방증)
- 심볼 기회: '판단 데이터의 구조'를 심볼화한 브랜드 전무 (SHL '연결의 점'이 유일한 유사 선례)
- 함정: 시뮬레이션·게임 톤 전면화 → 대학 조달 관문에서 '검증 안 된 게임 업체' 격하 위험. 시뮬레이션은 제품 언어로만.

**포지셔닝 선언: "대학이 믿고 도입하는, 판단을 길러주는 시스템" — 기관 격식 × 휴먼 온기의 유일 점유자.**

## 4. 워드마크 표기 — 확정: **TEMS Intelligence** (Title Case)

| 안 | 판정 | 근거 |
|---|---|---|
| **TEMS Intelligence** | **확정** | ①실리콘밸리 AI 현행 관례 = Title Case 지배(Anthropic·Scale AI·Perplexity·Harvey·Sierra) — 특히 Scale AI가 '대문자 약어+Title Case 보조어' 직접 선례 ②TEMS(대문자)=검증·기관 신뢰, Intelligence(Title Case)=Decision Intelligence 카테고리를 이름으로 소유 ③따뜻함은 케이싱이 아니라 자형·컬러가 담당하도록 역할 분담 |
| TEMS intelligence | 예비 | 온기는 얻지만 'intelligence'가 일반 설명어로 격하 — 카테고리 선언력 약화 |
| tems intelligence | 탈락 | 2015~20년 컨슈머 SaaS 문법. 약어 어원 소실, 'items' 오독 위험, 대학 결재 문서 격식 미달 |
| TEMS INTELLIGENCE | 탈락 | 레거시 엔터프라이즈·평가 진영(SHL·구 HireVue) 문법 — '차가운 신뢰' 과밀 지대 |

- 락업 위계는 케이싱 아닌 **웨이트**로: TEMS 강조 / Intelligence 라이트 웨이트 보조
- 명함 등 작은 지면에서 **TEMS 단독 축약** 성립

## 5. 디자인 방향 지시 (시안 전 단계 — 방향만)

### 형태
- '축적되는 구조'를 심볼 문법으로: 케이스×판단축 격자, 또는 노드가 연결되며 증식하는 모듈
- 정적 로고라도 **증식 규칙 내장** — 명함→파비콘→이후 매체에서 같은 규칙으로 확장
- 완결된 폐곡선(평가·선별의 조형) 회피. 뉴럴넷 노드 클리셰·보라 그라디언트 원천 배제 (2026 "끝난 트렌드": 그라디언트·글래스모피즘·벤토 그리드)
- '판단의 중심에 사람' 네거티브 스페이스 은유 검토 (Ideogram 2026.6 선례)

### 컬러
- **신뢰 베이스(딥 블루/네이비 계열) + 웜 액센트 1색(앰버·코랄 계열)** 이원 구조
- 네이비 단독 금지(과밀 지대), 웜 액센트가 베이스를 넘으면 B2C 캐주얼로 미끄러짐 — 비율 가드
- 직접 경쟁 HR테크(ICIMS 2026.3)가 보라 선점 → 네이비·크림이면 카테고리 내 자동 차별화

### 타이포
- 지오메트릭 뼈대 + 휴머니스트 디테일의 산세리프. TEMS는 자간 여유 있게(기관 신뢰), Intelligence는 라이트 웨이트(성장의 가벼움)
- 시리프 병용 옵션: 제안서·문서 헤드라인에서 시리프(학술 권위) + 본문 산세리프 — 교육 브랜딩 표준 문법
- 배제: 날카로운 테크 모노스페이스, 라운드 과다 캐주얼

### 명함 (제작 사양 방향)
- 비코팅 크림 450gsm 듀플렉스 + 네이비 1도 + 로고 레터프레스(또는 저채도 메탈릭 1포인트)
- 뒷면 QR → 데모/소개 페이지 연결
- 웜 액센트는 '축적 노드' 포인트로만
- 제안 화법: "왜 이 만듦새인가" — 산학 대면 신뢰 스토리로 판다
