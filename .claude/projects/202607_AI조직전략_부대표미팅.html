03 / 조직
주제 ② · 방향

MAX실을 'AIOS'로 재설계 — Litmus 4축 매핑

넷마블에서 확인한 건, 대형 조직도 결국 개인 툴 파편화를 끝내고 조직 OS로 간다는 것. 늦었지만 9명이라 오히려 더 빠르게 재설계할 수 있다.
Litmus AIOS 축우리 현황갭액션
① 넥서스
롤별 권한·데이터 격리		리스크 티어 + 절대위임금지 有롤별 조회/생성/승인 권한 없음액션 4
② 워크벤치+메모리
프로젝트 맥락 기억·참조		library + projects 아카이브 有시멘틱 메모리 아님(텍스트만)액션 2
③ 파운드리
실행 관제 + 피드백 루프		트리거 맵·게이트 정의됨레이팅 루프·상시 러너 부재액션 1
④ 컨설턴트화
직무 통합·AI 레버리지		실장 개인이 이미 증명팀 전파·표준화 안 됨액션 3 + 스터디
04 / 조직
주제 ② · 실행 로드맵

4개 액션 · 우선순위 순

01

Design Gatekeeper 공용 에이전트 격상 이번 주 목요일 착수

공용 프로필 등록 + BBD(밀러·피츠 법칙) 모듈 중앙 주입 + Good/Bad 피드백 루프 → 매주 목요일 스터디에서 프롬프트 공동 튜닝. 품질을 사람이 아니라 시스템이 보증.
02

프로젝트 '디자인 메모리' 자산화

미팅·협의 중 디자인 정책 변경을 1~2문장 시멘틱 요약으로 적재 → 에이전트가 프롬프트·Figma 초안 시 MCP로 호출. 클라 최신 요구 즉각 반영.
03

제안·세일즈 단계부터 BBD 시각화 수주율 직결

인바운드 → 기업 리서치 팩 자동 spawn → 질의서 자동 생성 → BBD 와이어프레임/초안을 AI로 즉시 시각화해 제안 데모. 세일즈 단계 USP 확보.
04

역할별 가드레일 + 시스템 온보딩 증원 직결

신규/주니어=조회·생성 / 시니어·리드=최종 승인·환경 제어. 신규 합류 시 SOP 교육 대신 "워크벤치에 접속해 에이전트에게 물어보며 작업" — 초고속 온보딩.
0~30일
액션 1 가동 + 목요일 튜닝 정례화 + 상시 러너 1건 연결
30~60일
액션 2를 진행 프로젝트 1건에 적용·효과 검증 + 액션 4 롤 권한 초안
60~90일
액션 3을 신규 인바운드 1건 실전 + 증원 온보딩에 시스템 적용
