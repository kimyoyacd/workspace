사용법 — 이렇게 말하면 됩니다

Claude Code(또는 이 채팅)에서 트리거를 말하면 알맞은 에이전트가 작동합니다.

① 제안 풀세트 → "○○ 제안 방향 잡아줘"

시장조사 → 관점 → 발산 → 점수화가 순서대로. 실장님은 셀렉만.

② 관점부터 → "이 제품 관점 한 문장으로"

concept-director가 핵심 제품 기준 관점을 고정. 이후 모든 디자인이 상속.

③ 견적 → "코웨이 UI Design 고급1 3.5MM 100%, 견적 뽑아줘"

quote-accountant가 원가표 + 마진 시나리오 + 우리실 매출(UI Design분) 분리. 최종가·마진은 실장님 선택.

④ 진행 현황 → "이번 주 현황" / "가동률"

project-manager가 노션·구글시트를 읽어 진행보드 + 매출 달성율 + 데이터오류 + 누가 몇% 투입을 정리.

⑤ 시안 검수 → "이 시안 봐줘" (첨부)

design-critique가 6영역으로 🟢/🟡/🔴. ⑥ "트렌드 봐줘" → design-trend-radar.

핵심: 말하기 → 에이전트가 일함 → 실장님이 최종 결정. 이 대시보드는 "무엇이 실제로 있는지" 보여주는 지도입니다.
