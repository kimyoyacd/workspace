# 편집 워크플로우 체크리스트

> Higgsfield에서 생성된 컷들을 실제 30초 Shorts로 조립하는 단계.

---

## 1단계: 소스 정리

- [ ] `04_Video_Output/[에피소드명]/` 에서 승인된 컷 6개 영상 확인
- [ ] 각 컷 4-6초, 1080×1920, 컷 순서대로 파일명 정리 (cut1.mp4 ~ cut6.mp4)
- [ ] 편집 툴(CapCut / Premiere / Final Cut / DaVinci Resolve)에 프로젝트 생성

## 2단계: 컷 편집

- [ ] 컷 순서대로 타임라인에 배치
- [ ] 컷 간 전환: 하드컷 기본 (빈티지 그림책 느낌 유지, 과한 트랜지션 X)
- [ ] 전체 러닝타임 30초 내외 확인
- [ ] 각 컷 길이 미세 조정 (템포감 있게, 특히 컷2 무너짐은 살짝 빠르게)

## 3단계: 사운드 추가 (`07_Voice_Subtitle` 참고)

- [ ] 배경음악(BGM) 삽입, 감정 흐름에 맞춰 볼륨 자동화
- [ ] 컷별 효과음(SFX) 싱크 맞춰 삽입
- [ ] (선택) 내레이션 삽입
- [ ] 오디오 레벨 확인 (-14 LUFS 권장, 유튜브 표준)

## 4단계: 자막 (선택 시)

- [ ] 자막 스타일 가이드 적용 (폰트/위치/색상)
- [ ] 오브제와 겹치지 않는 위치 확인
- [ ] 자막 없이도 스토리 이해 가능한지 재확인

## 5단계: 썸네일 (`06_Thumbnail` 참고)

- [ ] 썸네일 이미지 준비 (별도 Higgsfield 생성 또는 컷1/6 중 임팩트 있는 프레임 활용)
- [ ] 텍스트 오버레이 시 브랜드 폰트/컬러 가이드 준수

## 6단계: 최종 QC

- [ ] `MASTER_QC_CHECKLIST.md` 전체 항목 재확인
- [ ] 모바일 화면에서 실제 크기로 재생 테스트 (텍스트/디테일 가독성)
- [ ] 무음 상태 재생 테스트
- [ ] 색감/밝기 일관성 (컷 간 튀는 부분 없는지)

## 7단계: 내보내기(Export)

**권장 export 스펙**:
- 해상도: 1080×1920 (9:16)
- 프레임레이트: 30fps (또는 소스에 맞춰 24/30)
- 코덱: H.264
- 비트레이트: 8,000-12,000 kbps (고화질 유지)
- 오디오: AAC, 48kHz, -14 LUFS
- 포맷: MP4
- 파일명: `[에피소드ID]_[에피소드명]_final.mp4`

## 8단계: 저장 및 업로드 준비

- [ ] 최종 파일을 `04_Video_Output/[에피소드명]/final/`에 저장
- [ ] `08_YouTube_Upload_Log`로 이동해 업로드 메타데이터 작성

## 편집 툴 추천

| 툴 | 장점 | 비고 |
|-----|------|------|
| CapCut | 무료, 모바일/PC 모두, 자막 자동화 기능 좋음 | Shorts 편집에 최적 |
| DaVinci Resolve | 무료, 색보정 강력 | 러닝커브 있음 |
| Premiere Pro | 프로급, 팀 협업 좋음 | 유료 구독 |
| Final Cut Pro | Mac 전용, 빠른 렌더링 | 유료 (1회 구매) |
