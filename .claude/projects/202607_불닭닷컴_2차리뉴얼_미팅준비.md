## 1. 파악한 것 vs 확인 필요한 것

### ✅ 파악된 브랜드 컨텍스트 (선지식으로 미팅에서 주도권 확보)
- 삼양식품은 '불닭'을 일반명사에서 **글로벌 고유 브랜드 'Buldak'으로 격상** 중. 전 제품군에 Buldak 로고 병기, 소스 패키지도 Buldak 전면화.
- 글로벌 슬로건 **"Spicy or Nothing"** — 도전적·강렬한 태도. 브랜드 사이트를 "경험·소통 허브"로 규정.
- Buldak 관련 글로벌 콘텐츠 누적 **200억 뷰+**, ANUGA 2025 'Buldak Spicy Club' 부스 등 **브랜드 아이콘화** 국면.
- → **함의**: 2차의 방향은 "예쁘게 다듬기"가 아니라 **'Spicy or Nothing'을 커머스 UX로 번역**하는 것. 강렬함·오리지널리티·글로벌 톤을 한국 자사몰에 이식.

### ❓ 반드시 확인 (미팅에서 답 받아올 것)
- 1차 리뉴얼 **주체**(삼양 내부 / 타 에이전시 / 우리?)와 **산출물 인수인계** 가능 여부
- **BI 가이드라인 존재 여부**와 우리에게 주는 **확장 권한** (준수만? 재해석 가능?)
- 이 사이트가 **자사몰 커머스**인지(결제·회원·재고 연동) → 플랫폼(Cafe24/자체/기타)
- 1억의 **범위**(디자인만 / 퍼블·개발 포함 / 유지보수 포함)와 **VAT**

## 3. ★ 질의 리스트 — 놓치면 안 될 질문 (미팅 핵심)

> 우선순위: 🔴 반드시 / 🟡 중요 / 🟢 있으면 좋음. 미팅에서 이 순서로 확인.

### A. 목표·성공기준
- 🔴 이번 2차 리뉴얼로 **가장 바꾸고 싶은 한 가지**는? (브랜드 인상 / 매출·전환 / 콘텐츠 허브화 중 우선순위)
- 🔴 성공을 **무엇으로 측정**하나? (체류시간·전환율·재방문·브랜드 인지 등 KPI)
- 🟡 벤치마크로 생각하는 사이트/브랜드가 있나? (레퍼런스 눈높이)
- 🟡 "브랜드 강화"의 구체 이미지 — 글로벌 'Spicy or Nothing' 톤을 **한국 사이트에 어디까지** 가져오나?

### B. 1차 ↔ 2차 경계 (가장 중요, 리스크 직결)
- 🔴 1차 리뉴얼은 **누가** 진행했고, 우리는 그 **디자인 시스템/퍼블/CMS를 인계**받나?
- 🔴 1차의 **어디까지가 완성**이고, 2차가 **덮어쓰는 범위 vs 유지 범위**는?
- 🔴 1차 산출물(피그마·디자인 토큰·컴포넌트) **원본 제공** 가능한가? (없으면 재제작 공수 발생)
- 🟡 1차와 2차 사이 **책임 경계** — 1차 잔여 버그/미완성은 누구 몫인가?

### C. 브랜드 BI·컨셉
- 🔴 **BI 가이드라인 문서**가 있나? (로고·컬러·타입·모션·톤) 우리에게 **공유 가능**한가?
- 🔴 우리의 권한: BI **준수(가이드 내)** 인가, **재해석·확장(웹 전용 시스템 신규 구축)** 인가?
- 🟡 글로벌 사이트(buldak.com)와의 **일관성 강제** 수준 — 완전 통일? KR 로컬 변주 허용?
- 🟡 지켜야 할 **헤리티지 자산**(불닭 캐릭터 '호치'? 컬러? 불꽃 모티프)과 건드리면 안 되는 것?

### D. 범위·산출물 (스코프 확정)
- 🔴 6개 범위 각각 **PC/모바일 둘 다**인가? 반응형 1벌인가 별도인가?
- 🔴 페이지별 **총 화면(템플릿) 수** 대략 — PLP/PDP는 카테고리·상품유형별 변형 몇 종?
- 🔴 산출물 정의: **디자인만(피그마)** / **퍼블리싱(HTML/CSS)** / **개발 연동**까지 어디?
- 🟡 **다국어/영문** 대응 필요? (글로벌 브랜드라 KR 외 확장 가능성)
- 🟡 신규 **촬영·2D 소스·모션/영상** 필요? 삼양 제공 vs 우리 제작?
- 🟢 디자인 **가이드/핸드오프 문서** 산출물 포함?

### E. 커머스·기술 (제약 파악)
- 🔴 **자사몰 커머스**인가? (장바구니·결제·회원·주문조회) → **플랫폼/솔루션** 무엇?
- 🔴 개발사는 **누구**이고 우리는 **디자인만 넘기면 되나**, 개발까지 우리 몫인가?
- 🟡 CMS로 운영되는 영역(News/Blog/Recipe)의 **템플릿 제약** — 자유 디자인 vs 정형 모듈?
- 🟡 성능·접근성·SEO **준수 기준**이 정해져 있나?

### F. 콘텐츠·운영
- 🟡 Recipe/News/Blog의 **콘텐츠 양·업데이트 주기·작성 주체** — 우리는 틀만? 초기 콘텐츠도?
- 🟡 상품/카테고리 **수**와 향후 증가 계획 (PLP·PDP 확장성 설계 근거)
- 🟢 My Page 기능 범위 (등급·쿠폰·리뷰·위시 등 어디까지)

### G. 일정·의사결정·계약
- 🔴 **오픈 목표일**과 역산 마일스톤? (7.30 1차 직후 바로? 아니면 여유?)
- 🔴 **의사결정권자**는 누구이고, **컨펌 단계**는 몇 단인가? (실무→팀장→임원)
- 🔴 예산 **1억: VAT 별도/포함**, **선금·중도금·잔금** 구조, **수정 라운드 횟수**?
- 🟡 계약 형태(용역/도급), **저작권·2차사용** 범위, **NDA** 필요?
- 🟢 경쟁 PT인가 **수의(단독)** 인가?

## 4. 예산 1억 해부 — 우리실(MX) 매출 관점

> 규칙(unit-rates): **총액 중 'UI Design' 직군 직접인건비만 우리실 매출.** PM·UX·FE·BE·QA는 타 실/외주 몫.
> → 협상 목표: **1억 안에서 UI Design 비중을 최대화**하고, 개발이 붙으면 그건 별도(총액↑, 우리실분은 UI만).

### 시나리오 감(내부 참고 · 미팅 공개 금지)
- **Case α — 디자인 중심(퍼블 제외)**: 1억 대부분이 UX+UI. UI Design 비중 60~70% 확보 시 우리실 매출 6천~7천만 대. **가장 유리.**
- **Case β — 디자인+퍼블**: 1억에 FE(퍼블) 포함 → UI 비중 희석. 우리실분 4천~5천만 대. 퍼블을 외주로 빼면 관리마진만.
- **Case γ — 디자인+개발 통짜**: 1억에 BE/QA까지 → UI 비중 최소. **총액은 커도 우리실 실적은 얇음.** → 이 경우 **총액 증액 협상** 또는 개발 분리 제안.

### 규모 적정성 감각 (레드플래그 체크)
- 범위가 최소 **8개 템플릿(Home/PLP/PDP/Recipe/News/Blog/About/My) × PC·MO × 변형**. 여기에 BI 컨셉 도출까지.
- 코웨이 BEREX GUI/UX가 총 8천만(UI분 약 7천만)이었던 것과 비교 → **1억에 개발까지 넣으면 디자인 공수 부족** 가능성. **"1억=디자인 예산"인지 반드시 확인.**
- 만약 개발 포함 1억이면 → 페이지 수 축소 or 예산 증액 or 단계 분리(Phase 2a/2b) 역제안.

## 6. 리스크 & 레드플래그

- 🚩 **1차와의 연속성 붕괴**: 1차 시스템을 인계 못 받으면 2차에서 컴포넌트 재제작 → 숨은 공수. (질의 B)
- 🚩 **BI 권한 모호**: "브랜드 강화"인데 가이드 준수만 요구하면 디자인 자유도↓. 반대로 가이드 없으면 시스템을 우리가 세팅 → 공수↑(이건 매출 기회로 전환 가능).
- 🚩 **1억에 개발 포함**: 우리실 UI 매출 얇아짐 → 사전 시나리오(4장)로 대응, 개발 분리 or 증액 카드.
- 🚩 **커머스 결제/회원 제약**: 자사몰 솔루션의 UI 제약이 크면 디자인 자유도가 플랫폼에 갇힘 → PDP 구매모듈 디자인 한계 사전 확인.
- 🚩 **콘텐츠 미확보**: Recipe/News 실콘텐츠 없이 껍데기만 만들면 오픈 지연·재작업. 초기 콘텐츠 책임 경계 명확화.
- 🚩 **의사결정 다단계**: 삼양 규모상 임원 컨펌 단계 존재 가능 → 수정 라운드·일정 리스크. 컨펌 단계·횟수 계약 명시.
- 🚩 **7.30 직후 압박**: 1차 오픈 여파(버그·피드백)와 2차 착수가 겹치면 리소스 충돌. 착수 시점 조율.

## 7. 진행 계획 (미팅 → 제안 → 착수)

1. **[미팅 당일]** 3장 질의 리스트로 스코프·예산·BI 권한 확정. 견적은 미확약, "범위 확정 후 제안" 선언.
2. **[D+1~2]** 1차 오픈 버전(7.30) 확보 → 간이 휴리스틱 진단 완성 (design-critique).
3. **[D+2~3]** RFP-analyst로 요청서 재분해(명시/숨은/미정의/위험) → 미정의·위험 항목 클라 재확인.
4. **[D+3~5]** quote-accountant로 직군별 견적 시트 확정 (UI 매출 분리 표기) → 실장 최종가 결정.
5. **[D+5]** 제안서 초안 (proposal-playbook SOP: 컨셉 관점 → 진단 → 개선 방향 → 견적/일정). concept-director로 관점 한 문장 선고정.
6. **[제안 후]** review-panel 5관점 검토 → tone-guardian → delivery-gate 후 발송.

## 9. 현 사이트 재구성 + 잠정 진단 (리서치)

> ⚠️ **접근 제약**: 이 세션 환경에서 buldak.com은 **네트워크 정책으로 차단**(egress deny), 삼양 미국/글로벌 사이트는 **원본 서버 봇 차단**으로 실제 화면을 직접 감사(firsthand)하지 못함. 아래는 **공개 검색 정보로 재구성한 구조 + 잠정 진단**이며, **확정 진단은 7.30 KR 오픈본 캡처를 받아 design-critique로 재검증** 필요.

### 9-1. 재구성한 현 사이트 구조 (글로벌 buldak.com 기준)
- 글로벌 사이트는 지역 분기: `/kr` `/us` `/eu` 등. 슬로건 **"Spicy or Nothing"**, 히어로 카피 **"Unmatched addictive Heat"**.

- 특징 요소: **Buldak Scoville Club**(맵기 등급으로 취향 탐색·커뮤니티), 제품군이 매우 넓음(Original·Carbonara·Rosé·2X Spicy·Cheese·Quattro·Cream Carbonara·Yakisoba·Taco·Swicy·Birria·Big Bowl·Cup·소스 등).

### 9-2. 잠정 진단 — 미팅에서 "가설"로 던질 개선 포인트 (검증 전제)
> 화법: "직접 뜯어보기 전 가설입니다만—" 으로 열어 신뢰는 얻되 단정은 피한다.

- **[BI 번역]** 'Spicy or Nothing'의 강렬함이 **한국 커머스 UX(구매 동선)까지 관통하는가**, 아니면 히어로 비주얼에만 머무는가. → 2차 핵심은 "브랜드 톤 × 전환 UX"의 결합.
- **[제품 과다]** 제품군이 수십 종. **PLP 필터·정렬·카테고리 체계**(맵기/타입/신상/베스트)가 탐색 부하를 줄이는지가 관건. Scoville 등급을 **필터 축**으로 UX화하면 브랜드 자산이 곧 기능이 됨.
- **[PDP 전환]** 강한 비주얼 대비 **구매 모듈(옵션·수량·장바구니·정기구독·리뷰·별점)**의 위계·가시성 점검. 브랜드 페이지처럼 예쁘기만 하고 사게 만드는 힘이 약할 리스크.
- **[Recipe 허브]** 블로그 레시피가 흩어져 있으면 **아카이브 구조(재료·맵기·조리시간 필터, 스텝·영상 규격)**로 정돈 시 체류·SEO·재방문↑.
- **[일관성]** 글로벌↔KR, 페이지 간 **컴포넌트·타이포·컬러 시스템 일관성**. 1차가 시스템을 어디까지 세웠는지에 따라 2차 공수가 갈림(→ 질의 B·C).
- **[모바일 우선]** F&B·Z세대 타깃 → 모바일 비중 절대적. **모바일 퍼스트 정보위계·썸 리치 CTA** 기준으로 재설계 여부 확인.

### 불닭닷컴 적용 포인트 (요약)
1. **BI 번역 = 브랜드 월化** — 자사 US/EU는 서사는 강한데 구매 동선이 약함. 드롭·굿즈·멤버십을 구매 흐름에 얹어 "라면몰→매운맛 컬처 커머스".
2. **다크 하이컨트라스트로 프리미엄 격상** — TRUFF 수준 제품샷·다크 히어로로 저가 이미지 탈피(목표 ①·② 동시 충족).
3. **Recipe + News/Blog → 단일 에디토리얼 허브** — 각 레시피에서 제품 바로 담기(콘텐츠→커머스 루프), 매운맛 단계·글로벌 퓨전으로 농심 차별화.
4. **PLP: 맛 라인업 × 세트/기프트 × 정기구독 + 필터에 "매운맛 단계"** — 불닭 고유 축을 필터로 = 곧 아이덴티티.
5. **PDP: 번들·수량·정기구매 통합 + 하단 연관 레시피** — 결정 피로 최소화가 전환 핵심.
6. **자사몰 락인 = 스코빌 클럽 멤버십** — Tastemaker 모델로 리텐션 설계.

무드 A(Liquid Death→TRUFF)로 **톤** 각인 → 무드 B(Omsom/Graza)로 **콘텐츠+커머스 공존** 설득 → 무드 C(immi/Momofuku PDP)로 **전환 UX** 마무리 → 마지막에 **현행 Buldak US와 대비**시키면 개선 여지가 자명해짐.
