# 개인 데이터 허브 (Personal Data Hub)

> 흩어진 내 데이터(노션 회의록·Gmail·드라이브·캘린더)를 AI가 쓸 수 있는 형태로 모으는 곳.
> 목표: 몇 년치 회의록 = 내 일을 아는 비서, 쌓아둔 메일·문서 = 나만의 검색기.

## 구조
```
data/
├── README.md            # 이 파일 — 운영 규칙
├── data-inventory.md    # 소스 맵: 어디에 뭐가 있고 어떻게 읽는지
└── clients/             # 클라이언트별 히스토리 (회의록·메일·시트에서 추출)
    ├── _index.md        # 전체 클라이언트 인덱스 (노션 미팅 DB 기준)
    ├── coway.md
    └── netmarble.md
```

## 운영 규칙
1. **원본은 옮기지 않는다.** 노션·드라이브 원본은 그대로 두고, 여기엔 "요약 + 원본 링크"만 적재한다.
2. **프로젝트 단위로 채운다.** 전체를 한 번에 정리하지 않는다. 미팅·제안이 잡히면 그 클라이언트 파일부터 갱신.
3. **패턴을 기록한다.** 단순 사실보다 "이 클라는 항상 X한다" 같은 반복 패턴이 가장 가치 있다.
4. **에이전트 연동:** `account-radar`(미팅 브리핑), `quote-accountant`(과거 견적 근거), `project-manager`가 이 폴더를 우선 참조한다.
5. **갱신 시점:** 미팅 직후 회의록이 노션에 올라오면 → 해당 클라이언트 파일에 요약 1블록 추가.

## 클라이언트 파일 표준 포맷
```
# 클라이언트명
## 기본 정보 (담당자, 채널, 계약 형태)
## 진행 중 / 완료 프로젝트
## 미팅 히스토리 (날짜 · 요약 · 원본 링크)
## 패턴 & 주의사항 ⭐ (반복 행동, 피드백 성향, 리스크)
## 다음 액션
```
