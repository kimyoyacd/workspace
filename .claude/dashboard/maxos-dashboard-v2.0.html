사용설명서

Claude Code + MAX실 에이전트 시스템을 어떻게 쓰는지. 중학생도 이해할 수 있게.

01
에이전트 = 특정 일을 기억하는 전문가
Claude Code에서 새 대화를 열면 → 그게 "에이전트"가 됩니다. 각 에이전트는 특정 역할만 기억합니다.

비유: 회사에서 "법무팀한테 보내" 하면 법무팀이 알아서 처리하는 것처럼, "계약서 검토해줘" 하면 legal-compliance 에이전트가 자동으로 호출됩니다.
02
파일 = 에이전트의 설명서
.claude/agents/creative-director.md 파일이 있으면 → 그게 "크리에이티브 디렉터" 에이전트가 됩니다.

GitHub에 올리면 → 어디서나 같은 에이전트가 작동합니다. 폴더가 기억입니다. 파일이 없으면 에이전트도 없습니다.
03
이 대시보드 = 지도 (실제 작업은 Claude Code)
이 HTML 파일은 "누가 있는지 보여주는 지도"입니다.

실제 작업: Claude Code를 열고 → 키워드로 말하면 자동으로 적합한 에이전트가 일을 합니다. 이 대시보드를 열면서 대화하면 됩니다.

💬 이렇게 말하면 됩니다 — 12가지 시나리오

① 시안 검수 → "코웨이 시안 검수해줘"

Claude Code에 시안 파일을 첨부하고 → creative-director가 톤을 보고 → design-critique가 6영역 체크 → 결과를 사용자에게 보고합니다.

② 견적 요청 → "아모레 설화수 캠페인 견적 뽑자"

quote-accountant가 인건비·외주비·제작비·관리비 4축 원가표를 만들고 → 10/20/30/40% 마진율 시나리오 4가지를 보여줍니다. 마진율 선택은 사용자 직접.

③ 이번 주 가동률 → "이번 주 가동률 알려줘"

max-pm이 구글 시트를 읽어서 9명 각각의 투입률, 야근 여부, 기일 임박 프로젝트를 정리해서 보여줍니다.

④ 제안 발산 → "넷마블 신규 캠페인 컨셉 발산해줘"

brainstormer가 6관점 × 10개 후보를 생성 → 사용자가 1~2개 선택 → 제안서 풀패키지가 자동으로 돌아갑니다.

⑤ 클라 상황 파악 → "코웨이 요즘 상황 어때?"

account-radar가 코웨이 히스토리, 최근 커뮤니케이션, 리스크 포인트를 요약해서 미팅 직전에 보여줍니다.

⑥ 트렌드 확인 → "요즘 UI/UX 트렌드 뭐야?"

design-trend-radar가 Awwwards·Brand New 등에서 3개월 이내 글로벌 트렌드만 추려서 보고합니다. 분기 첫주에는 자동으로 업데이트됩니다.

⑦ 제안 풀패키지 → "설화수 제안 풀패키지 돌려"

12단계 자동 오케스트레이션: RFP분석 → 시장조사 → 트렌드 → 레퍼런스 → 발산 → 사용자 셀렉 → 견적 → 5관점패널 → 팩트체크 → 비평 → 시안검수 → 딜리버리 → 사용자 OK

⑧ 키비주얼 생성 → "설화수 키비주얼 뽑아줘"

visual-generator가 Higgsfield MCP로 AI 이미지 생성. 스타일프레임, 목업 이미지를 바로 만들어 줍니다.

⑨ 회신 초안 검토 → "넷마블에 CR 5차 답변 초안 잡아줘"

초안 작성 후 → tone-guardian이 톤 체크 → 사용자가 최종 OK하면 발송. 자동 발송은 절대 안 됩니다.

⑩ 신규 입사자 온보딩 → "이번에 새로 온 VM 정○○ 온보딩해줘"

creative-director가 브랜드 톤·매너, 클라별 주의사항, 디자인 시스템, QC 체크리스트가 담긴 1장 HTML 온보딩 팩을 만들어 줍니다.

⑪ 채용 현황 → "VM 채용 지금 어디까지야?"

hiring-radar가 9→20 파이프라인 현황, JD 상태, 소싱 후보, 면접 일정을 요약해서 보여줍니다.

⑫ 계약서 검토 → "하이브 NDA 계약서 검토해줘"

legal-compliance가 계약서 첨부를 받아 쟁점·리스크를 플래그. 단, 법무 자문을 대체하지 않습니다.

핵심 정리: Claude Code 열기 → 말하기 → 에이전트가 일함 → 사용자가 최종 결정.
이 대시보드는 "어떤 에이전트가 있는지" 보여주는 지도입니다. 실제 대화는 Claude Code에서 하세요.
