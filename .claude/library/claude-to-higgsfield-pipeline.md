# Claude → Higgsfield 캐릭터 시리즈 파이프라인

> 재사용 자산. **하나의 캐릭터를 여러 컷·여러 상황으로 "같은 얼굴"을 유지하며 시리즈로 뽑는** 표준 흐름.
> Claude(기획·프롬프트·검수) ↔ Higgsfield(이미지·영상 생성) 역할 분리.
> 개별 프로젝트는 이 파일을 참조하고, 캐릭터별 세부는 각 프로젝트 노트에 기록한다.

---

## 0. 핵심 원리 — "캐릭터 바이블"이 시리즈의 심장

시리즈가 무너지는 유일한 이유는 **컷마다 얼굴·비율·톤이 미세하게 달라지는 것**이다.
그래서 생성 전에 **불변(Locked) 요소 / 가변(Free) 요소**를 못 박고 시작한다.

| 구분 | 예시 | 규칙 |
|---|---|---|
| 🔒 **불변 (Locked)** | 얼굴 구조·헤어 형태/색·눈매·체형·시그니처 컬러·아트 스타일 | 전 컷 동일. 프롬프트에 매번 **똑같은 문구**로 박고, 캐릭터 레퍼런스로 고정. |
| 🎛 **가변 (Free)** | 표정·포즈·의상·배경·조명·앵글·소품 | 컷마다 자유. 시리즈의 '변주'는 여기서만 일어난다. |

> **원칙**: 불변 요소는 문장으로 외우지 말고 **고정 프롬프트 블록 + 캐릭터 등록(레퍼런스)** 이중 잠금. 말로만 하면 반드시 흔들린다.

> **코스튬 시리즈 패턴 (몽치치 모델)**: 캐릭터가 매 에피소드 옷을 갈아입는 IP라면, **불변 = 얼굴·헤어·눈·체형·아트 스타일(=인형 정체성)만**으로 좁히고 **의상·모자·소품은 가변(테마별 변신)** 으로 둔다. 레퍼런스 엘리먼트가 얼굴을 잡아주므로, 프롬프트에서 옷만 바꿔도 같은 인형이 유지된다. (브랜드 연상 코스튬은 로고 직접 사용 금지 → 색·모티프만.)

## 1. 역할 분리

```
Claude 쪽 (기획·언어·검수)                    Higgsfield 쪽 (생성)
─────────────────────────────               ─────────────────────────
관점 고정 (concept-director)                 generate_image (마스터/시리즈)
캐릭터 바이블 (불변/가변)          ──프롬프트──▶ show_characters / reference_elements (일관성 등록)
프롬프트 생성 (persona Step2)                 generate_video (모션)
검수 (design-critique)             ◀──결과──   upscale_image / remove_background (후처리)
발송 린트 (delivery-gate)
```

---

## 2. 단계별 흐름 (Stage 0 → 7)

### Stage 0 — 관점 & 캐릭터 바이블 고정 · `concept-director`
- 캐릭터의 **관점 한 문장**(이 캐릭터가 뭘 대표하는가)을 먼저 못 박는다.
- 불변 5요소(얼굴·헤어·눈·체형·시그니처 컬러) + 아트 스타일 1줄 확정.
- 산출: 프로젝트 노트의 "캐릭터 바이블" 표. **이게 없으면 생성 시작 금지.**

### Stage 1 — 마스터 프롬프트 작성 · Claude (`persona-directives.md` Step2)
- 영문 **명사 위주 키워드 + 쉼표 구분**. 불변 요소를 `[LOCKED]` 블록으로 분리해 매 컷 그대로 복사.
- 형식:
  ```
  [LOCKED — 매 컷 동일] : {얼굴·헤어·눈·체형·컬러·아트스타일 키워드}
  [FREE — 이 컷만]      : {표정·포즈·의상·배경·조명·앵글}
  [TECH]                : {aspect ratio, quality, lighting, camera}
  ```

### Stage 2 — 마스터 키비주얼 1컷 · Higgsfield `generate_image`
- 정면·중립 표정·전신 또는 상반신 1컷을 **기준 원판**으로 먼저 뽑는다.
- 모델 선택 확신 없으면 `models_explore(action:'recommend')` 먼저. (캐릭터 일관성엔 soul 계열이 강함)
- 마음에 들 때까지 여기서 끝낸다. **원판이 흔들리면 시리즈 전체가 흔들린다.**

### Stage 3 — 캐릭터 등록 (일관성 잠금) · Higgsfield
- 확정된 원판을 캐릭터/레퍼런스로 등록해 이후 컷이 같은 인물을 참조하게 한다.
- `show_characters` / `show_reference_elements`로 등록 상태 확인.
- 이 등록본 + Stage1의 `[LOCKED]` 블록 = **이중 잠금**.

### Stage 4 — 시리즈 배리에이션 · Higgsfield `generate_image` ×N
- `[LOCKED]` 고정, `[FREE]`만 교체하며 N컷.
- 시리즈 축(예: 표정 세트 / 사계절 / 감정 / 직업 코스프레 / 브랜드 콜라보)을 프로젝트 노트에 정의하고 그 축대로 뽑는다.
- **매 컷 등록 캐릭터를 참조**시켜 얼굴 드리프트 방지.

### Stage 5 — 모션/영상 (선택) · Higgsfield `generate_video`
- 정지 시트가 확정된 뒤에만. 마스터 이미지를 입력으로 image-to-video.
- 릴스·인트로·루프용. 필요 시 `storyboard-maker`로 컷 설계 선행.

### Stage 6 — 후처리 · Higgsfield
- `upscale_image` (2K/4K 납품본) / `remove_background` (캐릭터 시트·스티커·合成용 투명 배경) / `outpaint_image` (배너 여백 확장).

### Stage 7 — 검수 & 발송 · Claude
- `design-critique`: 얼굴 일관성·비율·시그니처 컬러 유지·잘림 6영역 검수 🟢🟡🔴.
- `delivery-gate`: 라이트모드·잘림·PII 린트 후 발송.

---

## 3. 시리즈 일관성 체크리스트 (컷마다)

- [ ] `[LOCKED]` 블록을 **글자 그대로** 복사했는가
- [ ] 등록 캐릭터/레퍼런스를 참조로 넣었는가
- [ ] 시그니처 컬러가 배경/의상에 잡아먹히지 않았는가
- [ ] 얼굴 구조·눈매가 원판과 같은가 (다르면 재생성, 통과 금지)
- [ ] aspect ratio가 용도(시트 1:1 / 배너 16:9 / 릴스 9:16)에 맞는가

---

## 4. 반복 실수 방지 (Guard)

- ❌ 원판 없이 바로 N컷 → 얼굴 제각각. **반드시 Stage2 원판 먼저.**
- ❌ 불변 요소를 말로만 → 드리프트. **`[LOCKED]` 텍스트 + 캐릭터 등록 이중 잠금.**
- ❌ 배경·의상을 불변에 넣음 → 시리즈가 '같은 옷 반복'으로 지루. 배경/의상은 가변.
- ❌ 영상부터 → 정지 원판 확정 후 image-to-video.
- ⚠️ 생성물의 라이선스·상업 사용 범위는 Higgsfield 플랜 기준 확인 (납품/판매 전 `legal-compliance`).

---

## 5. 요약 한 줄

**관점 1문장 → 불변/가변 분리 → 마스터 원판 1컷 → 캐릭터 등록(잠금) → 가변만 바꿔 N컷 → 후처리 → 검수.**
