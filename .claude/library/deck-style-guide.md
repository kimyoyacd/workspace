## 1. 캔버스 · 그리드
- 슬라이드 규격: **1920 × 1080 px** (16:9)
- 바깥 여백(세이프): 좌우·상하 **40px** 이상 (헤더/푸터 라인 기준)
- 콘텐츠 슬라이드는 **오프화이트 페이지 위에 흰 콘텐츠 패널**을 얹는 구조.
- 자산·카드형 슬라이드는 **격자 그리드**로 정렬 (도식 D5 참조).

## 2. 공통 크롬 (모든 슬라이드 상·하단 고정)
- **헤더** (y≈28): 좌 `coway X HIVELAB` · 중앙 `PROPOSAL` · 우 `2026`
  - 소형(14px) · **대문자 · 자간 넓게(tracking)** · Inter Semi Bold
  - 다크 슬라이드에서는 저채도 회색, 라이트 슬라이드에서는 진회색.
- **푸터(커버 전용)**: 좌측 저작권 고지 1문단 + 우측 라벨 3열
  - `DEVICE / DATE / CLIENT` 라벨(위) + 값(아래). 예: `7" 1024×600 Touch` · `2025. 04` · `Coway BEREX`
- 헤더/푸터는 **위치·크기 고정**. 슬라이드마다 흔들지 않는다.

## 3. 컬러 토큰
| 토큰 | 값 | 용도 |
|---|---|---|
| Cover BG | `#000000`~`#0A0A0A` | 커버·챕터·엔드 슬라이드 배경(다크) |
| Page BG | `#F7F6F2` | 콘텐츠 슬라이드 페이지(오프화이트) |
| Panel/Card | `#FFFFFF` | 콘텐츠 패널·카드 |
| Ink | `#111111` | 본문 텍스트(라이트) |
| Ink Invert | `#FFFFFF` | 텍스트(다크 배경) |
| Muted | `#8A8A8A` | 헤더/캡션/보조 텍스트 |
| Accent(dot) | 오렌지 `#FF5A2C`(계열) | 카테고리 점·강조 포인트 |
| Goal | 레드 `#E23B2E`(계열) | 지향점(Goal) 표기 — 스케일 맵 전용 |

> 프로젝트 GUI 팔레트(예: BEREX `#79A7C4`·`#202830`·`#030405`)는 **콘텐츠**용이며 덱 크롬 색과 별개다.

## 4. 타이포
- **라틴 디스플레이/헤더**: Inter (Bold~Semi Bold), 커버 타이틀은 초대형·타이트한 볼드.
- **한글**: Pretendard (제목 Bold / 본문 Regular~Medium).
- 위계: 커버 타이틀(초대형) > 섹션 라벨(소형 `Design Proposal`) > 슬라이드 타이틀(볼드 `B Type`) > 본문.
- 캡션/헤더는 14px 내외 · 자간 넓게 · 대문자.

## 5. 슬라이드 아키타입
1. **커버(다크)** — 초대형 볼드 타이틀 + 하단 DEVICE/DATE/CLIENT 라벨.
2. **인덱스/목차** — 번호(01·02…) + 영문 타이틀 + 한글 캡션 행 반복.
3. **콘텐츠(라이트)** — 좌상단 섹션 라벨 + 볼드 타이틀, 2열(요약 볼드 + 본문), 하단 자산 영역.
4. **챕터/엔드(다크)** — 중앙 초대형 문구 (예: `End Of Document`).

## 6. 도식화(Diagram) 방식 카탈로그

> 반복해서 쓰는 6종 레이아웃. 빈 와이어프레임 템플릿 → 콘텐츠를 채워 완성.
> 모든 도식은 §2 크롬 위에 얹고, 라이트 슬라이드 기준.

### D1. 허브–스포크 관계도  (SoT node `276:2660`)
- **용도**: 하나의 중심 개념 ↔ 다수 요소의 관계/파생 구조.
- **구조**: 중앙 큰 타원(허브) + 6개 위성 타원(스포크) + 좌우 **박스 그리드 클러스터**로 세부 항목.
- **규칙**: 허브는 1개, 스포크는 4~6개 권장. 연결선은 얇은 직선. 좌우 클러스터는 대칭 정렬.

### D2. 프로세스 · 저니 체인  (SoT node `276:2724`)
- **용도**: 단계/여정/타임라인, 여러 트랙(행)의 진행 비교.
- **구조**: **행 5개** — 각 행 = 좌측 아이콘+라벨 / 중앙 **7노드 연결 체인** / 우측 결과 박스.
- **규칙**: 노드 개수는 트랙 간 통일. 행마다 라벨-체인-결과 축을 수평 정렬.

### D3. 텍스트 + 이미지 갤러리  (SoT node `276:2737`)
- **용도**: 개념 설명 + 근거 비주얼(레퍼런스·목업) 동시 제시.
- **구조**: 좌측 텍스트 컬럼(제목 블록 + 칩 + 본문) / 우측 **이미지 5분할 갤러리**(대·중 혼합).
- **규칙**: 좌:우 ≈ 1:2. 이미지 비율 유지, 캡션 통일.

### D4. 3분할 핵심 철학 + 모티프  (SoT node `276:2752`)  ★"세 가지 차별화된 핵심 철학"
- **용도**: 브랜드/컨셉의 **3대 핵심 철학**과 각 전용 모티프 체계를 병렬로.
- **구조**: **세 컬럼**(수직 구분선) — 컬럼당 번호 칩 + 제목 2블록 + **칩 3개(모티프)** + 본문 + 하단 요약 박스.
- **규칙**: 항상 **3열 등분**. 세 철학은 서로 배타적·대등하게. 컬럼 간 시각 무게 동일.

### D5. 브랜드 자산 디렉토리 그리드  (SoT 빈템플릿 `276:2765` / 채운 예 `276:2778`)  ★"격자 그리드 자산 디렉토리"
- **용도**: 카테고리별 **브랜드/레퍼런스 자산을 한 화면에 카탈로그화**.
- **구조**: 상단 `Category X` 라벨 + 볼드 타이틀 + 2줄 요약 · 하단 **카드 격자(가로 5칸 기준, 행은 내용에 따라 2~4행)**.
- **카드**: 로고/이미지 + 이름(볼드) + 1~2줄 설명. 카드 배경 흰색, 페이지 오프화이트.
- **규칙**: 열 수 고정(5), 행은 자산 수에 맞게. 카드 간격·크기 통일. 카테고리 점(오렌지)로 구분.

### D6. 브랜드 지향점 스케일 맵  (SoT node `276:2793`)  ★"브랜드 지향점 정밀 스케일 세팅"
- **용도**: 경쟁 자산을 2축 위에 배치하고 **우리 지향점(Goal)**을 정밀하게 세팅.
- **구조**: 옅은 **격자 위 XY 좌표계** — 예: X `Commercial ↔ Cultural`, Y `Global ↔ Seoul&Local`. 브랜드 로고를 데이터 포인트로 배치 + **빨간 원 `Goal`**으로 지향점 표시.
- **규칙**: 축 라벨 4개 명시. Goal은 1개, 레드 원. 로고 위치는 근거 기반(임의 배치 금지). market-research 포지셔닝 맵과 논리 일치시킬 것.
