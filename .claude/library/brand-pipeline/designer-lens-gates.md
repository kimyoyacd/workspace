# 06 · Designer Lens Insertion Map

**역할:** 기존 1차·2차·3차 프로세스에 디자이너 관점을 어디에 꽂아야 하는지 안내하는 삽입 지도입니다.  
**중요:** 기존 흐름은 유지합니다. 제거해야 하는 것은 외부 출처명/프로세스명 노출이며, 내부 작동 구조가 아닙니다.

---

## 0. 핵심 정리

디자이너 관점은 별도 부록으로 붙이면 힘이 약해집니다.  
기존 프로세스 안에서 **판단 게이트**로 들어가야 합니다.

```text
1차: 리서치·포지셔닝 단계
→ 디자이너 관점은 ‘관점 한 문장’과 ‘시장 빈자리 해석’에 삽입

2차: 비주얼 아이덴티티·컨셉 단계
→ 디자이너 관점은 ‘자기 렌즈’와 ‘행동하는 비주얼 시스템’에 삽입

3차: 브랜드 시스템·슬라이드 생성 단계
→ 디자이너 관점은 ‘시스템 토큰’, ‘휴먼터치 레이어’, ‘리스크 정렬 체크’에 삽입
```

## 1. 어디에 넣어야 하는가

| 기존 단계 | 기존 역할 | 추가할 디자이너 관점 | 삽입 위치 | 이유 |
|---|---|---|---|---|
| 1차 · 경쟁 브랜드 30개 리서치 | 시장과 경쟁 구도 파악 | Blanding 탈출 / 동질화 진단 | 경쟁사 매트릭스 필드에 추가 | 경쟁 브랜드가 서로 비슷해지는 지점을 먼저 잡아야 포지셔닝이 살아납니다. |
| 1차 · 클러스터링 | 경쟁 브랜드 묶기 | 8개 비주얼 트렌드 태그 | 클러스터명 옆에 태그로 추가 | 클러스터가 단순 가격/타깃 분류에 머물지 않게 만듭니다. |
| 1차 · 포지셔닝 맵 | 브랜드가 들어갈 자리 찾기 | 핵심가치 → 관점 한 문장 | 추천 포지셔닝 맵 직후 | “어디에 위치할까”를 넘어 “무엇을 믿는 브랜드인가”로 전환합니다. |
| 1차 · 선호도 분석 | 타깃 취향 파악 | 헤리티지 보존 vs 파괴 스펙트럼 | 선호도 분석 후 | 과감한 방향을 택할 때 지킬 것과 바꿀 것을 분리합니다. |
| 1차-B · 제품 비주얼 프롬프트 | 제품 사진/패키지 프롬프트 | 휴먼터치 레이어 / 물성 / 불완전성 | Physics & Blending 뒤 | AI 이미지가 너무 매끈해지는 것을 막고 사람의 흔적을 설계합니다. |
| 2차 · Visual Identity DNA | 시각 DNA 추출 | 핵심가치 → 자기 렌즈 → 비주얼 시스템 | DNA 추출 첫머리 | 시각 요소가 취향 나열이 아니라 관점의 증거가 됩니다. |
| 2차 · Competitive Visual Code | 경쟁 시각 코드 요약 | 이미 과밀한 코드 / 덜 쓰인 코드 / 금지 코드 | 기존 코드 요약과 통합 | 레퍼런스 수집이 아니라 차별화 판단으로 전환합니다. |
| 2차 · Concept Territories | 컨셉 발산 | 행동하는 시스템 / 모션 비헤이비어 / 가변 규칙 | 각 컨셉 필드에 추가 | 정적 무드보드가 아니라 확장 가능한 브랜드 시스템이 됩니다. |
| 2차 · Client×Mood Matrix | 클라이언트 적합도 평가 | 리스크 정렬 체크 | 매트릭스 평가 항목에 추가 | 화제성만 있고 브랜드 가치와 어긋나는 방향을 걸러냅니다. |
| 3차 · Brand Board | 브랜드 기준판 | Behavior System / Human Touch System | Visual System 뒤 | 브랜드가 어떻게 움직이고 변주되는지까지 기준화합니다. |
| 3차 · Value Cards | 가치 실행 카드 | 가치별 시각 증거 / 행동 증거 | Visual Expression 뒤 | 가치가 추상어로 끝나지 않게 합니다. |
| 3차 · Copy & Symbol | 언어와 기호 체계 | 커스텀 타입 / 상징 / 반복 기호 | Symbol Association에 추가 | 핵심가치가 글자와 기호로 증명됩니다. |
| 3차 · Final Deck | 전체 슬라이드 생성 | 관점 한 문장 → 시스템 → 이미지 → 슬라이드 | Executive Summary 앞뒤 | 첫 장부터 “우리가 어떻게 생각했는가”가 보입니다. |

## 2. 추가해야 할 핵심 게이트

### Gate 1. Strategic POV Gate

**위치:** 1차 포지셔닝 전략 초안 직후  
**질문:**

```md
이 브랜드는 어떤 시장 빈자리를 보는가?
이 브랜드는 무엇을 다르게 믿는가?
이 관점은 한 문장으로 말할 수 있는가?
```

**통과 기준:**

```md
관점 한 문장:
이 관점이 필요한 이유:
경쟁사와 다른 믿음:
타깃이 반응할 감정:
```

---

### Gate 2. Lens Translation Gate

**위치:** 2차 Visual Identity DNA 앞  
**질문:**

```md
핵심가치가 어떤 렌즈로 세상을 해석하는가?
그 렌즈가 색, 타입, 소재, 움직임으로 번역되는가?
```

**통과 기준:**

```md
Core Value:
Hyojung Lens:
Visual Proof:
Motion / Behavior Proof:
Human Touch Proof:
```

---

### Gate 3. Living System Gate

**위치:** 2차 Concept Territories 직후  
**질문:**

```md
이 컨셉은 정적 이미지에서 끝나지 않는가?
모션, 가변폰트, 반응형 규칙, 생성 규칙으로 확장되는가?
```

**통과 기준:**

```md
Static Identity:
Motion Behavior:
Responsive Rule:
Variable Type Rule:
Generative Rule:
```

---

### Gate 4. Human Touch & Risk Gate

**위치:** 3차 브랜드 시스템 완성 직전  
**질문:**

```md
AI가 만든 것처럼 매끈하기만 하지 않은가?
어디에 인간의 흔적을 남길 것인가?
과감한 방향이 핵심가치와 정렬되어 있는가?
```

**통과 기준:**

```md
AI Use Scope:
Human Touch Layer:
Imperfection Rule:
Heritage Preserve:
Heritage Transform:
Risk Alignment:
```

## 5. 한 줄 결론

디자이너 관점은 “트렌드 참고”가 아니라 **관점 → 렌즈 → 시스템 → 검증 게이트**로 들어가야 합니다.  
그래야 단순 자료 요약이 아니라, 효정님식 전략 디자인 프로세스로 작동합니다.

---

## 6. Designer Lens 6원칙 (모든 단계 공통 반영)

1. **Core Value First** — 비주얼보다 먼저 핵심가치 + 전략 한 문장을 정한다.
2. **Own Lens** — 경쟁사 스타일을 그대로 쓰지 않고 브랜드만의 관찰 렌즈로 재해석.
3. **Behavior System** — 로고·컬러·타입을 정적 산출로 끝내지 않고 모션·반응형·생성 규칙까지 확장.
4. **Human Touch Layer** — AI 이미지 사용 시 반드시 사람의 흔적 설계(그레인·손그림·스캔 텍스처·의도적 비대칭).
5. **Risk Alignment** — 과감한 스타일은 핵심가치·고객 기대·헤리티지와 정렬될 때만 채택.
6. **Preserve vs Destroy** — 무엇을 보존하고 무엇을 파괴할지 명시.

## 7. 단계별 자체검토 4렌즈
각 단계 산출 후 다음 4렌즈로 자가 점검 → Gate 판정(🟢/🟡/🔴):
- **Strategy Fit** · **Visual Potential** · **System Expandability** · **Risk Alignment**
