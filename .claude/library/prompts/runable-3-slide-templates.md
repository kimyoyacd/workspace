# Runable 3 · Slide Template System (Brand/Project Story Deck)

**역할**: Runable 3 최종 슬라이드 생성 시 사용하는 재사용 가능한 슬라이드 타입 카탈로그. **브랜드 프로젝트 전용이 아니다** — `design-deck-wireframe.md`(범용 덱 골격)의 6번 섹션("최종 시스템 슬라이드")에서도 그대로 재사용하도록 설계되어 있다. 홈페이지·배너·게임UI 등 비-브랜드 프로젝트도 이 6종을 그대로 쓰되, "Brand"를 "Project"로 바꿔 읽으면 된다.
**원칙**: 모든 슬라이드는 아래 글로벌 디자인 시스템을 일관되게 따른다. 프로젝트마다 [내용]만 교체하고 [디자인] 레이아웃 스펙은 그대로 유지한다.

---

## ★ 두 카드 체계 공존 원칙 — "억지로 하나로 합치지 말 것"

같은 덱(deck.html) 안에 **의도적으로 서로 다른 두 카드 형태**가 공존한다. 둘 다 아래 색상 토큰·타이포 스케일을 공유하는 하나의 시스템이지만, 형태(고정/자유높이)는 용도에 따라 다르다 — 이걸 하나로 강제 통일하면 오히려 원본 스펙을 깨뜨린다.

| 구분 | 용도 | 형태 | 예시 |
|---|---|---|---|
| **cat-card** (회색, 자유높이) | 리서치/전략/과정 섹션 — 경쟁사 카드 그리드, 컨셉 카드, 무드 전략 등 내용량이 가변적인 곳 | 회색 배경(`--bg`), `border-radius:16px`, 42px 타이틀, 고정 비율 없음(내용에 따라 자연스럽게 늘어남) | Market Research 브랜드 카드, Concept Territories |
| **slide16** (흰색, 고정 16:9) | Runable 3 최종 결과물 — 이 문서의 6종 슬라이드 타입 | 흰 배경, `aspect-ratio:16/9` 고정, 얇은 하단 구분선 헤더 | Brand Story / Value Keywords / Mind Map / Direction / Graphic Motif / Competitor Research |

새로운 슬라이드 타입을 만들 때(예: 무드보드 오프닝, 퍼스널리티 슬라이더처럼 이 6종에 없는 콘텐츠)도 마찬가지 원칙 적용: **내용량이 가변적이고 자유 흐름이면 cat-card 계열**, **최종 확정 결과물로 한 화면에 딱 맞춰야 하면 slide16 계열**을 골라 쓴다. 어느 쪽이든 색상 토큰(`--bg #EBEBEB`, `--ink #111`, `--ink2 #4A4944`, `--ink3 #8A8780`, `--border #E2E2E2`, 링크색 `#2454C7`, 경고색 `#B23B3B`)과 타이포 굵기 위계(타이틀 800, 본문 400~700)는 항상 동일하게 가져간다 — 그래야 형태가 달라도 "같은 덱"으로 읽힌다.

## 글로벌 디자인 시스템 (모든 슬라이드 공통)

```
- 슬라이드 비율: 16:9 와이드스크린
- 배경: 순백색
- 서체: IBM Plex Sans KR (또는 Pretendard), 한글-영문 혼용
- 페이지 타이틀: 볼드, 좌측 상단 정렬, 아래에 얇은 가로 구분선
- 안내 문구: 우측 상단, 작은 크기, 타이틀과 같은 행
- 컬러: 블랙, 화이트, 컨셉별 포인트 컬러(A=레드, B=시안블루, C=블랙 — 프로젝트별 재정의 가능)
- 여백: 미니멀, 문단은 짧게, 문장 중간 줄바꿈 없이 끝까지
- 그리드: 모든 요소 일관된 그리드에 정렬
```

색상 매핑은 프로젝트마다 재정의한다 — GRAIN의 경우: A=스틸그레이(#8C8C8C), B=브라스(#8A6E3F), C=크림(#EDEAE2).

## Slide Type 1 — Brand Story

**[디자인]**
- 페이지 상단에 앞 가로 구분선
- 좌측 상단: 브랜드명, 아래로 2줄 서브헤드라인, 그 아래에 태그 뱃지 2개 나란히 배치
- 제품 사진 5장을 2열 그리드로 배치: 상단 행=좌측에 정사각형 사진 1장, 우측 끝에 세로형 사진 1장(가운데 여백), 하단 행=좌측에 세로형 사진 1장(가운데 여백)
- 본문: 단락 4개, 작은 크기, 타이틀들과 같은 폭

**[내용: 브랜드에 맞게 교체]**
- 페이지 라벨: Brand Story
- 브랜드명: [브랜드명]
- 서브헤드라인: [한 줄 슬로건]
- 태그: [카테고리1] / [카테고리2]
- 본문 단락1: [브랜드가 파는 것/일상 장면 묘사]
- 본문 단락2: [브랜드가 만드는 것, 제품 철학]
- 본문 단락3: [감각적 루틴에 대한 브랜드 믿음]
- 본문 단락4: [타겟 고객 정의]
- 제품 사진: [제품1~5 샷/앵글 이미지, 워터마크·로고 없음]

## Slide Type 2 — Brand Value Keywords (3 Concept Cards)

**[디자인]**
- 페이지 타이틀: Brand Value Keywords 좌측 상단, 안내 문구 우측 상단, 아래 얇은 구분선
- 3개 컨셉 카드를 동일한 폭으로 나란히 배치
- 각 카드: 좌측 상단에 대형 볼드 컨셉 알파벳(A/B/C), 아래에 대형 디스플레이 타입 컨셉 타이틀(2~3줄, 볼드), 아래에 작은 일반 웨이트 본문 단락, 아래에 "그래픽 모티프" 레이블과 모티프 키워드
- 각 카드 하단: 점으로 구분된 서브 키워드 태그
- 카드 배경 없음, 얇은 세로 구분선 또는 넉넉한 간격으로만 구분
- 컨셉 타이틀 컬러: A=포인트컬러1, B=포인트컬러2, C=포인트컬러3

**[내용: 브랜드에 맞게 교체]**
- 페이지 라벨: Brand Value Keywords
- 안내 문구: 이 브랜드의 핵심에 가장 가까운 방향을 1가지 선택해주세요
- 카드A 타이틀: [컨셉A 영문 타이틀] / 본문: [컨셉A 브랜드 철학 설명] / 모티프: [그래픽 모티프 키워드] / 서브 키워드: [키워드1, 키워드2, 키워드3]
- 카드B, 카드C 동일 구조

## Slide Type 3 — Brand Mind Map

**[디자인]**
- 페이지 타이틀: Brand Mind Map 좌측 상단, 안내 문구 우측 상단, 아래 얇은 구분선
- 중앙: 브랜드명, 바로 아래 슬로건 소형 텍스트, 어두운 채워진 둥근 사각형 또는 타원 안에 배치
- 6개 가지 노드가 중앙에서 바깥으로 방사, 얇은 곡선 또는 직선으로 연결
- 각 가지 노드: 볼드 대문자 카테고리명, 얇은 아웃라인 박스 안에 배치
- 각 노드 주변: 연관 키워드들을 작은 일반 텍스트로 자연스럽게 흩어 배치
- 전체 구조: 유기적, 비대칭, 완벽한 좌우대칭 아님, 넉넉한 여백, 선은 얇고 미니멀

**[내용: 브랜드에 맞게 교체]**
- 중앙: [브랜드명] / [핵심 슬로건]
- 가지1~6: [브랜드 가치] / [키워드1, 키워드2, 키워드3...]

## Slide Type 4 — Brand Direction (Spectrum)

**[디자인]**
- 페이지 타이틀 좌측 상단, 안내 문구 우측 상단, 아래 얇은 구분선
- 5개 축을 충분한 간격을 두고 세로로 나열
- 각 축: 좌측 끝에 축 번호(01~05), 좌측 개념 레이블 + 작은 설명 텍스트, 중앙에 5~7개 선택 점이 있는 가로 점선/실선, 우측 개념 레이블 + 작은 설명 텍스트
- 모든 축 라인 완벽하게 정렬

**[내용: 브랜드에 맞게 교체]**
- 페이지 라벨: Brand Direction
- 안내 문구: 각 축에서 이 브랜드가 위치해야 한다고 생각하는 지점을 선택해주세요
- 축 01~05: [좌측 개념] ↔ [우측 개념] (예: 정적인↔역동적, 실용적인↔예술적인, 대중적 가격대↔프리미엄 가격대, 단일시스템↔시스템적, 함축적인↔서술적인)

## Slide Type 5 — Graphic Motif (Image Board)

**[디자인]**
- 페이지 타이틀 좌측 상단, 안내 문구 우측 상단, 아래 얇은 구분선
- 3개 컨셉 섹션을 동일한 폭으로 나란히 배치
- 각 섹션: 상단에 볼드 컨셉 레이블 + 포인트 컬러(A/B/C)
- 레이블 아래: **컨셉 1개당 6컷을 3x2 그리드 1장으로 생성**(Higgsfield 그리드 프롬프트, aspect_ratio "1:1") — 낱장 4~6개를 따로 생성해 배치하지 않는다. 조명·톤 일관성을 한 생성 안에서 강제하기 위함(`brand-design-proposal.md`의 "Higgsfield 이미지 생성 원칙" 참조)
- 모든 이미지: 흑백 또는 저채도, 추상적 그래픽 모티프와 아날로그 질감 혼합, 대비감, 인물·제품 없음
- 열 구분: 얇은 세로 구분선 또는 넉넉한 여백

**[내용: 브랜드에 맞게 교체]**
- 페이지 라벨: Graphic motif
- 섹션A/B/C 레이블 + 컨셉 타이틀
- 섹션A/B/C 이미지: [컨셉당 6컷 3x2 그리드 1장, 컨셉 무드에 맞는 추상 텍스처/패턴]

## Slide Type 6 — Competitor Research

**[디자인]**
- 페이지 타이틀 좌측 상단, 부제 우측 상단, 아래 얇은 구분선
- 5열 x N행 브랜드 카드 그리드
- 각 카드: 상단에 브랜드 로고 또는 고유 서체 스타일로 브랜드명, 아래에 볼드 브랜드명, 작은 회색 카테고리 고리 텍스트, 작은 본문 텍스트 설명 2줄
- 카드 구분: 얇은 연한 회색 구분선
- 전체 느낌: 레퍼런스 시트, 정보 밀도 있지만 가독성 유지, 미니멀

**[내용: 브랜드에 맞게 교체]**
- 페이지 라벨: Competitor Research
- 부제: [시장 카테고리] 경쟁사
- 브랜드 카드: [브랜드명 / 카테고리 / 설명 2줄] × N개 (기존 리서치한 30개 브랜드 재사용)

## 실행 순서 (Runable 3 단계에서)

1. Manus 1의 `01-competitor-matrix.md`를 Slide Type 6(Competitor Research)로 변환
2. Manus 2의 Concept Shortlist(2~3개)를 Slide Type 2(Brand Value Keywords)로 변환
3. Manus 1의 Positioning Strategy + Manus 2의 Visual DNA를 Slide Type 3(Brand Mind Map)으로 변환
4. Manus 1의 헤리티지 스펙트럼 등 방향성 결정들을 Slide Type 4(Brand Direction)로 변환
5. Manus 2의 Visual DNA(Form/Color/Material 등)를 Slide Type 5(Graphic Motif)로 변환 — 이미지는 Higgsfield로 메인이 직접 생성
6. 관점(Gate 1 POV) + 브랜드/프로젝트 핵심 서사를 Slide Type 1(Brand/Project Story)로 변환 — 이 슬라이드가 보통 덱 맨 앞(오프닝)에 배치되지만, 타입 자체는 이 6종 세트의 일부로 마지막에 완성해도 무방(오프닝 배치 순서와 제작 순서는 별개).

## 비-브랜드 프로젝트에서 재사용할 때
- `design-deck-wireframe.md`를 따르는 홈페이지/배너/게임UI 등 프로젝트도 이 6종을 그대로 쓴다. 용어만 치환: "Brand Story"→"Project Story", "Brand Value Keywords"→"Project Value Keywords" 등. [디자인] 레이아웃 스펙은 절대 바꾸지 않는다 — 바뀌는 건 항상 [내용]뿐.
- Manus/Runable 게이트를 생략·축약한 프로젝트(design-deck-wireframe 경유)라도, 이 6종 슬라이드는 축약하지 않고 6개 다 채운다 — "이 프로젝트는 작으니 3개만" 식으로 임의로 개수를 줄이지 않는다. 정말 해당 없는 타입(예: 경쟁사 리서치가 없는 내부 툴 프로젝트)만 명시적으로 "해당없음" 표기 후 제외.

모든 슬라이드는 `deck.html`에 순서대로 누적하며, 색상/서체 등 디자인 시스템은 슬라이드 전체에서 통일한다.
