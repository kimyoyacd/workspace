# 무드보드 마스터 프레임워크 (Moodboard Master Framework)

> 무드보드는 이미지 수집 도구가 아니라 **"왜 이 방향인가"를 설득하는 시각적 논증(Creative Argument)**이다.
> 이 파일은 브랜드/디자인 프로젝트에서 무드보드 단계에 진입할 때 **자동으로 먼저 확인**한다 (`brand-design-proposal.md`, `design-deck-wireframe.md`와 함께 트리거).
> 담당 에이전트: `moodboard-builder`(빌드) / `brainstormer`(퍼스널리티 슬라이더·키워드 발산) / `market-research`(리서치 근거) / `concept-director`(오프닝 관점 게이트) / `critic`(실현가능성 검증) / `project-continuity-auditor`(사후 일관성 점검).

---

## 0. 덱 구조 원칙 — "결론 먼저, 근거는 그다음"

무드보드를 포함한 제안 덱은 다음 순서로 짠다. 리서치 나열부터 시작하지 않는다.

### 오프닝 3~4장 상세 스펙

| 장 | 이름 | 내용 | 분량 기준 |
|---|---|---|---|
| 1 | 관점(POV) | Gate 1에서 확정한 한 문장 + 왜 이 관점이 필요한지(경쟁사와 다른 믿음) | 1장, 문장 1개 + 근거 3줄 |
| 2 | 결과물 정의 | 그 관점에서 무엇을 만들기로 했는가 — 제품/시스템/톤을 명사형으로 명확히 | 1장, 키워드 3~5개 + 스케치·목업 1컷 |
| 3 | 메시지 | 타깃이 이걸 보고 느껴야 할 감정과 행동(구매/공유/재방문 등) | 1장, 감정어 1개 + 행동 1개 |
| 4 (선택) | 두 루트 예고 | Safe Route와 Wild Card Route를 이 시점에 한 줄씩만 예고(§2로 연결되는 다리) | 1장, 각 루트 1줄 요약 |

**체크: 오프닝 3~4장만 보고도 클라이언트가 "그래서 결론이 뭔데?"를 다시 묻지 않아야 한다.** 다시 묻는다면 관점 문장이 너무 추상적이거나, 결과물 정의가 형용사 위주(예: "고급스러운")로만 되어 있고 명사형 결정이 없는 것 — 즉시 §5 [1단계]로 되돌아가 재작업한다.

**그다음부터 상세 리서치** — 위 3~4장에서 이미 결론을 봤기 때문에, 이후 리서치 파트는 "증거 제시"로 읽힌다. 순서가 뒤바뀌면(리서치→결론) 지루하고 설득력이 떨어진다. 리서치 파트 진입 시 각 리서치 슬라이드 상단에 "이 자료는 앞의 O번 결론을 뒷받침한다"는 역참조 한 줄을 넣어 오프닝과의 연결을 끊지 않는다.

### ★ 슬라이드 분량 원칙 — "산출물 1개 = 슬라이드 1장, 압축 금지"

- 어느 단계(§1 리서치, §5 각 단계, §6 이미지 주석 등)에서든 **도출된 결과물은 분량에 관계없이 각각 별도 슬라이드로 만들어 덱에 즉시 반영**한다. 여러 산출물을 한 슬라이드에 욱여넣어 "요약"하지 않는다.
  - 예: 타깃 관심사 리서치에서 근거 있는 항목이 5개 나왔으면 "타깃 관심사 요약 1장"이 아니라 **5장**으로 만든다. 퍼스널리티 슬라이더 축이 5개면 "슬라이더 종합 1장"이 아니라 **축마다 1장씩 5장**(또는 한 장에 5축을 다 넣더라도 그 장표 자체는 독립 슬라이드로 관리하고 세부 근거는 각 축별 하위 슬라이드로 펼침).
  - 이유: 분량을 줄이려고 압축·요약하면 §6의 "선정 이유/담긴 이야기" 같은 서사적 근거가 잘려나가 무드보드가 다시 "그냥 예쁜 이미지 나열"로 퇴행한다. 슬라이드 수가 많아지는 건 문제가 아니다 — 오히려 산출물 하나하나가 논증으로 살아있다는 증거다.
  - 예외: §0의 오프닝 3~4장만은 분량을 압축한 요약본으로 유지한다(전체 덱의 "결론 먼저" 원칙 때문에 의도적으로 응축). 오프닝 이후의 모든 파트는 압축하지 않는다.
- **업데이트 방식**: 산출물이 나올 때마다 그 즉시 해당 슬라이드를 만들어 `deck.html`/`deck-v2.html`에 append 한다(브랜드 파이프라인의 "산출물 누적 원칙"과 동일 — 세션 끝에 몰아서 정리하지 않는다). 슬라이드 순서는 산출 순서를 따르되, 최종 정리 시 §0→§1→§2… 섹션 순서로 재배열해도 된다(내용 손실 없이 순서만 바꾸는 것은 허용).
- **너무 많아지면 어떻게 하나**: 슬라이드 수 자체를 줄이지 않고, 대신 목차/네비게이션(섹션 탭, 앵커 링크)을 추가해 탐색성을 확보한다. "발표용 요약 버전"이 필요하면 원본은 그대로 두고 별도 export만 만든다(원본을 줄이지 않는다).

## 1. 리서치는 시장조사만이 아니다 — 3중 리서치 원칙

기존 경쟁사/포지셔닝 리서치(`market-research`)에 아래 2개 축을 **항상 같이** 채운다. 하나만 있으면 불완전하다.

| 축 | 무엇을 담나 | 담당 | 산출 포맷 |
|---|---|---|---|
| ① 경쟁/포지셔닝 | 경쟁 브랜드 XY 매핑, 화이트스페이스 | `market-research` | 포지셔닝 맵 + 브랜드 카드(전체 필드+출처) |
| ② 디자인 트렌드 | 최근 3개월 글로벌 디자인 트렌드, 출처 필수 | `design-trend-radar` | 트렌드명 + 사례 이미지/링크 + "우리와 무슨 관계인가" 1줄 |
| ③ 타깃 관심사 | **이 타깃층이 지금 무엇에 관심 있는지**(문화적 신호·소비 패턴·화제), 왜 그렇게 판단했는지 근거·출처 | `market-research` 또는 `brainstormer`에 위임, 반드시 출처 표기 | 관심사 항목 + 근거자료(기사/데이터/SNS) 링크 + 발췌문 |

### ③ 타깃 관심사 리서치 — 세부 체크리스트 (가장 자주 누락되는 축)
"이 타깃이 요즘 이런 방향을 좋아한다"는 문장은 반드시 근거를 동반해야 하며, 근거 없이 감으로 쓰지 않는다. 근거로 인정하는 자료 유형:
1. **문화적 신호** — 화제가 된 전시/콘텐츠/밈, 발생 시점과 확산 경로
2. **소비 패턴** — 실제 판매·검색량·펀딩 데이터(가능한 범위에서), 출처 표기
3. **커뮤니티 발화** — 포럼/SNS/리뷰에서 반복되는 표현(직접 인용 발췌 권장)
4. **인접 카테고리 이동 신호** — 이 타깃이 최근 다른 카테고리에서 무엇을 사기 시작했는가(카테고리 이동은 취향 변화의 선행 지표)

각 항목은 `타깃 관심 항목 / 근거 자료명·링크 / 근거 발췌 / 이게 우리 방향에 시사하는 바 1줄` 4칸 표로 정리한다. 근거 없는 항목은 "가설(미검증)"로 표기하고 별도 색상/뱃지로 구분해 확정 리서치와 섞이지 않게 한다.

### 3중 리서치 종합 원칙
세 축을 따로 나열만 하지 않고, 마지막에 "3축 교차점" 슬라이드 1장을 추가한다 — 경쟁사 빈자리 × 트렌드 방향 × 타깃 관심사가 겹치는 지점이 바로 §0의 "관점"이 서야 할 자리다. 세 축이 서로 다른 곳을 가리키면 관점을 다시 점검해야 한다는 신호.

## 2. 개념적 이원화 — Safe Route vs Wild Card Route

무드보드/방향 제안은 항상 **2개 루트**로 병행 제시한다. 하나만 보여주지 않는다.

| 항목 | 세이프/익스펙티드 루트 | 아스피레이셔널/와일드카드 루트 |
|---|---|---|
| 정의 | 클라이언트가 이미 머릿속에 그리고 있거나, 해당 업계에서 가장 안정적·클래식하게 통하는 예측 가능한 방향 | 디자이너의 크리에이티브 피네스를 주입, 현재 고객층이 아니라 **앞으로 끌어와야 할 타깃(Aspirational Audience)**의 시선에 맞춘 대담한 대안 |
| 근거 소스 | §1 ①경쟁/포지셔닝 리서치(업계 평균) | §1 ②트렌드 + ③타깃 관심사(특히 아직 검증 안 된 "가설" 항목) |
| 리스크 수준 | 낮음 — 클라이언트 승인 확률 높음 | 중간~높음 — `critic` 에이전트로 별도 실현가능성 검증 권장 |
| 실패 시 대응 | 거의 실패 없음(안전판) | 반려 시 Safe Route가 즉시 대체안으로 기능하도록 두 루트는 **완전히 다른 슬라이드로 분리**(하나를 수정해서 다른 하나로 만들지 않는다) |

- 두 루트 모두 §0의 "오프닝 3~4장" 구조(관점→정의→메시지)를 **각각** 갖는다. 클라이언트가 비교하며 판단할 수 있게, 같은 포맷·같은 슬라이드 순서로 나란히 제시한다.
- 프레젠테이션 시 반드시 "이 두 개 중 하나를 고르는 게 아니라, 두 극단 사이 어디에 설지를 같이 정하는 자리"라는 프레이밍으로 연다 — 이분법적 양자택일로 들리면 클라이언트가 방어적으로 Safe만 고르는 경향이 생긴다.
- Wild Card가 채택되지 않더라도 그 안의 개별 요소(컬러 하나, 모티프 하나)를 Safe Route에 이식하는 "부분 채택"을 항상 옵션으로 열어둔다.

## 3. 6버킷 프레임워크 — 비주얼 에셋 전개 고정 카테고리

이미지를 모을 때 아래 6개 서랍으로 나눠 담아 누락을 방지하고, 촬영/영상 스태프와의 테크니컬 싱크를 미리 확보한다.

| 버킷 | 내용 | 예시 | 누락 시 흔한 실패 |
|---|---|---|---|
| 1. Setting(장소/배경) | 공간 구조·톤·질감, 천장고·자연광 유입 등 | 콘크리트/벨벳 벽면, 야외 가로등 톤 | 촬영 당일 로케이션이 안 정해져 급하게 스튜디오로 대체 |
| 2. Action(행동) | 이야기가 있는 움직임(단순 포즈 X) | 무언가를 쥐는 손, 몰두한 자세 | 모델이 뭘 해야 할지 몰라 뻣뻣한 카탈로그 포즈로 귀결 |
| 3. Angle(카메라 앵글) | 아이레벨을 벗어난 구도 | 하이앵글, 매크로 클로즈업 | 전부 정면 아이레벨 컷만 나와 시리즈 전체가 밋밋 |
| 4. Lighting(조명) | 비주얼의 깊이·감정 | 강한 스트로보, 부드러운 면광, 역광 실루엣 | 조명 방향 미정으로 현장에서 임기응변, 톤 불일치 |
| 5. Edit(후반 편집) | 촬영 이후 톤앤매너 | 틸앤오렌지 보정, 콜라주 레이아웃, 필름 그레인 | 컷마다 보정자가 달라 세트로서 통일감 상실 |
| 6. BTS(비하인드 씬) | 숏폼용 현장감 | 짐벌 무빙, 스태프 인터뷰 톤 | 본 촬영만 신경 쓰고 SNS용 자료를 별도로 재촬영해야 함 |

**적용 팁**: 6버킷을 각각 최소 2~3장씩 채우면 한 컨셉당 12~18장 규모가 된다. 이걸 그대로 다 보여주지 않고, §7의 그리드 공식으로 9칸(또는 6칸)으로 압축 편집한 뒤 나머지는 부록(레퍼런스 풀)으로 남긴다. 6버킷은 "수집 단계의 체크리스트"이지 "발표 단계의 그리드 칸수"가 아니다 — 둘을 혼동해 18장을 그대로 다 붙이지 않는다.

## 4. 프로젝트 성격별 이원화 — Vibe Board vs Tactical Board

| 구분 | 바이브 보드 | 택티컬 보드 |
|---|---|---|
| 담는 것 | 브랜드 원형(Archetype), 추상적 감성, 지향 분위기·페르소나 | 타이포 시스템, 3D 아웃라인/레이아웃, 패키지 지기구조, 실제 UI 요소 |
| 목적 | 클라이언트가 "이 브랜드는 이런 세계관이구나"를 감으로 이해 | 제작 파트너(개발자·인쇄소·CAD)가 바로 작업 착수 가능한 수준의 구체성 |
| 만드는 시점 | §5 1~3단계 (전략·발산·내러티브) | §5 4단계 진입 직전, 승인된 방향에 대해서만 |
| 담당 | `moodboard-builder`(무드보드), `brainstormer`(모티프 발산) | `design-system-guardian`, `figma-bridge`, `moodboard-builder`(레이아웃 목업) |

두 보드를 한 슬라이드에 섞지 않는다 — 바이브 보드에 UI 목업이 끼어들면 "이게 최종 디자인인가"라는 오해를, 택티컬 보드에 감성 이미지가 끼어들면 "이 스펙이 확정인가 감인가"라는 혼란을 만든다.

## 5. 무드보드 4단계 파이프라인

### [1단계] 프리-디렉션 & 전략 싱크
- **퍼스널리티 슬라이더**: 이미지 조사 전, 클라이언트의 주관적 언어를 대립 축 매트릭스로 정량화. 5개 축 제안 → 업계 평균 위치(§1①에서 도출)와 우리가 튈 수 있는 경계선을 함께 표시.
  - 축 예시: `대중성 ─┼─ 희소성`, `진지함 ─┼─ 위트`, `전통적 ─┼─ 반항적`, `절제 ─┼─ 과잉`, `차가움 ─┼─ 따뜻함`
  - 각 축마다: 업계 평균 위치(점 하나) + Safe Route 위치(점) + Wild Card 위치(점), 3점을 한 축 위에 함께 표시해 "우리가 얼마나 튀는지"를 한눈에 보이게 한다.
  - 5축을 무작위로 고르지 않고, §1의 3중 리서치에서 실제로 갈린 지점(경쟁사가 몰려있는 축, 타깃 관심사가 바뀌고 있는 축)에서 뽑는다.
- **와일드카드 전략**: 위 §2의 두 루트 중 Wild Card 쪽을 최소 1개 준비. 이때 "왜 지금 이 타깃을 끌어와야 하는가"를 §1③ 근거로 뒷받침한다(감이 아니라 근거 기반 대담함).

### [2단계] 분석적 레퍼런스 분해
- **알고리즘 트랩 탈출**: 핀터레스트/인스타 알고리즘이 떠먹여주는 이미지를 무작위 수집(Referencing by Attraction) 금지. 검색어를 브랜드 키워드 그대로 넣지 않고, 의도적으로 다른 업종·다른 언어권 키워드로 우회 검색.
- **비주얼 데컴포지션**: 수집 이미지를 아래 4대 축으로 완전히 분해한 뒤 재조합한다. 이미지 하나를 통째로 "레퍼런스"라 부르지 않고, 그 이미지에서 실제로 가져올 요소만 축별로 추출한다.
  - **배색 조합** — 어떤 색이 몇 대 몇 비율로 쓰였는가
  - **화면 구도(Composition)** — 프레임 안 피사체 위치, 여백 비율, 시선 유도선
  - **질감 및 소재(Texture)** — 표면이 무엇으로 만들어진 것처럼 보이는가
  - **문화적 맥락(Cultural Reference)** — 이 이미지가 어떤 장르/시대/서브컬처를 연상시키는가(그대로 베끼면 안 되는 클리셰 지점 파악용)
  - 큐레이션 채널은 Are.na·Cosmos 등 표준 높은 곳 우선, 핀터레스트는 발견 단계로만 쓰고 최종 수록은 재촬영/재생성 이미지로 대체.

### [3단계] 내러티브 주입 & 비주얼 에디팅
- **그레이스케일 오버레이**: 컬러 충돌 방지용으로 이미지 위에 회색 레이어(블렌딩 Color/Multiply)를 얹어 모노톤 정제. 클라이언트 시선이 색상이 아니라 형태·레이아웃에만 집중해야 하는 초기 검토 단계(§5 1단계 직후)에서 특히 유효 — 컬러가 아직 미확정인데 특정 색 하나 때문에 전체 방향이 오해받는 것을 방지.
- **스토리텔링 실체화**: 클리셰를 넘어 브랜드 고유 은유를 비주얼 모티프로 치환, 로고타입·UX 핵심 터치포인트 디테일로 발전. "이 이미지가 왜 여기 있는지"를 브랜드 서사 언어로 재작성 — 이 결과물이 바로 §6의 3종 주석이 된다.

### [4단계] 프레젠테이션 & 승인
- **에디토리얼 가중치 배분**: 이미지 크기는 전략적 편집 — 핵심 비주얼을 가장 크게. 9칸 그리드 시 묵직한 텍스처는 모서리, 밝은 에셋/타깃 페르소나는 중앙(§7 그리드 공식 참조).
- **정밀 라벨링(Annotation)**: 모든 이미지 유닛에 A-1, B-2 식 라벨. "C-3의 타이포 무드 + A-2의 텍스처를 결합해달라"처럼 정밀 피드백이 가능하게. 라벨이 없으면 "왼쪽에서 세 번째 그 이미지요"식 모호한 피드백 루프가 생겨 회의 시간이 늘어난다.
- **비동기 워크스루**: 5~10분 내외 화면 녹화(또는 텍스트 내레이션)로 의도·당위성을 설명하는 자료를 함께 전달. 실시간 미팅 없이도 의사결정권자가 디자이너의 논리 구조를 온전히 흡수한 뒤 승인하도록 유도.
- **승인 이후 즉시**: 승인된 라우트만 §4의 택티컬 보드로 넘기고, 반려된 라우트는 삭제하지 않고 프로젝트 폴더에 "폐기·아카이브" 뱃지로 보관(기존 버전 관리 원칙과 동일하게 취급).

## 6. ★ 무드보드 이미지별 키워드·근거 주석 원칙 (필수)

무드보드에 들어가는 **모든 이미지**는 다음 3가지를 반드시 함께 표기한다. 이미지만 붙여넣고 끝내지 않는다.

1. **키워드** — 이 이미지에서 뽑은 핵심 키워드 1~3개
2. **선정 이유** — 왜 이 이미지를 선택했는가 (다른 후보 대신 이걸 고른 이유 — 반드시 "대안 대비" 서술로, "예쁘니까"류 무근거 사유 금지)
3. **담긴 이야기** — 이 이미지가 브랜드 서사의 어느 지점을 말하는가 (오프닝 §0의 관점/메시지와 직접 연결되는 문장이어야 함 — 연결이 안 되면 그 이미지는 무드보드에서 뺀다)

라벨 형식 예: `A-1 · 키워드: 표면장력/불신 · 선정이유: 크롬 반사에 왜곡된 형태가 "진짜인지 의심하게" 만드는 유일한 레퍼런스(비슷한 후보 3장 중 왜곡도가 가장 컸음) · 서사: Fast 라인의 "찰나에만 진실"을 시각적으로 증명 → 관점 슬라이드 1번 문장과 직결`

**셀프 체크 질문 (주석 완결성 확인용)**: 이 세 줄만 읽고 이미지를 못 봐도 "왜 이게 여기 있는지"가 이해되는가? 안 된다면 주석이 이미지 설명(what)에 머물러 있고 서사(why)로 못 올라간 것 — 다시 쓴다.

## 7. 시각적 균형 그리드 공식 (Visual Balance)

- **공식**: 9칸 그리드에서 **밝은 톤/디지털 에셋은 가운데 십자(중앙+상하좌우)** 에, **어둡고 묵직한 텍스처 에셋은 코너 4개 모서리**에 배치한다.
- **효과**: 사진(현실)과 그래픽(디지털) 레퍼런스를 교차 배치해 시선이 한 곳에 고이지 않고 보드 전체를 리드미컬하게 순환(Dance)하며 읽히게 한다.
- **적용 순서**: (1) 9장 중 톤 밝기 순으로 정렬 → (2) 가장 밝은 4~5장을 중앙 십자에 배정 → (3) 가장 어두운 4장을 모서리에 배정 → (4) 남는 자리는 채도·질감 대비가 가장 큰 조합으로 채운다.
- **6칸/12칸 변형**: 9칸이 아니라 6칸(6버킷 1:1 매핑) 또는 12칸(라인이 여러 개일 때)으로 갈 경우도 같은 원칙 적용 — 중앙 열은 밝은 톤, 바깥 열은 어두운 톤으로 그라데이션을 만든다.
- **실패 신호**: 완성된 그리드를 눈을 가늘게 뜨고 봤을 때 한쪽에만 어둡거나 밝은 덩어리가 뭉쳐 보이면 재배치. "덩어리짐"이 균형 공식이 깨졌다는 가장 빠른 육안 신호다.

## 8. 디렉터 핵심 원칙

1. **10초 원칙** — 완성도 높은 무드보드는 말로 설명하기 전에 클라이언트가 보드를 보고 10초 안에 최종 결과물을 머릿속에 그릴 수 있어야 한다. 검증법: 무드보드만 먼저 5초 보여주고 "이게 뭘 위한 보드일 것 같냐"고 물었을 때 카테고리(제품군)와 톤(고급/캐주얼 등)을 맞히면 통과.
2. **R&R 분리** — 아트 디렉터의 보드(바이브 보드)는 360° 세계관·감정을 홀딩하고, 디자이너의 보드(택티컬 보드)는 이를 뒷받침하는 구체적 시스템·증거를 줌인한다. 두 경계가 섞이면 프로젝트의 명확성이 흐려진다.
3. **근거 없는 아름다움 배제** — §6의 주석 원칙과 연동: "예쁘다/느낌이 좋다"만으로 이미지를 넣지 않는다. 모든 이미지는 §0 오프닝 결론에 대한 증거로 기능해야 한다.
4. **두 루트는 끝까지 대칭으로** — §2의 Safe/Wild Card는 한쪽만 디테일하게 다듬고 다른 쪽을 성의 없이 채우지 않는다. 클라이언트가 "이건 안 판 것 같다"고 느끼면 신뢰가 깨진다.

## 9. 앞단 변경 전파 원칙 — "빈틈 없이 유연하게 이어가기"

- 중간에 사용자가 새 아이디어를 던지면(예: 폼·소재 확장 브레인스토밍, 새 레퍼런스 이미지 투척), 그 아이디어가 **오프닝 3~4장(관점/정의/메시지)이나 퍼스널리티 슬라이더, 혹은 Safe/Wild Card 두 루트 중 어느 쪽에 영향을 주는지 먼저 판단**한다. 판단 기준:
  1. 이 아이디어가 §0의 "관점 문장" 자체를 바꾸는가, 아니면 그 관점 아래 "표현 방식"만 바꾸는가 (전자면 Gate 재확인 필수)
  2. 이 아이디어가 퍼스널리티 슬라이더의 축 위치를 이동시키는가 (예: "원목"은 `차가움↔따뜻함` 축을 따뜻함 쪽으로 이동)
  3. 이 아이디어가 Safe Route/Wild Card Route 중 어디에 속하는지, 혹은 완전히 새로운 3번째 후보인지
- 영향을 준다면: 영향받는 앞단 슬라이드를 구체적으로 짚어("이 아이디어가 반영되면 관점 슬라이드의 O번 문장, Safe Route의 P번 이미지 톤이 바뀌어야 함") 사용자에게 확인 후 갱신한다. 조용히 뒷단만 고치고 앞단은 방치하지 않는다.
- 영향이 없다고 판단되면(예: 단순히 6버킷 중 Setting 카테고리에 이미지 1장 추가) 바로 반영하고 별도 확인 없이 진행 — 모든 사소한 추가까지 게이트로 막으면 발산 속도가 죽는다. 게이트는 §0/§1축/§2루트급 변화에만 건다.
- `00-project-index.md`의 "확정 사항 로그"에 이 전파 사실을 기록한다 — 연속성 관리 원칙과 동일하게 취급. 기록 형식: `날짜 | 새 아이디어 | 영향받은 앞단 요소 | 처리(반영/보류/기각)`
- 이 판단·전파 작업은 `project-continuity-auditor`가 사후 점검하되, 실시간 반영은 메인 세션이 그 자리에서 수행한다(감사는 사후 백업, 즉시 처리가 우선).
- **삽입 전 스크린샷 사전 점검(CLAUDE.md 원칙과 동일)**: 새 아이디어를 어디에 넣을지 텍스트로만 판단하지 않는다. 먼저 현재 덱을 스크린샷으로 실제로 보고 전체 흐름·틀이 지금 어떤 상태인지 확인한 뒤에만 삽입 위치를 정한다 — 입력받은 순서 그대로 밀어넣으면 흐름이 깨진다.

## 10. 무드보드 전용 조직도 — 이 영역만 분리 실행 시

무드보드만 따로 떼서 진행해도(전체 Manus/Runable 파이프라인 없이) 아래 단계·담당이 유지되도록 조직도를 고정한다.

| 단계 | 산출물 | 담당 에이전트 | 사용자 확인 필요? | 실패 시 되돌아갈 단계 |
|---|---|---|---|---|
| 0. 오프닝 3~4장 초안 | 관점/결과물정의/메시지 | `concept-director` | ✅ Gate — 반드시 확인 | — (최초 단계) |
| 1. 퍼스널리티 슬라이더 5축 + 키워드 마인드맵 | 대립축 매트릭스, 키워드 | `brainstormer` | ✅ 슬라이더 위치 확인 | 0단계(관점이 안 맞으면 축 자체가 흔들림) |
| 2. 3중 리서치(경쟁/트렌드/타깃관심사) | 근거+출처 표 | `market-research` + `design-trend-radar` | 근거 부족 시에만 확인 | 1단계(근거가 약하면 축 재조정) |
| 3. Safe/Wild Card 2루트 발산 | 각 루트별 6버킷 이미지 셋 | `brainstormer` → `moodboard-builder`(이미지 조합) | ✅ 두 루트 비교 확인 | 2단계(근거 부족한 루트는 리서치 보강) |
| 4. 이미지별 키워드·근거 주석 | A-1식 라벨 + 3종 주석 | `moodboard-builder` | 주석 누락 여부만 셀프체크 | 3단계(주석이 안 써지면 그 이미지가 애초에 부적합) |
| 5. 그리드 배치(9칸, 균형 공식) | HTML 무드보드 | `moodboard-builder`(메인 세션이 Higgsfield 이미지 생성) | 배치 완료 후 확인 | 4단계 |
| 6. 실현가능성 검증(선택) | 리스크 점수 | `critic`(Wild Card Route 대상) | 고위험 판정 시 확인 | 3단계(대체 이미지로 보완) |
| 7. 앞단 전파 점검 | 변경 영향 요약 | `project-continuity-auditor`(사후) + 메인 세션(즉시) | 영향 있을 시에만 | 해당 단계로 즉시 |

이 조직도는 브랜드 파이프라인(Manus/Runable)에 종속되지 않는다 — 무드보드만 단독으로 의뢰받은 프로젝트(예: 시안 리프레시, 톤 재정의)에도 그대로 적용한다. 단, 단독 의뢰일 경우 0단계의 "관점"은 새로 만들지 않고 기존 브랜드 가이드에서 발췌·요약해 대체 가능.

## 11. 완결성 체크리스트 (제출 전 최종 점검)

- [ ] 오프닝 3~4장이 리서치보다 먼저 배치되어 있는가
- [ ] 3중 리서치(경쟁/트렌드/타깃관심사) 모두 출처가 달려 있는가, "가설(미검증)" 항목은 별도 표기되어 있는가
- [ ] Safe Route와 Wild Card Route가 대칭 포맷·대칭 디테일 수준으로 둘 다 존재하는가
- [ ] 6버킷 중 빠진 카테고리는 없는가(있다면 의도적 생략인지 확인)
- [ ] 무드보드의 모든 이미지에 키워드/선정이유/담긴 이야기 3종 주석이 있는가
- [ ] 9칸(또는 변형) 그리드가 균형 공식(밝은톤 중앙/어두운톤 모서리)을 따르는가
- [ ] 라벨링(A-1식)이 전체 이미지에 일관되게 붙어 있는가
