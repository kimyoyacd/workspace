# UI Copy Corpus — UI 마이크로카피 기출 저장소

광고 카피와 **성격이 다른** UI 카피(마이크로카피) 전용 코퍼스.
광고 코퍼스(`../copy-corpus/`)와 분리해서 관리한다 — 섞으면 검색이 오염된다.

## 광고 카피 vs UI 카피 (기준이 다르다)

| | 광고 카피 (`copy-corpus`) | UI 카피 (여기) |
|---|---|---|
| 목적 | 주목·설득·감정 | 안내·행동·마찰 제거 |
| 기준 | **누르고 싶은가** | **헷갈리지 않고 바로 다음 행동을 하는가** |
| 분류축 | 업종 | 컴포넌트·상황 |
| 톤 | 감성·위트·도발·프리미엄 | 명확·친근·안심·격려·단호 |

UI 카피의 좋고 나쁨은 "예쁜가"가 아니라 **명확한가 / 실수해도 안심되는가 / 다음에 뭘 할지 아는가**로 판단한다.

## 작동 원리 (광고 코퍼스와 동일한 RAG)

컴포넌트·상황 키워드로 grep해서 기출을 꺼내고, 그 패턴 위에서 새 UI 카피를 쓴다.
```bash
# 팀 공용 헬퍼 (UI 모드)
.claude/skills/copywriter/search.sh --ui "빈 상태 장바구니" 20
.claude/skills/copywriter/search.sh --ui "에러 결제 실패"
.claude/skills/copywriter/search.sh --ui "가챠 확정" --category game

# raw grep 폴백
grep -h -iE "빈상태|장바구니|비어" .claude/library/ui-copy-corpus/*.jsonl
```

## 파일 구성 (컴포넌트/상황별 JSONL)

| 파일 | 커버 범위 |
|---|---|
| `buttons-cta.jsonl` | 버튼·CTA 라벨, 액션 링크, 탭 |
| `states-errors.jsonl` | 빈 상태, 로딩, 에러·경고, 입력 검증 |
| `feedback-dialogs.jsonl` | 성공/완료 토스트, 확인 다이얼로그, 권한 요청 |
| `onboarding-forms.jsonl` | 온보딩·툴팁·코치마크, 폼 라벨·힌트·플레이스홀더 |
| `game-ui.jsonl` | 게임 UI 특화 — 상점·재화·가챠·결과·미션·이탈방지·복귀 |

## 스키마 (한 줄 = UI 카피 한 건)

```json
{"copy":"저장","component":"버튼","context":"편집 완료 후 변경사항 저장","keywords":["저장","버튼","완료","편집"],"tone":"명확","principle":"동사로 결과를 약속","platform":"공통","source":"crafted","why":"무엇을 하는 버튼인지 한 단어로 분명해 망설임이 없다"}
```

| 필드 | 설명 |
|---|---|
| `copy` | UI 문구 본문 (필수) |
| `component` | 버튼 / 빈상태 / 로딩 / 에러 / 검증 / 토스트 / 다이얼로그 / 권한 / 온보딩 / 툴팁 / 폼 / 게임-상점 등 |
| `context` | **언제 어디서 뜨는가** — UI 카피는 맥락이 전부다 (필수) |
| `keywords` | 검색용. 컴포넌트어 + 상황어 + 화면어를 섞어 5개 내외 |
| `tone` | 명확 / 친근 / 안심 / 격려 / 단호 / 위트 중 택1 (UI 톤) |
| `principle` | 이 카피가 지키는 UI 원칙 한 줄 (아래 참조) |
| `platform` | 공통 / 앱 / 웹 / 게임 |
| `source` | `verified`(실제 프로덕트) / `recalled`(기억 기반 실사례) / `crafted`(원칙 기반 제작) |
| `why` | 왜 좋은 UI 카피인가 — 기준은 "누르고 싶은"이 아니라 "헷갈림 없이 다음 행동으로" (필수) |

## UI 카피 7원칙 (`principle`·`why`의 기준)

1. **명확성** — 다음에 뭘 할지 즉시 안다. 모호어("확인") 대신 결과어("삭제")
2. **행동 유도** — 버튼은 동사로, 무엇이 일어날지 약속한다
3. **마찰 제거** — 없어도 되는 단어는 뺀다. 짧을수록 좋다
4. **비난 금지** — 에러를 사용자 탓으로 돌리지 않는다 ("잘못 입력했습니다" ✗ → "이메일 형식을 확인해 주세요" ○)
5. **결과 명시** — 되돌릴 수 없는 행동은 결과를 분명히 경고한다
6. **안심 제공** — 실수·실패 상황에서 다음 출구를 준다 (재시도·복구·문의)
7. **톤 일관** — 제품/게임 세계관의 말투를 유지한다 (특히 게임은 컨셉 말투)

## 수집·성장 규칙

- 광고 코퍼스와 **절대 섞지 않는다**. UI 문구는 여기, 광고 문구는 `../copy-corpus/`
- `context` 없이 추가하지 않는다 — 같은 단어도 맥락이 다르면 다른 카피다
- `keywords`에 컴포넌트어(버튼, 빈상태)와 상황어(결제실패, 재고없음, 첫방문)를 함께 넣는다
- 좋은 프로덕트에서 만난 UI 문구는 `source:"recalled"`로, 우리 실 작업물은 `verified`로 축적
- 사용은 `.claude/skills/copywriter/SKILL.md`의 "UI 카피 모드" 참조
