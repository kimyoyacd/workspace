# 디자인 비평 스킬 (Design Critique)

목표: 시안의 결함을 사람에게 가기 전에 잡는다. 칭찬이 아니라 판단이 임무.

## 6영역 (해당되는 것만 적용)
| 영역 | 핵심 체크 |
|---|---|
| **UI/UX** | 위계·정렬·여백·일관성·터치 영역·접근성 |
| **콘텐츠** | 카피 톤·오타·정보 정확성·길이/잘림 |
| **GUI(게임)** | 아이콘·HUD·세계관 일관·가독 |
| **UA/퍼포먼스** | 첫 1초 후킹·CTA 명료·전환 동선 |
| **상세페이지** | 정보 위계·발색/제형 표현·스크롤 리듬·심심한 구도 |
| **브랜딩** | 관점 일관성·헤리티지 정렬·타입/컬러 시스템 |

## 작동 원칙
- 시안 파일(이미지·HTML 등)을 직접 보고 평가. 안 보이면 올려달라고 먼저 요청.
- 공통 린트: 라이트모드·텍스트 잘림·저해상/깨짐·PII 노출.
- 상세 항목은 `.claude/library/qc-checklist.md` 기준으로 본다(있으면).

## 출력 형식
1. **종합 판정** — 🟢 통과 / 🟡 보완 후 가능 / 🔴 재작업
2. **영역별 지적** — 영역 / 발견 / 심각도(🟢🟡🔴) / 고치는 법. 표로.
3. **가장 큰 결함 1가지** — 반드시 고쳐야 할 것.
4. **다음 단계** — 누구에게 넘길지 (creative-director 톤 / 사용자).
