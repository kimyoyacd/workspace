# Skill Creator — 2차 · 비주얼 아이덴티티 + 컨셉/무드보드

> 1차에서 정의한 문제·포지셔닝을 **비주얼 전략으로 번역**하는 단계.
> 하나로 좁히기 전에 **서로 다른 결의 컨셉을 여러 개 펼쳐** 비교·선택한다.

## 언제 쓰나
- 1차 기획서/포지셔닝 다음, 비주얼 방향을 잡을 때
- 컨셉 디렉션(무드보드)을 여러 종 발산해 사용자가 고르게 할 때
- 브랜드 비주얼 아이덴티티(컬러·타이포·톤·키워드) 초안

## 프로세스
1. **Brand Visual Identity Extraction** — 1차 결과물에서 핵심 가치·키워드·톤을 추출해 비주얼 언어로 변환 (컬러·타이포·소재·무드 키워드)
2. **컨셉 디렉션 N종 발산** — 결이 다른 컨셉을 3~6종 생성. 각 컨셉 = 이름 + 한 줄 정의 + 무드 키워드 + 레퍼런스 그리드 + 컬러/타이포 방향
   → 디렉션 팔레트(이름·성격): **`concept-directions.md`** (이 폴더)
3. **Client × Mood matrix** — 컨셉을 클라 성향 × 무드 축으로 매핑해 적합도 비교
4. **셀렉 → 정련** — 사용자가 1~2개 선택 → 해당 컨셉을 디벨롭

## 산출물
- 컨셉별 무드보드 (HTML 1장, 라이트모드 3종) — 컬러 팔레트 + 이미지 그리드 + 키워드 + 컨셉 설명
- Client×Mood 매트릭스

## 연계
- 발산 엔진: `brainstormer`(6관점 발산)
- 무드보드 조립: `moodboard-builder`(레퍼런스+생성)
- 레퍼런스: `reference-curator` · 이미지: `visual-generator`
- 앞단계 1차 `skill-creator-plan` → 뒷단계 3차 `skill-creator-system`

## 주의
- 컨셉은 "예쁨"이 아니라 **1차 포지셔닝 문제를 푸는 비주얼 전략**이어야 한다 (각 컨셉이 어떤 문제를 푸는지 한 줄 명시).
- 레퍼런스 직접 모사 금지. 무드보드 이미지엔 출처 표기.
