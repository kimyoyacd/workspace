# 브랜드 가이드라인 덱 제작

디자인 외주 실장이 클라이언트 프로젝트마다 반복 제작하는 **한글 브랜드 가이드라인 문서**를 표준 구조로 빠르게 만들기 위한 스킬. TEMS Intelligence 프로젝트(`.claude/projects/202607_TEMS브랜딩_가이드_v1.html`)에서 처음 완성했고, 이후 모든 브랜딩 프로젝트에 재사용한다.

## 언제 쓰나
- 로고·컬러·타이포 등 BI 요소가 **확정**된 뒤, 클라이언트에게 전달할 정식 가이드 문서가 필요할 때
- 참고할 피그마 브랜드 가이드 템플릿 링크가 있을 때(구조 참고용 — 내용을 그대로 베끼지 않는다)
- 심볼이 아직 미확정이면 이 스킬을 실행하지 말고, 먼저 심볼부터 확정할 것 (명분 없는 가이드 문서는 무의미)

## 선행 조건 (반드시 확인)
1. **브랜드 전략이 확정돼 있는가?** 최소한 아래가 있어야 한다 — 없으면 먼저 만들라고 사용자에게 요청:
   - 브랜드명 / 하는 일 / 타깃 / 산업군 / 브랜드 인상
   - 미션 · 비전 · 포지셔닝 스테이트먼트
   - 브랜드 핵심가치(Pillars) 5개 — **각각 제목+1~2문장 설명**. "키워드+한줄 태그라인" 수준은 부족하다는 피드백을 받은 적이 있으니 반드시 문단 단위로 작성.
   - 브랜드 퍼스낼리티(형용사 8~10개)
   - 톤앤보이스 Do/Don't
2. **심볼/로고가 확정돼 있는가?** 벡터(SVG) 형태로 있어야 한다. 래스터 이미지만 있으면 트레이싱부터 하고, 트레이싱 결과물엔 반드시 "래스터 기반 추정 — 원본 벡터 수신 시 교체" 주석을 남긴다. 확정 안 된 상태로 가이드를 만들지 말 것 — 심볼이 나중에 바뀌면 문서 전체를 다시 만들어야 한다.
3. **참고 피그마 템플릿이 있으면** `mcp__Figma__get_metadata`로 페이지 목록만 훑어 섹션 순서·구성을 파악한다. 이미지/텍스트 내용을 그대로 복사하지 않는다 — 구조(어떤 종류의 페이지가 몇 개, 어떤 순서인지)만 참고.

## 표준 섹션 구조 (18페이지 기준, 프로젝트 규모에 따라 가감)
1. **Cover** — 심볼 + "Brand Identity Guidelines" + 버전·날짜
2. **Welcome** — 이 문서의 목적을 한 문단으로
3. **Table of Contents** — 4대 섹션(Strategy/Logo/Color&Type/Applications) 요약
4. **Section Cover: Brand Strategy**
5. Mission · Vision · Positioning
6. Brand Pillars (5개, 각 제목+설명)
7. Brand Personality (형용사 8~10개) + Tone of Voice (Do/Don't)
8. **Section Cover: Logo**
9. Logo Construction — 심볼이 왜 이 형태인지 논리 (브랜드 핵심가치와 반드시 연결지어 설명)
10. Clear Space & Lockup — 가로/세로 락업 + 클리어스페이스 그리드(점선 박스, x단위 표기) + 락업 간격
11. Minimum Size — 인쇄/디지털 최소 크기, 파비콘 전용 락업 안내
12. Logo Color Variations — 라이트/다크 배경, 그라데이션 배경 예외 규칙
13. **Section Cover: Color & Typography**
14. Color System — 로고용 모노크롬 + (있다면) 브랜드 월드용 확장 팔레트, 역할을 명확히 분리해서 설명
15. Typography — 워드마크 + 본문 서체, 커스텀 서체 여부 명시
16. **Section Cover: Applications**
17. 명함 등 실제 응용물 (앞/뒤, 라이트/그라데이션 버전 등)
18. **Next Steps** — 미확정 항목, 다음 버전에서 보완할 것

섹션 4/8/13/16처럼 색이 있는 "Section Cover" 페이지는 그라데이션이나 브랜드 다크 컬러의 풀블리드 배경 + 큰 타이틀 하나만 놓는다(내용 없음, 챕터 구분용).

## 페이지 템플릿 (재사용 CSS 패턴)
모든 페이지는 16:9 비율 카드, 흰 배경(또는 다크/그라데이션 섹션 커버), 공통 헤더/푸터:
```
헤더: [브랜드명 볼드] [섹션명 회색]  ........................  [페이지번호]
      ─────────────────────────────────────── (얇은 구분선)
본문: (섹션별 콘텐츠)
푸터: [브랜드명 · 섹션명]  ........................  [페이지번호]
      (얇은 구분선 위)
```
`aspect-ratio:16/9`인 `.page` div를 세로로 나열해 스크롤 덱으로 보여준다(실제 인쇄/PDF 변환은 다음 단계). 다크 섹션 커버는 `.page.dark`(잉크 단색) 또는 `.page.gradbg`(브랜드 그라데이션 방사형) 클래스로 구분.

작업 시 TEMS 프로젝트의 `_가이드_v1.html`을 뼈대로 복제한 뒤 브랜드명·색상·심볼 path·전략 텍스트만 치환하는 방식을 권장 — CSS 변수(`--ink`, `--gray*`, 브랜드 액센트 컬러)만 바꾸면 새 브랜드에 맞는 톤으로 전환된다.

## 산출 후 처리
1. Playwright로 렌더링해 실제 스크린샷으로 레이아웃 깨짐(텍스트 겹침, 심볼 오배치 등)을 반드시 확인한다 — 코드만 보고 완성됐다고 판단하지 말 것.
2. Artifact로 발행해 사용자가 바로 볼 수 있게 한다.
3. 프로젝트의 버전 정책을 따른다 — 기존 파일을 덮어쓰지 않고 `_가이드_v1.html` → `_가이드_v2.html` 순으로 새 파일 저장(CLAUDE.md 및 프로젝트 노트에 버전 정책이 명시돼 있으면 그것을 따른다).
4. 프로젝트 마스터 노트(`.claude/projects/<프로젝트>.md`)에 가이드 문서 제작 사실과 버전을 기록한다.
