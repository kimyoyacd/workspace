# 디자인 트렌드 스킬 (Design Trend Radar)

목표: 최근 3개월 이내 글로벌 디자인 흐름을 출처와 함께 추려, 제안에 바로 쓸 수 있게 만든다.
트렌드 스캔 → 프로젝트 필터링 → **Direction A·B·C 3방향** 순서로 진행한다.
이 3방향이 `moodboard` 스킬의 입력이 된다.

## 핵심 원칙
- **최근성 강제** — 원칙적으로 최근 3개월 이내. 오래된 건 "기준 시점" 명시.
- **출처 필수** — 모든 트렌드·사례에 매체명 + URL. 없으면 "미확인" 표기.
- **상업 블로그 경계** — 판촉 블로그보다 1차 에이전시 사례(COLLINS·Wolff Olins·JKR·W+K)에 비중.
- **우리 적용까지** — 트렌드 나열 후 반드시 Direction 3방향으로 수렴한다.

## 6영역 필터 (요청에 맞춰 선택)
UI/UX · 콘텐츠 · GUI(게임) · UA/퍼포먼스 · 상세페이지 · 브랜딩

## 실행 순서

### Step 1. 트렌드 스캔 (기존 출력 유지)
- 조사 범위: 영역 / 기준 시점 / 참고 매체
- 글로벌 트렌드 5~8개 추출 (한 줄 정의 + 대표 사례 + 출처 URL)
- "한물간 흐름" 1~2개 명시
- 우리 적용 시사점 2~3개 (실장님 실무에 바로 쓸 포인트)

### Step 2. 프로젝트 필터링
- 해당 프로젝트의 카테고리·타겟·채널 기준으로 트렌드 적합도 판단
- 적합한 트렌드를 3개 방향으로 군집화

### Step 3. Direction 3방향 정의 (필수 출력)

각 Direction마다 아래 항목을 반드시 포함:

```
Direction [A/B/C] · [배지: RECOMMENDED / HIGH RISK / LOW PRIORITY]
방향명: (감각적 한 줄 타이틀 — 예: "오전 빛 · 선택의 순간")
설명: (2~3줄. 어떤 감성·무드인가)
컬러 팔레트: 6색 (이름 + HEX)
키워드: 6~8개
레퍼런스 브랜드: 3개 (브랜드명 + 왜 참고하는지 한 줄)
리스크: (채널·타겟·경쟁 환경 기준 리스크 판단. 없으면 "낮음")
```

**배지 기준:**
- **RECOMMENDED** — 타겟·채널 적합도 높고, 경쟁 대비 차별화 가능
- **HIGH RISK** — 임팩트 크지만 채널 환경·클라이언트 맥락에서 충돌 위험
- **LOW PRIORITY** — 완성도 있으나 현 채널에서 임팩트 부족. 자사몰·리뉴얼 시 재검토 권장
