# Copywriter — 기출문제 기반 카피라이팅

문법책이 아니라 좋은 글을 읽고 따라 쓰며 배우는 방식.
카피를 쓰기 전에 반드시 코퍼스에서 관련 기출을 검색하고, 그 패턴 위에서 쓴다.
**맨땅에서 쓰지 않는다 — 검색 없이 쓴 카피는 이 스킬의 산출물이 아니다.**

## 프로세스 (5단계)

### 1단계 — 브리프 확인
빠진 게 있으면 되묻지 말고 합리적으로 가정하고 가정을 명시한다.
- 제품/서비스는 무엇인가
- 매체는 무엇인가 (배너 / 앱푸시 / 헤드라인 / 슬로건 / 상세페이지 / 영상 엔드카드)
- 타겟은 누구인가
- 원하는 행동은 무엇인가 (클릭 / 가입 / 구매 / 사전예약 / 복귀)

### 2단계 — 키워드 확장 후 기출 검색 (RAG)
요청 단어를 그대로 검색하지 말고 3층으로 확장한다:
- **제품어**: 인터넷, 요금제
- **정서어**: 답답함, 아깝다, 눈치
- **상황어**: 이사, 자취, 버퍼링, 위약금

그다음 코퍼스를 검색한다:
```bash
# 1차: 전체 코퍼스에서 확장 키워드로
grep -h -iE "인터넷|요금제|속도|버퍼링|이사|위약금" .claude/library/copy-corpus/*.jsonl | head -40

# 2차: 해당 카테고리 파일 전체 훑기 (카테고리가 명확할 때)
head -80 .claude/library/copy-corpus/telecom-internet.jsonl

# 3차: 매체가 요구하는 톤으로 보강 (예: 푸시 → 긴급/위트)
grep -h '"tone":"긴급"' .claude/library/copy-corpus/*.jsonl | head -15
```
- 15~30건을 확보할 때까지 키워드를 넓힌다. 카테고리가 달라도 구조가 같으면 좋은 기출이다 (금융의 손실회피 문법은 통신 위약금 카피에 그대로 쓰인다)
- 3건 미만이면 인접 카테고리·톤 검색으로 보완하고, 작업 후 이번에 쓴 카피를 코퍼스에 추가해 빈 곳을 메운다

### 3단계 — 기출 패턴 분석
꺼낸 기출에서 뽑아낼 것:
- **반복되는 구조**: 재정의형(A가 아니라 B다), 질문형, 숫자형, 손실회피형, 대구형, 반전형 중 이 주제에서 뭐가 자주 이기는가
- **어휘의 온도**: 존댓말/반말, 구어체/문어체, 어미 처리
- **why 필드**: 각 기출이 "왜 눌리는지"의 공통분모

### 4단계 — 생성
- 분석한 패턴으로 **서로 다른 구조 4~6안**을 쓴다 (같은 구조의 어미 변형은 1안으로 침)
- 각 안에 근거 기출을 1줄로 표기한다: `← 기출: "국물이 끝내줘요" (제품 핵심을 구어체 한 마디로)`
- 매체 제약 준수: 배너 15자 내외, 앱푸시 제목 20자·본문 40자, 슬로건 10자 내외 권장
- 금지: "지금 바로 만나보세요", "특별한 혜택", "놓치지 마세요" 단독 사용 등 아무 데나 붙는 범용 문장

### 5단계 — 셀프 게이트
- 각 안에 스스로 묻는다: **"내가 이걸 누르나?"** — 아니면 버리고 다시
- `anti-ai-writing` 스킬 기준으로 최종 점검 (매끄럽지만 알맹이 없는 문장 제거)
- 사실 주장(1위, 최저가, N만 명)이 들어간 안은 반드시 "클라 확인 필요" 플래그를 붙인다

## 출력 형식

```
## 기출 검색 결과
- 검색어: …
- 참고 기출 N건 (대표 3건 인용)
- 발견한 패턴: …

## 카피 제안 (N안)
1. "카피" — 구조 · ← 기출 근거
2. …

## 추천 1안 + 이유 (누르고 싶은 이유 기준)
```

## 코퍼스 성장 (작업 후 습관)

- 이번 작업에서 채택된 카피는 해당 카테고리 JSONL에 append한다 (`source`: `"crafted"`, `brand`: 클라이언트명)
- 검색이 빈약했던 키워드는 기존 기출의 `keywords`를 보강하거나 새 기출을 추가한다
