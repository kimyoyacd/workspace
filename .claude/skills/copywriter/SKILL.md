# Copywriter — 기출문제 기반 카피라이팅

문법책이 아니라 좋은 글을 읽고 따라 쓰며 배우는 방식.
카피를 쓰기 전에 반드시 코퍼스에서 관련 기출을 검색하고, 그 패턴 위에서 쓴다.
**맨땅에서 쓰지 않는다 — 검색 없이 쓴 카피는 이 스킬의 산출물이 아니다.**

## 0단계 — 모드 판별 (광고 카피 vs UI 카피)

**광고 카피와 UI 카피는 성격이 다르다. 먼저 어느 쪽인지 판별하고 다른 코퍼스·다른 기준을 쓴다.**

| | 광고 카피 (기본) | UI 카피 (`--ui`) |
|---|---|---|
| 코퍼스 | `.claude/library/copy-corpus/` | `.claude/library/ui-copy-corpus/` |
| 기준 | **누르고 싶은가** | **헷갈리지 않고 바로 다음 행동을 하는가** |
| 매체 | 배너·헤드라인·슬로건·앱푸시·사전예약·상세페이지 | 버튼·빈상태·에러·검증·토스트·다이얼로그·권한·온보딩·툴팁·폼·게임UI |
| 톤 | 감성·위트·도발·프리미엄·공감·긴급 | 명확·친근·안심·격려·단호·위트 |
| 분류축 | 업종 | 컴포넌트·상황 |

판별 신호:
- **광고**: "배너/슬로건/헤드라인/캠페인/후킹/사고 싶게/브랜드 카피"
- **UI**: "버튼/라벨/에러/빈 화면/토스트/팝업/온보딩/툴팁/권한/폼/상점·가챠·결과 화면/마이크로카피"
- 앱푸시는 목적에 따라 갈린다: 마케팅 유인(할인·복귀 유혹) → 광고 / 상태 알림(배송·처리 완료) → UI
- 애매하면 사용자에게 한 줄로 묻는다: "광고 카피인가요, UI 문구인가요?"

아래 5단계는 두 모드 공통이다. 단 UI 모드에서는 **검색·기준·출력이 UI 규칙을 따른다**(각 단계의 〔UI〕 표시 참조).

## 프로세스 (5단계)

### 1단계 — 브리프 확인
빠진 게 있으면 되묻지 말고 합리적으로 가정하고 가정을 명시한다.
- 제품/서비스는 무엇인가
- 매체는 무엇인가 (배너 / 앱푸시 / 헤드라인 / 슬로건 / 상세페이지 / 영상 엔드카드)
- 타겟은 누구인가
- 원하는 행동은 무엇인가 (클릭 / 가입 / 구매 / 사전예약 / 복귀)

### 2단계 — 키워드 확장 후 기출 검색 (RAG)
요청 단어를 그대로 검색하지 말고 3층으로 확장한다:
- **제품어**: 인터넷, 요금제
- **정서어**: 답답함, 아깝다, 눈치
- **상황어**: 이사, 자취, 버퍼링, 위약금

그다음 코퍼스를 검색한다. **팀 공용 검색 헬퍼를 우선 사용**한다 — 팀원 누구나 같은 방식으로 기출을 꺼내 검색 편차를 없앤다:
```bash
# 팀 공용 헬퍼 (권장) — 카피 · 톤 · 구조 · why를 보기 좋게 출력
.claude/skills/copywriter/search.sh "인터넷 요금제 속도 버퍼링 이사 위약금" 30
.claude/skills/copywriter/search.sh "야식 배달" --tone 위트        # 톤으로 좁히기
.claude/skills/copywriter/search.sh "무제한" --category 통신        # 카테고리로 좁히기

# 헬퍼가 없는 환경이면 raw grep 폴백
grep -h -iE "인터넷|요금제|속도|버퍼링|이사|위약금" .claude/library/copy-corpus/*.jsonl | head -40
```
- 15~30건을 확보할 때까지 키워드를 넓힌다. 카테고리가 달라도 구조가 같으면 좋은 기출이다 (금융의 손실회피 문법은 통신 위약금 카피에 그대로 쓰인다)
- 3건 미만이면 인접 카테고리·톤 검색으로 보완하고, 작업 후 이번에 쓴 카피를 코퍼스에 추가해 빈 곳을 메운다

〔UI〕 UI 모드는 `--ui` 플래그로 UI 코퍼스를 검색한다. 키워드는 **컴포넌트어 + 상황어**로 확장한다(제품어·정서어가 아님):
```bash
# UI 카피 검색 — 컴포넌트어(버튼·빈상태·에러) + 상황어(결제실패·재고없음·첫방문)
.claude/skills/copywriter/search.sh --ui "빈상태 장바구니 비어 없음" 20
.claude/skills/copywriter/search.sh --ui "에러 결제 실패 카드"
.claude/skills/copywriter/search.sh --ui "가챠 확정 천장 부족" --category game   # 게임 UI만
```

### 3단계 — 기출 패턴 분석
꺼낸 기출에서 뽑아낼 것:
- **반복되는 구조**: 재정의형(A가 아니라 B다), 질문형, 숫자형, 손실회피형, 대구형, 반전형 중 이 주제에서 뭐가 자주 이기는가
- **어휘의 온도**: 존댓말/반말, 구어체/문어체, 어미 처리
- **why 필드**: 각 기출이 "왜 눌리는지"의 공통분모

〔UI〕 UI 모드는 구조가 아니라 **원칙**을 뽑는다: 같은 컴포넌트·상황의 기출에서 `principle`·`why`의 공통분모를 찾는다 (예: 에러는 "비난 금지 + 고칠 곳 지목", 파괴적 다이얼로그는 "비가역성 경고"). UI 7원칙은 `.claude/library/ui-copy-corpus/README.md` 참조.

### 4단계 — 생성
- 분석한 패턴으로 **서로 다른 구조 4~6안**을 쓴다 (같은 구조의 어미 변형은 1안으로 침)
- 각 안에 근거 기출을 1줄로 표기한다: `← 기출: "국물이 끝내줘요" (제품 핵심을 구어체 한 마디로)`
- 매체 제약 준수: 배너 15자 내외, 앱푸시 제목 20자·본문 40자, 슬로건 10자 내외 권장
- 금지: "지금 바로 만나보세요", "특별한 혜택", "놓치지 마세요" 단독 사용 등 아무 데나 붙는 범용 문장

〔UI〕 UI 모드의 생성 규칙:
- 버튼·라벨은 **최대한 짧게**(버튼 2~4자, 다이얼로그 본문 1~2문장). 안 눌러도 되는 단어는 뺀다
- 모호어("확인") 대신 **결과어**("삭제")를 쓴다. 버튼은 동사로
- 에러·검증은 **사용자 탓 금지** + 고칠 곳/다음 수를 준다. 파괴적 행동은 결과를 명시
- 제품/게임의 **세계관 말투**를 유지한다 (게임은 특히)
- 안(案)은 톤 스펙트럼으로 낸다: 중립·명확 / 친근 / (게임·캐주얼이면) 위트 버전

### 5단계 — 셀프 게이트
- 〔광고〕 각 안에 스스로 묻는다: **"내가 이걸 누르나?"** — 아니면 버리고 다시
- 〔UI〕 각 안에 스스로 묻는다: **"헷갈리지 않고 바로 다음 행동을 할 수 있나? 실수해도 안심되나?"** — 아니면 버리고 다시
- `anti-ai-writing` 스킬 기준으로 최종 점검 (매끄럽지만 알맹이 없는 문장 제거)
- 사실 주장(1위, 최저가, N만 명)이 들어간 안은 반드시 "클라 확인 필요" 플래그를 붙인다

## 출력 형식

```
## 기출 검색 결과
- 모드: 광고 카피 / UI 카피
- 검색어: …
- 참고 기출 N건 (대표 3건 인용)
- 발견한 패턴/원칙: …

## 카피 제안 (N안)
1. "카피" — 구조(또는 컴포넌트·원칙) · ← 기출 근거
2. …

## 추천 1안 + 이유
- 광고: "누르고 싶은" 이유 기준 / UI: "헷갈림 없이 다음 행동" 기준
```

## 코퍼스 성장 (작업 후 습관)

- 이번 작업에서 채택된 카피는 **해당 모드의 코퍼스**에 append한다. 광고↔UI를 절대 섞지 않는다
  - 광고: `.claude/library/copy-corpus/<카테고리>.jsonl` (`source:"crafted"`, `brand`: 클라이언트명)
  - UI: `.claude/library/ui-copy-corpus/<컴포넌트파일>.jsonl` (`context` 필수)
- 검색이 빈약했던 키워드는 기존 기출의 `keywords`를 보강하거나 새 기출을 추가한다
- 스키마·수집 규칙: 광고 → `.claude/library/copy-corpus/README.md`, UI → `.claude/library/ui-copy-corpus/README.md`

## 팀으로 함께 쓰기

이 스킬은 저장소에 포함돼 있어 **팀원이 이 저장소를 Claude Code에서 열면 자동 로드**된다. 별도 설치가 필요 없다.

**팀원 온보딩 (3줄)**
1. 저장소를 최신으로 받는다 (`git pull`).
2. Claude에게 `카피 써줘` / `배너 문구 뽑아줘` 라고 하거나 `/copywriter` 로 부른다.
3. 좋은 카피를 발견하면 "코퍼스에 추가해줘"라고 말한다 → 팀 공용 기출이 쌓인다.

**팀 협업 원칙**
- 코퍼스(`.claude/library/copy-corpus/`)는 **팀 공유 자산**이다. 한 사람이 추가하면 커밋·푸시로 전원이 물려받는다.
- 카피를 추가할 땐 반드시 스키마를 지키고, 특히 `keywords`에 정서어·상황어를 섞고 `why`(왜 눌리는가)를 채운다 — 이게 팀의 카피 감각을 데이터로 축적하는 지점이다.
- 성과가 확인된 우리 실 카피는 `source:"verified"`, `brand`에 클라이언트명을 넣어 자체 기출로 승격한다.
- 개인 실험 카피는 함부로 커밋하지 말고, "누르고 싶은가" 게이트를 통과한 것만 코퍼스에 올린다.

> 다른 저장소·개인 PC에서도 쓰고 싶다면 이 폴더를 플러그인으로 설치할 수 있다 → `plugin/README.md` 참조.
