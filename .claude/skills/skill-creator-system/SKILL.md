# Skill Creator — 3차 · 브랜드 시스템화

> 2차에서 고른 컨셉을 **전달·운영 가능한 시스템**으로 굳히는 단계.
> 흩어진 비주얼 결정을 모듈(카드·맵·카피·패턴)로 정리해 누구나 같은 톤으로 쓸 수 있게 한다.

## 언제 쓰나
- 컨셉 확정 후 브랜드 시스템/가이드 산출물로 정리할 때
- 가치 카드·키워드맵·카피·경쟁 CI 분석이 필요할 때
- 제안서/온보딩에 넣을 브랜드 1장 보드

## 산출 모듈 (필요한 것만 선택 조립)
→ 각 모듈 양식: **`brand-system-modules.md`** (이 폴더)
1. **브랜드 보드 (노션 스타일)** — 한 장 요약(로고/무드/핵심값), 미니멀 정리
2. **브랜드 가치 카드 / 지표** — 핵심 가치를 카드+지표 표로
3. **가치 키워드맵** — 중심 가치 → 연관 키워드 마인드맵
4. **브랜드 카피 / 헤드라인 세트** — 예: QUIET ISLAND · SENSORY DESIGN · POWER OF RETAIL
5. **기호·연상 패턴** — 브랜드 모티프를 패턴/심볼로 전개
6. **경쟁사 CI/BI 분석** — 경쟁 로고/아이덴티티 그리드 + 차별점

## 산출물
- 브랜드 시스템 보드/덱 (HTML, 라이트모드 3종). 데이터·수치 정리는 `list-deck-design` 스킬 연계.

## 연계
- 시스템 표준화: `design-system-guardian`(토큰·컴포넌트로 편입)
- 톤 일관성: `creative-director` · 경쟁 리서치: `market-research`
- 앞단계 2차 `skill-creator-concept`
- 산출 포맷: `list-deck-design`(에디토리얼 리포트), `figma-bridge`(피그마 반입)

## 주의
- 시스템은 **재사용 가능**해야 한다(일회성 시안 X) → design-system SoT로 편입 제안.
- 경쟁사 CI/BI는 출처·상표 권리 확인. 분석은 참고용, 모사 금지.
