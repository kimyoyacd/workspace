# Skill Creator — 1차 · 브랜드 기획 + 패키지 비주얼

> 디자인 = 하나의 상품의 문제를 해결하면서 **비주얼 전략을 토대로 컨셉을 정의**하는 일.
> 1차는 그 출발점: **리서치로 문제·기회를 정의하고, 첫 비주얼로 컨셉을 눈에 보이게** 만든다.

## 언제 쓰나
- 신규 브랜드/제품 컨셉 착수 (예: "고수 전문 F&B 브랜드")
- 경쟁/레퍼런스 브랜드 리서치 + 포지셔닝 맵이 필요할 때
- 패키지·제품 키비주얼/목업을 AI 이미지로 만들 때

## Phase A — 브랜드 기획서 초안
입력: 카테고리·핵심 원료/소재·타깃. 산출 흐름:
1. **경쟁 브랜드 N개(기본 30) 리서치** — 각 브랜드별: Logo · 창립연도 · 공식 홈페이지 · Core value · key message · 제품군 · 차별점 · 패키지 디자인. (※ 현재 운영 중인 브랜드만, 출처 URL 필수 — fact-checker 연계)
2. **X·Y 포지셔닝 맵** — 두 축을 정해 N개 브랜드를 배치
3. **선호도·평가 분석** — 카테고리/소비자 인식을 긍·부정으로 분해
4. **신생 브랜드 포지셔닝 전략** — 빈 공간(white space) 기반 진입 전략
산출물: `brand_planning_report`(md/PDF) + `brand_positioning_map`(이미지)

## Phase B — BX/패키지 비주얼 생성
입력: 제품 사진 + 라벨/로고. 4단계 이미지 합성 프롬프트 프레임워크를 사용.
→ 상세 프롬프트 템플릿: **`package-visual-prompts.md`** (이 폴더)
연출 베리에이션: 미니멀 럭셔리 / 초현실(달리식 그림자) / 톤 매칭 합성 등.

## 연계
- 리서치: `market-research`, `reference-curator`, `fact-checker`
- 이미지 생성: `visual-generator`(Higgsfield) — package-visual-prompts.md의 프레임워크 적용
- 다음 단계: 2차 `skill-creator-concept`(컨셉·무드 확장)

## 주의
- 리서치 수치·브랜드 정보는 출처 확인(미확인 숫자 금지). 추정은 추정으로 표기.
- 레퍼런스 직접 모사 금지 — 전략/구조를 참고하되 고유 컨셉으로.
- 생성 이미지는 시안. 최종 채택은 사용자.
