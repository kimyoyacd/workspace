# Quality Gate — 브랜드 전략 프로젝트 완료 게이트

marketer 파이프라인의 게이트. **DEEP 모드에서는 아래를 모두 충족하기 전까지
전략(STEP 6~7) 또는 HTML(STEP 8) 단계로 넘어가지 않는다.**

## 점검 방법

프로젝트 폴더의 `outputs/` 파일을 실제로 읽고 각 항목을 ✅/❌로 판정한다.
JSON은 실제 배열 길이를 세고, 출처는 실제 URL 존재를 확인한다.

## 게이트 체크리스트

### 프로젝트·브랜드
- [ ] 프로젝트 성격이 정의되었는가 (01_project_discovery.md)
- [ ] 브랜드 팩트북이 작성되었는가 (02_brand_factbook.md)
- [ ] 타깃이 행동·욕구 기준으로 정의되었는가
- [ ] FACT·CLIENT CLAIM·INFERENCE·HYPOTHESIS·UNKNOWN이 구분됐는가
- [ ] 현재 상태와 목표 상태가 비교됐는가 (03_problem_definition.md)
- [ ] 진짜 해결 문제가 의사결정 질문으로 정의됐는가

### 경쟁 시장
- [ ] 경쟁·대체 브랜드 30개 이상 (04_competitive_research.json — 실제 count)
- [ ] 모든 브랜드에 출처 존재
- [ ] 핵심 브랜드 2개 이상 출처 검증
- [ ] 경쟁 브랜드가 3~5개 카테고리로 군집됐는가 (05)
- [ ] 카테고리별 공통점과 한계가 있는가
- [ ] 축 후보가 최소 3세트 존재하는가 (06)
- [ ] 최종 축 선정 이유가 있는가
- [ ] XY 좌표 근거가 있는가
- [ ] 화이트스페이스가 기회와 함정으로 구분됐는가 (07 — 유형 표시 포함)

### 이종업계
- [ ] 이종업계 사례 30개 이상 (08 — 실제 count)
- [ ] 모든 사례에 출처 존재
- [ ] 프로젝트 문제와 연결되는 이유가 있는가
- [ ] 사례가 3~5개 카테고리로 군집됐는가 (09)
- [ ] 차용할 원리와 모방 금지 요소가 구분됐는가 (WHAT TO BORROW/TRANSLATE/AVOID)

### 전략
- [ ] 브랜드 자산과 시장 기회가 연결됐는가 (10_insights.md)
- [ ] 최소 2개 이상의 전략 대안이 있는가 (11_strategy.md)
- [ ] 추천 전략의 선택 이유가 있는가
- [ ] 선택하지 않은 방향이 명시됐는가
- [ ] 리스크와 검증 계획이 있는가
- [ ] 핵심 주장에 Evidence ID가 연결됐는가

### HTML (deck.html 생성 후)
- [ ] 기존 HTML 디자인 문법이 유지됐는가 (template.html·골든 예시 기준)
- [ ] 브랜드 팩트북이 보이는가
- [ ] 경쟁사 30개 이상이 실제 카드로 확인되는가
- [ ] 이종업계 30개 이상이 실제 카드로 확인되는가
- [ ] 주요 3개 카테고리가 카드로 보이는가
- [ ] 각 카드에서 출처를 확인할 수 있는가
- [ ] XY맵과 좌표 근거를 확인할 수 있는가
- [ ] 화이트스페이스의 기회와 위험이 함께 보이는가
- [ ] 리서치에서 전략까지의 연결이 보이는가

## 판정 규칙

- **미달 항목이 있으면 HTML을 완성 처리하지 않는다.** 미달 목록과 필요한 추가 조사를 명시한다.
- 기존 자료만으로 30개가 불가능하면 수량을 축소하지 말고 **웹 리서치를 추가**해 기준을 충족한다.
- QUICK/STANDARD 모드는 해당 모드의 수량 기준(8+/6+, 15+/15+)으로 게이트를 적용한다.
- T4(추정) 출처는 수량 충족으로 세되, "사실 근거 사용 금지 + 추가 검증 대상" 플래그를 확인한다.

## 출력 형식

```
## Quality Gate 판정 — {프로젝트명} ({모드})

### 통과 (N/M)
✅ ...

### 미달 (K건)
❌ 경쟁 브랜드 30개 미달 — 현재 24개. 대체재 카테고리 6개 추가 조사 필요.

### 판정: PASS / HOLD
HOLD 시: 다음 단계 진행 불가. 추가 조사 항목 → ...
```
