## 1. Brand Overview

- **Brand Name**: FANLIGHT (팬라이트)

- **Value**: 팬라이트의 브랜드 정신과 철학을 함축적으로 담은 디자인

## 4. Draft A — "Unity of Light" (빛의 연대)

### Concept
팬들의 빛이 함께 모여 아티스트와 하나가 되는 유대를 표현.
하나의 빛 → 수많은 빛들이 모여 → 팬과 아티스트가 하나가 되는 → 팬라이트

### Symbol
- **형태**: 모듈형 블록으로 구성된 'F' 레터마크
- **스타일**: 지오메트릭, 각진 모듈 블록이 계단식으로 쌓인 형태
- **특징**: 픽셀/LED를 연상시키는 정사각형 모듈 조합, 관객석 빛의 집합을 상징

### Logotype
- **스타일**: 올 캡스(ALL CAPS) 산세리프, 볼드, 기하학적
- **표기**: **FANLIGHT** (대문자)

### Color System

#### Basic Color
| Name | Role | Note |
|------|------|------|
| **Illumination** | Primary Light | 순백색 (White) — 빛의 상징 |
| **Blackout** | Primary Dark | 순흑색 (Black) — 어둠의 상징 |

#### Sub Color — Neon Multicolor (30색 스펙트럼)
빛이 만들어내는 다양한 색상을 표현하는 네온 멀티컬러 팔레트.
풀 스펙트럼으로 Solid 버전과 Gradient 버전 존재.

**Solid Spectrum** (6x5 Grid):
- Row 1: Yellow-Green → Neon Yellow → Chartreuse → Lime → Green → Bright Green
- Row 2: Spring Green → Emerald → Mint → Cyan → Turquoise → Sky Blue
- Row 3: Blue → Indigo → Violet → Purple → Magenta → Hot Pink
- Row 4: Rose → Crimson → Cherry → Red → Coral → Orange-Red
- Row 5: Orange → Deep Orange → Tangerine → Amber → Gold → Yellow

**Gradient Combinations** (8 sets):
- Yellow ↔ Orange, Pink ↔ Rose, Gold ↔ Purple
- Orange ↔ Red, Cyan ↔ Lavender, Magenta ↔ Purple
- Teal ↔ Navy, Cyan ↔ Green

### Typography

#### English: Neue Haas Unica Pro
- **Designer**: Monotype, 2015
- **Weights**: UltraLight / Thin / Light / Regular / Medium / Bold / Heavy / Black / ExtraBlack
- **Primary Weights**: Bold (제목), Medium (본문), Light (보조)
- **특성**: Helvetica와 Univers의 장점을 결합, 현대적 디지털 환경에 최적화

#### Korean: Pretendard (프리텐다드)
- **Designer**: Gil Hyeong Jin, 2020
- **Weights**: Thin / ExtraLight / Light / Regular / Medium / SemiBold / Bold / ExtraBold / Black
- **Primary Weights**: Bold (제목), Medium (본문), Light (보조)
- **특성**: 본고딕 기반, 182개 언어 지원, 무료 배포 서체

### Key Visual System
| Type | Name | Description |
|------|------|-------------|
| Type A | **Module** | 모듈을 활용한 그래픽 — 관객의 빛을 대변, 상하좌우 확장 가능 |
| Type B | **Connection** | 연결을 활용한 그래픽 — 스타와 팬을 이어주는 이미지 |
| Type C | **Diagonal** | 사선을 활용한 그래픽 — 빛의 이미지를 표현 |

### Iconography Style
- 심볼에서 파생된 기하학적 픽셀 모듈 아이콘
- 카테고리: FAN, ARTIST, MUSIC, STAGE, SPOTLIGHT, PLAY, FL WRIST BAND, FL LED STICK, FL BEACH BALL

### Application Tone & Manner
- **배경**: 블랙 기반 (공연장/어둠의 환경)
- **악센트**: 네온 멀티컬러 (LED 빛의 다양성)
- **무드**: 테크놀로지 + 엔터테인먼트의 융합, 깔끔하고 모던한 느낌

## 5. Draft B-1 — "Shining Moment" (빛나는 순간)

### Concept
평생의 감동으로 남는 장면, 팬들과 아티스트의 빛나는 순간을 표현.
팬들의 마음이 모여 → 더욱 밝은 빛이 되어 → 함께할 때 가장 빛나는 순간

### Symbol
- **형태**: 별/스파크 형태 — 'F'와 'L' 이니셜을 활용하여 빛과 스타를 형상화
- **스타일**: 유기적 라운드 스트로크, 중심에서 방사형으로 뻗어나가는 5개의 선
- **컬러**: 네온 옐로우(Creative Neon Yellow)

### Symbol Character
- 대표 이니셜 'F'와 'L'을 활용
- 빛을 통해 최고의 순간을 만들어가는 팬라이트

### Logotype
- **스타일**: 소문자 (lowercase), 라운드형 산세리프
- **표기**: **Fanlight** (첫 글자만 대문자)
- **특성**: Futura 'a' 스타일의 Geometric 라운드, 심볼의 곡선과 조화

### Color System

#### Primary Color: Creative Neon Yellow
- **상징**: Energy & Excitement / Innovation & Technology / Visibility & Safety / Joy & Positivity / Youth & Vitality / Unity & Togetherness / Motivation & Inspiration

#### Supporting Colors
| Name | Role |
|------|------|
| **Creative Neon Yellow** | Primary (on Black) |
| **Black** | Primary Background |
| **Mid Grey** | Secondary Neutral |
| **Light Grey** | Tertiary / Background |

### Typography

#### English: Circular Std
- **Designer**: Laurenz Brunner, 2013
- **Weights**: Thin / Light / Regular / Book / Medium / Bold / Black / ExtraBlack
- **Primary Weights**: Bold (제목), Medium (본문), Book (보조)
- **특성**: 지오메트릭 산세리프, 원과 곡선의 이미지 — 세련되고 젊은 브랜드 이미지

#### Korean: Pretendard (프리텐다드)
- 동일 (Draft A와 같음)

### Key Visual System
| Type | Name | Description |
|------|------|-------------|
| Type A | **Light** | 타원을 활용한 그래픽 — 관객의 빛, 축소/확대 가능 |
| Type B | **Spotlight** | 빛의 움직임을 표현한 그래픽 — 스타와 팬을 연결하는 스포트라이트 |

### Application Tone & Manner
- **배경**: 블랙 기반
- **악센트**: 네온 옐로우 단색 강조
- **무드**: 젊고 에너지 넘치는, 팝/스트리트 감성, 대담하고 유니크한

## 7. Draft C — "Infinite Sync" (무한한 교감)

### Concept
팬과 아티스트를 잇는 기술을 통해 공동의 경험과 감정의 무한한 교감을 표현.
팬들의 무한한 사랑(하트) → 빛을 통해 연결(X형 교차) → 빛을 통해 연결해주는 팬라이트(F 심볼)

### Symbol
- **형태**: 연속된 하나의 선(One-line)으로 그려진 'F' 레터마크
- **스타일**: 라운드 코너의 연결된 스트로크, 끊기지 않는 연속선
- **구성 요소**: 하트 → X(교차/연결) → F(팬라이트) 로 모핑되는 스토리
- **컬러**: Zenith Mint (네온 민트 그린)
- **특성**: 아웃라인(stroke) 기반, 빛의 궤적을 연상, 유기적이면서 기술적

### Logotype
- **스타일**: 올 캡스(ALL CAPS), 산세리프, 미디엄-볼드 웨이트
- **표기**: **FANLIGHT** (대문자)
- **특성**: 기하학적이면서 약간의 라운드 터치, Galano Grotesque 계열

### Color System

## 10. Proposal Design System (제안서 디자인 시스템)

> 이 섹션은 제안서/프레젠테이션 제작 시 적용할 레이아웃, 위계, 도식화, 밀도 규칙을 정의합니다.

### 10.1 슬라이드 그리드 & 레이아웃

#### 기본 그리드
- **캔버스**: 16:9 (1920×1080px 또는 동비율)
- **외곽 마진**: 상하좌우 균등 (60–80px)
- **컬럼**: 12컬럼 그리드, 거터 24px
- **기본 레이아웃 단위**: 2등분(50/50), 3등분(33/33/33), 비대칭(40/60, 30/70)

#### 영역 분배 원칙 — "밀도 밸런스"
```
┌─────────────────────────────────┐
│  상단 1/3: 제목 + 컨텍스트 텍스트  │  ← 텍스트 중심, 밀도 낮음
│─────────────────────────────────│
│  중단 1/3: 핵심 비주얼/데이터      │  ← 시각 요소 중심, 밀도 높음
│─────────────────────────────────│
│  하단 1/3: 보조 정보 + 출처/범례   │  ← 보충 정보, 밀도 낮음
└─────────────────────────────────┘
```
- 시각적 무게가 위나 아래 한쪽으로 쏠리지 않도록 상-중-하 영역에 고르게 분배
- 히어로 숫자나 차트가 상단에 있으면 하단에 설명 텍스트 블록을 배치하여 균형 확보
- 반대로 상단이 텍스트 중심이면 하단에 차트/도식을 배치
- **대각선 밸런스**: 좌상단과 우하단, 또는 우상단과 좌하단에 시각적 무게 요소를 교차 배치

#### 다중 카드 레이아웃
- 2×2 그리드: 4개 동일 크기 카드 균등 배치 (각 카드 내부에 텍스트+비주얼 조합)
- 1+2 비대칭: 좌측 대형 카드 1개 + 우측 소형 카드 2개 세로 스택
- KPI 행: 수평으로 3~5개 메트릭 카드 1줄 + 아래에 상세 차트 카드 1개

### 10.2 타이포그래피 위계 (제안서 전용)

슬라이드 내 텍스트 요소 간 크기 대비를 극대화하여 시선 흐름을 명확하게 유도한다.

#### 크기 스케일
| Role | EN Weight | KR Weight | 크기(px) | 용도 |
|------|-----------|-----------|---------|------|
| **Hero Stat** | Bold / Black | Bold | 72–120 | 핵심 수치 (%, MWh, 금액 등) — 슬라이드당 최대 1~2개 |
| **Page Title** | Bold | Bold | 32–40 | 페이지 제목, 섹션명 |
| **Subtitle** | Medium | Medium | 20–24 | 부제, 카드 제목, 차트 타이틀 |
| **Body** | Regular / Medium | Medium | 14–16 | 본문 설명, 인사이트 텍스트 |
| **Caption** | Light | Light | 11–13 | 출처, 범례, 부연, 축 라벨 |
| **Badge/Tag** | Medium | Medium | 12–14 | 데이터 포인트 하이라이트 뱃지, 변화율 표시 |

#### 위계 적용 규칙
- Hero Stat은 반드시 주변 텍스트 대비 **4× 이상 크기 차이**를 둔다
- 한 슬라이드에 Hero Stat이 2개 이상이면 동일 크기/동일 웨이트로 병렬 배치
- Page Title은 슬라이드 좌상단 고정 위치
- Body 텍스트는 최대 3~4줄 이내, 길어지면 분할

### 10.3 카드 컴포넌트

#### 기본 카드 구조
```
┌──────────────────────────────┐
│ [Label/Title]      [Icon]    │  ← 상단: 라벨 좌, 아이콘/뱃지 우
│                              │
│                              │  ← 중단: 여백 (breathing room)
│                              │
│ [Hero Number]   [Indicator]  │  ← 하단: 핵심 수치 좌, 변화지표 우
└──────────────────────────────┘
```

#### 카드 스타일링
- **배경**: 슬라이드 배경과 구분되는 면 (다크 배경 위 약간 밝은 면, 또는 라이트 배경 위 화이트)
- **모서리**: 라운드 코너 12–16px
- **테두리**: 없음 또는 1px subtle stroke (배경과 구분 필요 시)
- **내부 패딩**: 24–32px 균등
- **그림자**: 없거나 극히 미세 (flat 기조 유지)

#### 변화 지표 (Indicator) 표현
- 상승: `↑12.4%` — 포지티브 컬러 (브랜드 악센트 또는 그린 계열)
- 하락: `↓12.4%` — 뉴트럴 블루 또는 그레이 (경고가 아닌 정보 전달)
- 뱃지 형태: 필(pill) 모양 라운드 배경 (예: 노란 필 안에 수치)

### 10.4 데이터 시각화 — 차트 & 그래프 스타일

#### 공통 원칙
- **미니멀**: 불필요한 장식 요소 제거 (3D 효과, 그라데이션 배경, 장식 테두리 금지)
- **그리드라인**: 수평 점선(dashed) 만 사용, 수직 그리드라인 없음
- **축 라벨**: Caption 크기, Light 웨이트, 회색 계열
- **데이터 강조**: 특정 데이터 포인트에 필(pill) 뱃지 또는 원형 마커로 하이라이트
- **범례**: 차트 하단 또는 좌측에 인라인 배치, 별도 박스 금지

#### 바 차트 (Bar Chart)
```
Style:
- 바 너비: 균일, 간격 = 바 너비의 40–60%
- 바 색상: 단색 (브랜드 프라이머리 또는 Black)
- 바 모서리: 상단만 약간 라운드 (2–4px) 또는 완전 직각
- 하이라이트: 특정 바에 악센트 컬러 + 상단 뱃지로 수치 표시
- X축: 카테고리 라벨 (Caption 크기)
- Y축: 수치 라벨 (Caption 크기, 점선 그리드 대응)
```

#### 도넛/파이 차트 (Donut / Pie)
```
Style:
- 도넛 선호 (파이보다 현대적)
- 스트로크 두께: 굵게 (전체 반지름의 25–35%)
- 세그먼트 간격: 2–4px 갭
- 중앙: 핵심 수치 또는 비워둠
- 최대 세그먼트: 4개 이하 (그 이상이면 바 차트로 전환)
```

#### 라인/스파클라인 (Line / Sparkline)
```
Style:
- 스트로크: 2px, 단색
- 영역 채움(area fill): 10–20% 투명도 그라디언트 (선택)
- 미니 카드 내 스파클라인: 축/라벨 없이 순수 선만 표시, 높이 40–60px
- 데이터 포인트 마커: 끝점만 표시 (시작·끝 또는 최대·최소)
```

#### 벤 다이어그램 / 원형 비교
```
Style:
- 원 채움: 30–50% 투명도, 겹침 영역은 색 혼합으로 자연 표현
- 원 테두리: 없음 (채움으로만 구분)
- 라벨: 각 원 내부 또는 바로 옆에 직접 배치
- 최대 3개 원 (그 이상이면 다른 도식으로 전환)
```

### 10.5 데이터 시각화 — 도식 & 인포그래픽 스타일

#### 도트 매트릭스 (Dot Matrix / Unit Chart)
```
용도: 비율·분포를 직관적으로 표현 (예: "4명 중 1명")
Style:
- 그리드: N×M 배열 (예: 4×10 = 40개 도트)
- 도트 모양: 원형 또는 정사각 (브랜드 모듈과 연계)
- 활성 도트: 브랜드 악센트 컬러 (채움)
- 비활성 도트: 20% 투명도 또는 아웃라인만
- 하단에 캡션으로 비율 수치와 설명
```

#### 포지셔닝 맵 (2-Axis Positioning Map)
```
용도: 경쟁 분석, 시장 포지셔닝
Style:
- 축: 가는 실선 (1px, 그레이), 축 라벨 양 끝에 배치
- 교차점: 정중앙
- 데이터 포인트: 원형 버블 (브랜드별 고유 컬러)
- 자사 포인트: 가장 크게 또는 악센트 컬러로 강조
- 범례: 좌측 또는 하단에 브랜드명 + 컬러 도트 인라인
- 배경: 사분면 구분선 없이 축만으로 구분 (깔끔)
```

#### 수평 스택 바 (Horizontal Stacked Bar)
```
용도: 구성 비율 비교 (예: Generalist IT 74% vs Dedicated Security 26%)
Style:
- 높이: 32–48px, 라운드 모서리 없음 (직각)
- 세그먼트 색상: 브랜드 팔레트 내 2–3색 계조
- 세그먼트 라벨: 바 위 또는 바 안에 직접 표기
- 전체 합산 = 100% 명시
```

#### 프로세스/타임라인
```
용도: 단계, 히스토리, 로드맵
Style:
- 수평 또는 수직 연결선 (1–2px 실선)
- 노드: 원형 또는 정사각 (8–12px)
- 활성 노드: 채움, 비활성: 아웃라인
- 라벨: 노드 상하 또는 좌우에 직접 배치
- 연도/단계명: Bold, 설명: Light
```

### 10.7 밀도 밸런스 규칙

#### 슬라이드 단위 밀도 체크리스트
1. **상하 균형**: 슬라이드를 수평 3등분했을 때, 시각적 무게가 한 영역에 70% 이상 집중되지 않는다
2. **좌우 균형**: 비대칭 레이아웃(40/60)은 허용하되, 한쪽이 완전히 비는 것은 금지
3. **여백 비율**: 전체 면적의 30–40%는 여백으로 유지 (과밀 방지)
4. **요소 개수**: 슬라이드당 시각 요소(차트, 카드, 이미지) 최대 4개, 그 이상은 분할
5. **텍스트 분량**: 본문 텍스트 총량 80단어 이하 (한글 기준 160자)

#### 무게 분산 전략
| 상황 | 해결 |
|------|------|
| 상단에 큰 차트 집중 | 하단에 텍스트 블록 + 출처 배치 |
| 좌측에 텍스트만 | 우측에 차트/아이콘 배치 |
| 숫자만 크게 중앙 배치 | 상단에 컨텍스트 제목, 하단에 설명 문단 |
| 카드 4개 균등 | 각 카드 내부도 라벨+수치+미니비주얼 3요소 구성 |
| 전체가 텍스트 | 핵심 문장 1개를 Hero Stat 크기로 끌어올려 시각 앵커 생성 |

#### 페이지 간 리듬
- 데이터 밀집 페이지 다음에는 여백이 많은 구분/인용 페이지 배치
- 동일 레이아웃 3회 이상 연속 금지 — 변주를 넣어 리듬감 유지
- 차트 유형도 연속 반복 피함 (바 차트 → 도넛 → 포지셔닝 맵 등 순환)

### 10.8 이미지 & 그래픽 사용 규칙

#### 사진/이미지
- 다크 톤 보정 (공연장, 조명, 무대 이미지 중심)
- 블랙 배경과 자연스럽게 블렌딩되도록 페이드 처리
- 이미지 위 텍스트 배치 시: 반투명 오버레이(Black 60–80%) 또는 하단 그라디언트

#### 아이콘
- 브랜드 아이콘 세트 우선 사용 (심볼 파생 스타일)
- 외부 아이콘 사용 시: 라인 스타일, 1.5–2px 스트로크, 라운드 조인
- 크기: 24×24 ~ 48×48px (카드 내), 보조 요소로만 활용
- 카드 우상단에 아이콘 배치 시 장식이 아닌 카테고리 식별 목적

#### 키 비주얼 그래픽
- 심볼에서 파생된 모듈/스트로크 패턴을 배경 텍스처로 활용
- 투명도 5–15%로 은은하게 깔아 페이지에 브랜드 뉘앙스 부여
- 구분 페이지(Divider)에서는 투명도 높여 30–50%로 존재감 강조

## Usage Notes

이 파일은 FANLIGHT 브랜드 아이덴티티의 4가지 디자인 제안(Draft A/B-1/B-2/C)과
제안서 제작을 위한 레이아웃·도식화·밀도 밸런스 규칙을 포괄적으로 정리한 것입니다.

### 제안서 제작 시 참조 순서
1. **브랜드 전략** (Section 2) → 메시지와 톤 확인
2. **선택 Draft** (Section 4–7) → 컬러, 서체, 비주얼 스타일 결정
3. **제안서 디자인 시스템** (Section 10) → 레이아웃, 차트, 밀도 규칙 적용
4. **슬라이드 템플릿** (Section 10.6) → 페이지 유형별 구조 참조

### 스킬 활용 예시
