# 05. 업무 운영 시스템 — 파일·폴더 정리

> 출발점: "여러 일을 동시에 하니 폴더가 안 잡힌다" = 게으름이 아니라 **구조의 부재**.
> 동시 진행이 많을수록 "모든 게 들어갈 정해진 집"이 필요하다.

마지막 갱신: 2026-06 (설계)

---

## 진단

- 보관: 로컬 + Google Drive + 클라우드 **혼합**
- 프로젝트 관리: **노션**(상태 체크) / 리소스: **시트**
- 주력 작업 3종: **제안서 · 회사소개 · 견적** (= "일을 따내는 도구" = 강점인 '맥락 포장'이 발휘되는 곳)
- 문제: 동시 진행 多 → 일관된 보관 위치 없음 → 흩어짐

## 빼기 원칙 5가지

1. **상태는 딱 3개** — `진행중` / `보관` / `자산`. "나중에 볼 것" 폴더 금지(=쓰레기장).
2. **한 프로젝트 = 한 집**, 내부 구조는 항상 동일하게 반복(뇌가 길을 외움).
3. **"최신본은 단 하나"** — 회사소개·제안서 버전 지옥 종료. `_최신본` 하나 + 나머지 `_구버전`.
4. **끝나면 즉시 `보관`으로** — 진행중을 항상 가볍게.
5. **이름 통일** — `날짜_클라이언트_종류` (예: `250624_코웨이_제안서`).

## 폴더 지도

```
📁 00_자주쓰는        ← 주력 3종 (최상단, 빠른 접근)
   ├── 제안서/
   │   ├── _템플릿/
   │   └── 250624_코웨이_제안서/
   ├── 회사소개/
   │   ├── _최신본          ← 진짜 최신 단 하나
   │   └── _구버전/
   └── 견적/
       ├── _템플릿/
       └── 250624_코웨이_견적/

📁 01_진행중          ← 지금 돌아가는 프로젝트만
   └── 코웨이_안마기/
       ├── 01_브리프·자료
       ├── 02_작업
       ├── 03_납품
       └── _메모

📁 02_보관            ← 끝난 건 전부 (연도별)
   └── 2026/코웨이_안마기/

📁 03_자산            ← 재사용 리소스
   ├── 폰트
   ├── 로고·CI
   ├── 레퍼런스
   └── 포트폴리오 소스
```

## 플랫폼 역할 (단일 기준 원칙)

| 플랫폼 | 단 하나의 역할 | 규칙 |
|--------|---------------|------|
| **노션** | 사령탑(상태·진행) | "지금 뭐가 돌아가는가"의 유일한 기준. 파일은 링크만. |
| **로컬/드라이브** | 파일 실물 | 위 폴더 지도 그대로. |
| **시트** | 견적·리소스 데이터 | 숫자·목록만. 파일은 안 둠. |

## 실행 체크리스트 (B 단계에서 진행)

- [ ] 최상단 폴더 4개 만들기 (`00_자주쓰는` `01_진행중` `02_보관` `03_자산`)
- [ ] 회사소개 `_최신본` 단 하나 확정 → 나머지 `_구버전`으로
- [ ] 제안서·견적 `_템플릿` 정리
- [ ] 진행중 프로젝트만 `01_진행중`에 남기고 나머지 `02_보관`으로
- [ ] 노션 = 상태, 시트 = 데이터로 역할 정리
