## 페르소나 설정

### 페르소나 A: 이선민 (색조 마케팅팀, 평가 주도)

```
역할: 실무 운영·평가 주도
관심: 디자인 감각 / "요즘 색조스러움" / AI 활용 / 빠른 실무 대응
평가 기준: "내가 매달 같이 일하고 싶은 팀인가?"
심리: 설명회 열었을 때 기대 이상 제안이 없으면 실망
권한: 평가 점수 최대 영향력
```

### 페르소나 B: 김민지 (구매팀, 계약 담당)

```
역할: 계약·단가·정산 관리
관심: 임의 예가 범위 / 계약 조건 명확성 / 외주 관리 가능 여부
평가 기준: "계약하기 쉬운 업체인가? 문제 없이 정산되는가?"
심리: 단가와 조건만 확인 → 나머지는 이선민에게 맡김
권한: 견적서 검토·계약 프로세스
```

### 페르소나 C: 고주현 (웨이크메이크 담당, 감각 판단)

```
역할: 웨이크메이크 브랜드 무드 최종 판단
관심: "힙하고 세련된 프로페셔널" / "다채로운 레이아웃"
평가 기준: "이 팀이 웨이크메이크 무드를 이해하는가?"
심리: 과제 시안 보는 순간 즉각 반응 (감각적 직관)
권한: 웨이크메이크 상세 페이지 최종 감수 예정
```

## 페르소나 A (이선민) 시뮬레이션 — 전체 35장

### Opening (P-0~P-4)

**P-0 COVER**
```
[3초 반응] "제안사 이름 + 3개 브랜드 나왔다. 표지는 깔끔한가?"
[IL 포인트] 표지에서 브랜드 무드가 느껴지는가?
→ 위험: 밋밋한 PPT 표지 → "이미 감각 없어 보여"
→ 대응: 3개 브랜드 키 컬러 분할 배경 필수, 타이포가 감각적이어야
```

**P-1 THE REAL QUESTION**
```
[3초 반응] "이 팀이 우리 문제를 알고 있네"
[IL 포인트] 긴장 3쌍이 정확히 OY 내부에서 논의되던 것인가?
→ 강점: "노후화된 상세 페이지"를 직접 언급 → 즉각 공감
→ 강점: 일본·미국 글로벌 수치 인용 → 우리 팀도 인지 못하던 것
→ THE ANSWER "감각이 설계되지 않았기 때문이다" → 고개 끄덕임
✅ 이 슬라이드는 가장 강력한 오프닝
```

**P-2 PROJECT GOAL**
```
[3초 반응] "3-Layer Goal이 우리 내부 언어랑 비슷한데?"
[IL 포인트] Bridge "감각"이 억지스럽게 느껴지지 않는가?
→ 위험: "감각"이라는 단어가 너무 많이 반복되면 공허하게 들림
→ 대응: "감각" 사용 횟수 조절. Goal에서는 1~2회만 사용
```

**P-3 EXECUTIVE SUMMARY**
```
[3초 반응] "3기둥이 뭔지 빠르게 보인다. 과제 시안 어디 있지?"
[IL 포인트] "과제 시안이 곧 실무 첫 산출물입니다"가 설득력 있는가?
→ 강점: 임원은 본문 안 읽어도 이 장에서 판단 가능하도록 설계됨
→ 개선: 과제 시안 썸네일 이 장에 포함시키면 더 강력
```

**P-4 CONTENTS**
```
[3초 반응] "5 Part 구조구나. Part 3 Strategic Vision 빨리 보고 싶어"
→ 크게 문제 없음. 페이지 넘기는 속도 빨라질 것
```

### Part 1 (1-1 ~ 1-4)

**1-1 브랜드 개요**
```
[3초 반응] "우리 데이터를 찾아봤구나. +50% 수치는 어디서 가져왔지?"
[IL 포인트] 출처가 명확한가? 이선민은 내부 데이터와 다를 경우 신뢰 하락
→ 위험: 수치 출처 불명확하면 "혹시 틀린 거 아냐?"
→ 대응: "CJ 뉴스룸", "뷰티누리" 등 출처 명기 또는 각주 처리
```

**1-2 레퍼런스 분석**
```
[3초 반응] "롬앤·3CE·NARS·Fenty — 잘 골랐네. 국내+글로벌 밸런스 좋아"
[IL 포인트] 우리 브랜드와 직접 연결되는 시사점이 있는가?
→ 강점: "우리의 학습점" 1줄이 명확하면 이선민이 바로 메모
✅ 레퍼런스 선택이 설명회 담당자 기대치와 일치
```

**1-3 Competitive Audit**
```
[3초 반응] "우리 상세 페이지 문제를 5가지로 짚었네. 좀 찔리는 부분도..."
[IL 포인트] 지나치게 비판적이면 클라이언트 불쾌감
→ 위험: 현재 상태 비판이 "우리가 잘못했다"로 들리면 역효과
→ 대응: 진단은 팩트 중심, 감정적 표현 없이. "기회 영역"으로 프레이밍
→ 슬라이드 헤드라인: "문제"가 아닌 "개선 기회" 언어 사용
```

**1-4 핵심 인사이트**
```
[3초 반응] "Sensory Textures 데이터 — 2026 K-뷰티 트렌드. 글로벌 맥락이구나"
[IL 포인트] "3초 안에 감각" 통계가 출처 있는가?
→ 위험: "소비자는 3~5초 안에 결정한다"는 근거 필요
→ 대응: Nielsen, Baymard Institute 등 이커머스 UX 연구 데이터 인용
    또는 "알려진 UX 원칙"으로 표현 조정
```

### Part 2 (2-1 ~ 2-4)

**2-1 웨이크메이크**
```
[3초 반응] "고주현이 했던 말이 여기 나오네. 우리 이야기 하고 있잖아"
[IL 포인트] 담당자 발언 인용이 정확한가?
→ 강점: 설명회 발언 직인용 = "이 팀은 설명회를 진지하게 들었다"
→ 주의: 발언 인용은 맥락을 왜곡하지 않는 범위에서만
✅ 이 슬라이드에서 이선민의 신뢰 급상승
```

**2-2 컬러그램**
```
[3초 반응] "윤진희 발언도 반영했네. 트렌디+정리된 구성 — 맞는 분석"
[IL 포인트] IP 콜라보 시스템 대응력에 대한 언급이 충분한가?
→ 강점: IP 콜라보 상세 페이지 퀄리티 격차 분석 → 정확한 진단
→ 개선: 잔망루피 사례 직접 언급하면 더 구체적
```

**2-3 필리밀리**
```
[3초 반응] "도구 브랜드가 색조 감성 흡수 — 이 분석은 새롭다"
[IL 포인트] 필리밀리 담당자가 별도 있는가? (윤진희가 겸담?)
→ 개선: 필리밀리 관련 구체적 발언이 있다면 인용 추가
→ 없다면: "이선민 공통 발언 기반"으로 명기
```

**2-4 공통 과제**
```
[3초 반응] "AI 활용 허용된다는 내용이 우리 발언 인용으로 나왔네"
[IL 포인트] RA 검수 대응 언급 → 김민지에게도 긍정적
→ 강점: 4가지 공통 과제가 설명회에서 실제 언급된 것들
✅ "이 팀은 설명회를 전략 인풋으로 활용했다"는 인상 강하게 줌
```

### Part 3 (3-1 ~ 3-6)

**3-1 전략 총괄**
```
[3초 반응] "3가지 컨셉. 같은 방향의 다른 입구라고 설명하네"
[IL 포인트] 3개 컨셉이 실제로 다른 것처럼 보이는가?
→ 위험: 3개 컨셉 이름(Hook/Color/Scroll)이 너무 비슷하게 느껴질 수 있음
→ 대응: 각 컨셉에 강렬한 비주얼 차별화 필요 (색상·이미지·아이콘 달리)
```

**3-2 Hook First**
```
[3초 반응] "3초 안에 잡는다 — 맞아. 그런데 실제로 어떻게 생긴 건지 보고 싶어"
[IL 포인트] ← 이 질문이 가장 중요
→ 위험: 설명만 있고 시각 예시 없으면 이선민 실망
→ 대응: 웨이크메이크 도입부 와이어프레임 스케치 반드시 삽입
✅ 스케치 한 컷이 설명 10줄보다 강하다
```

**3-3 Color Speaks**
```
[3초 반응] "퍼스널 컬러 매핑을 AI로 만든다? 실제로 가능한가?"
[IL 포인트] AI 퍼스널 컬러 시뮬레이션의 품질에 대한 의구심
→ 위험: "가능합니다"만 써놓으면 믿지 않음
→ 대응: AI 생성 샘플 이미지를 이 슬라이드에 직접 삽입
    (설명회 "너무 열려 있다" 발언 = 품질만 좋으면 됨)
```

**3-4 Scroll to Cart**
```
[3초 반응] "전환율 구조는 이해됐어. 그런데 이 컨셉만의 특별함은?"
[IL 포인트] 3번째 컨셉이 앞 두 개보다 약하게 느껴질 수 있음
→ 개선: "컬러그램 IP 콜라보 무드→스토리→소장 여정" 구체 사례로 강화
→ 또는: 컬러그램 상세 페이지 스크롤 여정 와이어프레임 삽입
```

**3-5 3 Concept × 3 Brand 매트릭스**
```
[3초 반응] "브랜드별로 어떻게 적용하는지 한눈에 보이네. 유용하다"
[IL 포인트] 도트(●) 강도 표현이 과학적 근거가 없음
→ 위험: "왜 웨이크메이크 Hook First가 5점인가?" 질문 올 수 있음
→ 대응: 도트 대신 "주력/보조/선택" 3단 레이블로 변경 (더 명확)
```

**3-6 AI 활용 전략**
```
[3초 반응] "과제 시안에 AI 연출컷이 실제로 들어있구나"
[IL 포인트] 이 슬라이드와 과제 시안이 연결되는 순간
→ 강점: "말만 하는 AI가 아닌 실물을 보여주는 AI" 선언
→ 강점: 이선민 발언 직인용 = "우리가 들었다"
✅ AI 연출컷 Before/After가 제안서에서 가장 강력한 증거 중 하나
```

### Part 4 (4-1 ~ 4-7)

**4-1 수행 범위**
```
[3초 반응] "A~F 범위가 우리가 원하는 원스톱이네"
[IL 포인트] 번역·RA 대응이 포함된 것 → 이선민 실무 고민 해결
✅ 이 슬라이드는 빠르게 확인 후 넘어감 (문제 없음)
```

**4-2 워크플로우**
```
[3초 반응] "D-30 기준 30일 사이클. 현실적인가?"
[IL 포인트] 매달 정기 납품이 가능한가에 대한 현실성 검증
→ 위험: "D-20에 시안 제출" → 실제로 리뷰 시간이 충분한가?
→ 대응: 클라이언트 피드백 기간(D-15~D-10, 5일)이 명시되어 있음 ✅
→ 추가: AI 파이프라인으로 단축 가능하다는 내용 강조
```

**4-3 ~ 4-7**
```
[이선민 반응] 빠르게 스캔. 인력 프로필에서 잠깐 멈춤
[IL 포인트] 실제 인력의 얼굴과 포트폴리오가 있는가?
→ 위험: 빈 프로필 박스 = 신뢰 급락
→ 필수: 4-5 인력 프로필에 실제 인물 사진 + 대표 프로젝트 반드시 삽입
```

✅ 열린 질문: "과제 시안은 별도 파일인가? 여기서도 볼 수 있나?"
→ 과제 시안을 제안서 Part 3 이후 또는 별첨으로 명확하게 안내 필요

## 페르소나 B (김민지) 시뮬레이션 — 요약

```
관심 슬라이드: P-0(커버 확인), A-1(견적서), 4-4(인력), 4-2(일정)

P-0   "어떤 회사인지 이름 확인"
A-1   "단가가 예가 범위 안인가?" ← 가장 중요
       → 단가 구성 명확히, 항목별 분리, 글로벌 버전 추가 단가 명시
4-4   "외주 인력이 많으면 관리 어렵지 않나?" → 단일 창구 원칙 (4-7) 확인
4-2   "일정이 매달 지켜질 수 있나?" → 납품 후 사후 대응 3일 명시 ✅
나머지  "이선민이 OK 하면 됨"

리스크: 견적서 항목이 불명확하거나 예가 범위를 벗어나면 즉시 탈락
대응: 단가는 RFP 임의 예가 범위 안에서 항목 분리하여 투명하게 제시
```

## 페르소나 C (고주현) 시뮬레이션 — 요약

```
관심 슬라이드: 과제 시안 (가장 먼저), 3-2 Hook First, 2-1 웨이크메이크

과제 시안  "도입부 무드가 웨이크메이크처럼 느껴지는가?" ← 즉각 직관 판단
           → "힙하고 세련된 프로페셔널" + "다채로운 컬러 레인지" = 충족해야
           → "요즘 색조스러움" = 2026 트렌드 감각이어야 (올드한 느낌 금지)

3-2        "3초 Hook 와이어프레임이 우리 브랜드에 맞는가?" 확인
2-1        "우리 브랜드를 정확히 파악했구나" 확인

결정적 순간: 과제 시안을 펼쳤을 때 첫 1초의 감각적 반응
→ "이거다" vs "음... 좀 아쉬운데"
```
