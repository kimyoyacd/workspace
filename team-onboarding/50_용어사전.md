# 50. 용어 사전 — "이 단어가 대체 뭐야?"

> 클로드를 쓰다 보면 스킬·마크다운·깃허브·MCP 같은 낯선 단어가 튀어나온다. **하나도 몰라도 일은 시작할 수 있다.** 이 사전은 "겁먹지 말라"고 있는 것이다.
> 먼저 아래 **딱 3줄**만 보고, 필요할 때 해당 항목을 찾아 읽으면 된다.

> 1. **당장 알아야 할 건 거의 없다.** 챗에서 한국어로 부탁하는 것만으로 대부분의 일이 된다.
> 2. **아래 용어들은 "우리 팀 작업 환경에서 지나가다 보게 될 간판"** 이다. 뜻만 알면 안 무섭다.
> 3. **★ 표시 = 알면 편함 / (몰라도 됨) = 지금은 신경 끄기.**

---

## A. 파일·문서 형식

### 마크다운 (.md) ★
- **한 줄**: 서식 있는 글을 아주 간단한 기호로 쓰는 **가벼운 문서 형식**. (워드의 초경량 버전)
- **왜 쓰나**: 어디서나 열리고, 누가 뭘 고쳤는지 추적하기 좋아서 협업 문서로 애용된다. 이 온보딩 문서들(`00_기초개념.md` 등)이 전부 마크다운이다.
- **알아둘 기호**: `#` 제목, `**굵게**`, `-` 목록, `|`로 표. 이게 전부다.
- **비유**: 메모장인데 제목·굵게·목록이 되는 메모장.

### HTML (몰라도 됨)
- **한 줄**: 웹페이지를 만드는 형식. `onboarding.html`이 그 예 — 브라우저에서 보기 좋게 열린다.
- 우리는 결과물을 "보기 좋게 전달"할 때 클로드가 대신 만들어준다. 직접 손으로 쓸 일은 없다.

---

## B. 깃허브(GitHub) 생태계 — 팀 공동 창고

> **먼저 안심**: **대부분의 팀원은 깃허브를 직접 안 만져도 된다.** "우리 결과물이 여기에 보관·버전관리되고, 링크로 공유된다" 정도만 알면 충분하다.

### 깃허브 (GitHub) ★
- **한 줄**: 문서·코드를 팀이 함께 보관하고, **누가 언제 뭘 바꿨는지 전부 기록**하는 클라우드 창고.
- **비유**: 구글 드라이브 + "변경 이력 되감기" 기능이 강력하게 붙은 것.

### 저장소 (Repository / Repo)
- **한 줄**: 하나의 프로젝트 폴더 전체. 우리의 `workspace`가 저장소 하나다.

### 커밋 (Commit)
- **한 줄**: "저장 + 뭘 바꿨는지 한 줄 메모"를 한 번 찍는 것. 변경의 **스냅샷(체크포인트)**.
- **비유**: 게임 세이브. 언제든 그 지점으로 돌아갈 수 있다.

### 브랜치 (Branch)
- **한 줄**: 원본을 안 건드리고 **따로 떼어 작업하는 사본**. 실험하다 잘못돼도 원본은 안전하다.

### PR (Pull Request, 풀 리퀘스트)
- **한 줄**: "내 브랜치에서 바꾼 걸 원본에 **합쳐도 될까요?**" 하고 검토·병합을 요청하는 것.
- **비유**: "이 수정본 컨펌해 주세요" 하고 올리는 결재 상신.

## C. 클로드 확장 개념 — 자주 듣게 될 단어

### 프로젝트 / 세션 / 모델 / 컨텍스트 / 토큰
- 이건 [`00_기초개념.md`](./00_기초개념.md)에서 자세히 다룬다. 한 줄 재요약:
  - **세션** = 대화 하나 = 맥락 하나 (옆 대화는 못 봄)
  - **프로젝트** = 관련 대화·파일을 묶는 폴더
  - **모델** = 두뇌 등급 (Sonnet 기본 / 어려우면 Opus)
  - **컨텍스트** = 지금 들고 있는 대화 전체 / **토큰** = 글자 세는 단위

### 스킬 (Skill) ★
- **한 줄**: 특정 작업을 "이렇게 해라"라고 **미리 패키징해 둔 설명서 묶음**. 클로드가 필요할 때 알아서 꺼내 쓴다.
- **왜 좋나**: 매번 같은 지시를 길게 반복 안 해도 된다. 예: "슬라이드 만들기 스킬", "우리 브랜드 리포트 스타일 스킬"을 만들어두면, 그냥 "슬라이드 만들어줘" 하면 그 규칙대로 나온다.
- **비유**: 자주 쓰는 요리의 **레시피 카드**. 부를 때마다 재료·순서를 안 불러줘도 된다.

### 에이전트 (Agent) ★
- **한 줄**: 특정 역할에 특화된 **클로드 분신**. 부르면 그 관점으로 일한다.
- **예**: "견적 담당", "트렌드 리서치 담당", "시안 비평 담당" 같은 전문 분신.
- **스킬과 차이**: 스킬 = *작업 방법 설명서* / 에이전트 = *그 방법을 든 담당자*. (초기엔 이 차이까지 몰라도 됨)

### 아티팩트 (Artifact) ★
- **한 줄**: 클로드가 만든 결과물(웹페이지·문서 등)을 **링크로 바로 볼 수 있게 발행**한 것.
- 이 온보딩의 HTML 페이지가 아티팩트로 발행돼 링크로 공유된다.

## D. 연결·도구

### MCP / 커넥터 (Connector) ★
- **한 줄**: 클로드를 **외부 도구에 연결하는 다리**. 연결하면 클로드가 그 안의 자료를 읽거나 쓸 수 있다.
- **예**: 노션(Notion), 지메일(Gmail), 구글 드라이브, 피그마(Figma), 캘린더 등.
- **비유**: 클로드에게 "노션 열쇠", "드라이브 열쇠"를 쥐여주는 것. (연결은 보통 관리자가 세팅)
- **주의**: 연결된 도구의 자료는 곧 기밀·PII일 수 있다 → 다룰 때 보안 규칙(`30`, `99`) 준수.

### Claude Code / 터미널·CLI (몰라도 됨)
- **한 줄**: 검은 화면에 명령으로 파일·자동화를 직접 다루는 **개발자용 환경**.
- **대부분의 팀원은 지금 몰라도 된다.** 파일을 대량으로 만들거나 자동화가 필요해질 때 배우면 된다.

## E. "몰라도 됨 / 알면 좋음" 한눈에

| 지금 몰라도 되는 것 | 뜻만 알면 편한 것 | 나중에(필요할 때) |
|---|---|---|
| 커밋·브랜치·PR의 원리 | 마크다운(.md)이 뭔지 | Claude Code·터미널 |
| HTML 문법 | 깃허브 = 팀 창고 | 스킬·에이전트 직접 만들기 |
| MCP의 내부 작동 | 스킬 = 레시피 카드 | MCP 커넥터 세팅 |
| — | 에이전트 = 전문 담당 분신 | — |
| — | 아티팩트 = 링크로 보는 결과물 | — |

> **핵심**: 위 단어가 대화나 문서에 나와도 **당황하지 말 것.** "아 그거 그런 뜻이었지" 하고 넘어가면 된다. 정말 필요해지는 순간, 그때 이 사전을 다시 펴면 된다.
