# 30. 그룹 사용 vs 개인 사용 — 운영 규칙

> 핵심 질문: "무엇을 **각자** 하고, 무엇을 **팀 공유**로 관리하나." 그리고 "세션·프로젝트를 어떻게 정리하나."

---

## 1. 경계 나누기 — 개인 영역 vs 그룹 영역

| | **개인 영역** | **그룹(공유) 영역** |
|---|---|---|
| 무엇 | 각자의 실무 대화, 초안, 실험, 개인 프롬프트 | 팀 공통 규칙·템플릿·검증된 프롬프트·클라이언트 프로젝트 노트 |
| 왜 | 자유롭게 빠르게 시도 | 노하우 축적·일관성·중복 제거 |
| 예 | "이 문구 5안 뽑아줘" 같은 일회성 대화 | "제안서 표준 골격", "견적 요청 문구 템플릿" |
| 관리 | 각자 알아서 (아래 세션 규칙만 지킴) | 공유 저장소/위키/프로젝트로 관리 |

**원칙**: 개인은 **속도**, 그룹은 **일관성**. 개인이 잘 쓴 게 반복되면 → **그룹으로 승격**한다.

---

## 2. 세션(대화) 관리 규칙 — 전원 공통

세션은 "맥락 하나"다(→ `00_기초개념.md`). 방치하면 대화가 수십 개로 엉킨다. 아래 3규칙만.

### 규칙 1 — 세션은 "일 단위"로 연다
- 한 세션 = 하나의 작업/주제. 클라이언트 A 시안 얘기하다 B 견적으로 넘어가면 **새 세션**.
- 잡담·실험은 "실험" 세션 하나로 몰아두고, 실무 세션과 섞지 않는다.

### 규칙 2 — 세션 이름을 규칙대로 짓는다
> **`[클라이언트]_[업무]_[날짜]`** 형식 권장.
- 예: `코웨이_상세페이지_0720`, `내부_배너아이디어_0721`
- 이름만 봐도 뭔지 알면 나중에 찾기 쉽다.

### 규칙 3 — 길어진 세션은 "결정"을 밖으로 뺀다
- 세션이 길어지면 앞부분이 **요약**되어 디테일이 흐려질 수 있다.
- **확정된 방향·수치·문구는 그때그때 문서(프로젝트 노트)로 복사**해 둔다.
- 완전히 새 국면이면, 핵심 맥락만 붙여넣고 **새 세션으로 리셋**.

---

## 3. 프로젝트(폴더) 관리 — 클라이언트/건 단위

**건별로 프로젝트를 하나씩** 만든다. 그 안에 관련 대화·파일·지침을 모은다.

- 네이밍: **`YYYYMM_클라이언트_업무`** (예: `202607_코웨이_상세페이지`)
- 각 프로젝트에 넣을 것: 클라이언트, 업무 범위, 일정, 핵심 결정 메모, 참고 링크
- 프로젝트 지침(고정 맥락)에 "이 건은 무엇을 목표로, 어떤 톤으로"를 적어두면 매 대화마다 다시 설명 안 해도 된다.

---

## 4. 공유 자산 관리 — 노하우를 팀 것으로

개인이 잘 쓴 프롬프트/흐름을 **팀 자산으로 적립**하는 자리를 하나 정한다(사내 위키, 공유 문서, 또는 이 저장소 폴더).

**적립 대상**
- 반복해서 쓰는 프롬프트 → **템플릿**으로 정리
- 자주 하는 검토·산정·정리 흐름 → **표준 절차**로 문서화
- (고급) 매번 부르는 흐름 → **에이전트/스킬**로 패키징

**적립 규칙 (간단하게)**
1. 잘 된 프롬프트를 발견하면 → 공유 저장소에 "제목 + 프롬프트 + 언제 쓰는지" 3줄로 붙인다.
2. 월 1회 정리 담당이 중복 병합·분류.
3. 새 팀원 온보딩 때 이 저장소를 함께 보여준다.

## 5. 계정·플랜 관점 (초기 세팅)

> 세부 값은 `99_우리팀_규칙.md`에 채운다. 여기선 결정할 항목만.

- **어떤 플랜으로 몇 명?** — 사용량·예산에 맞춰 결정. 헤비 유저(코드/자동화)와 라이트 유저(챗) 구분.
- **공유 vs 개인 계정?** — 원칙: **개인별 계정**(보안·책임 추적). 공용 계정은 지양.
- **접근 범위·보안**: 클라이언트 기밀·개인정보(PII)는 입력 전 **마스킹/최소화**. 외부 반출 금지 자료 규칙을 명문화.
- **비용 관리**: 모델 3줄 규칙이 곧 비용 규칙(불필요하게 Opus/Fable 남발 금지).

## 6. 그룹으로 할 때 흔한 함정

| 함정 | 처방 |
|---|---|
| 각자 제각각 프롬프트 → 결과 편차 | 검증된 프롬프트를 공유 템플릿으로 통일 |
| 세션 수십 개로 엉킴 | 세션 네이밍 규칙 + 건별 프로젝트 |
| 같은 걸 여러 명이 반복 | 공유 저장소에 먼저 검색하는 습관 |
| 기밀·PII를 무심코 입력 | 입력 전 마스킹 규칙 + 검수 게이트 |
| "누가 뭘 잘 쓰는지" 모름 | 주간 15분 공유로 노하우 순환 |
