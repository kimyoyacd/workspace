# 팀 클로드(Claude) 온보딩 & 운영 계획

> **목적**: 팀원 전원을 **같은 출발선**에 세운다. 클로드를 처음 쓰는 사람도, Sonnet·Fable 같은 모델을 몰라도 어려움 없이 실무에 투입할 수 있게 한다.
> **성격**: 이 폴더는 **자체 완결형(portable)** 이다. 특정 회사·프로젝트 자산에 의존하지 않으므로, 폴더째 복사해 다른 팀·다른 워크스페이스로 그대로 **이식**할 수 있다. 회사 고유 규칙만 `99_우리팀_규칙.md`에 채우면 된다.

## 3단계 롤아웃 로드맵

전원을 같은 수준으로 끌어올리는 것은 **한 번의 교육이 아니라 3단계**로 설계한다.

### 1단계 — 베이스라인 통일 (1주차)
- **목표**: 전원이 개념을 이해하고, 혼자서 기본 대화·모델 선택·세션 관리를 할 수 있다.
- **활동**:
  1. `00_기초개념.md` 각자 읽기 (15분)
  2. 90분 공통 온보딩 세션 진행 (`20_레벨맞추기_커리큘럼.md`)
  3. `10_치트시트.md` 인쇄/핀 고정
  4. 이수 체크리스트 8항목 통과
- **산출**: 전원 체크리스트 통과 = "같은 출발선" 달성.

### 2단계 — 실무 적용 (2~4주차)
- **목표**: 각자 자기 업무에 클로드를 붙인다.
- **활동**:
  - `40_역할별_레시피.md`에서 자기 직군 레시피 3개를 실제 업무에 적용
  - 주 1회 "이번 주 클로드로 뭘 아꼈나" 15분 공유 (라이트닝 토크)
  - 잘 된 프롬프트는 공유 저장소(`30_그룹_개인_운영.md` 참조)에 적립
- **산출**: 1인당 최소 3개의 검증된 실무 워크플로.

### 3단계 — 팀 표준화 (2개월차~)
- **목표**: 개인 노하우를 팀 자산으로 전환.
- **활동**:
  - 반복되는 프롬프트 → **공유 프로젝트/템플릿**으로 승격
  - 자주 쓰는 검수·산정 흐름 → **에이전트/스킬**로 패키징 (선택)
  - `99_우리팀_규칙.md` 업데이트
- **산출**: "개인이 잘 쓰는 팀" → "팀 시스템으로 잘 쓰는 팀".

## 성공 기준 (어떻게 "같은 수준"을 확인하나)

| 시점 | 확인 지표 |
|---|---|
| 1주차 말 | 전원 이수 체크리스트 8/8 통과 |
| 4주차 말 | 1인당 검증된 워크플로 ≥ 3개, 주간 공유 4회 누적 |
| 2개월차 | 공유 템플릿/프로젝트 ≥ 5개, "혼자 막힐 때 물어볼 곳"이 명확 |

> 핵심 철학: **완벽한 프롬프트 엔지니어를 만들 필요 없다.** 전원이 "겁 없이 시도하고, 막히면 물어보고, 좋은 건 공유한다"는 **같은 습관**을 갖추는 것이 목표다.
