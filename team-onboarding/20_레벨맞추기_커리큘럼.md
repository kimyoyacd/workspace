# 20. 레벨 맞추기 — 공통 온보딩 커리큘럼

> 목표: **90분 한 번**으로 전원을 같은 출발선에. 강의가 아니라 **같이 손으로 해보는** 세션이다.

---

## 왜 "공통 90분"인가

사람마다 클로드 경험이 제각각이면 팀 대화가 안 통한다("세션이 뭐예요?" "그 모델 뭔데요?"). 그래서 **모두가 같은 개념·같은 용어·같은 기본 습관**을 갖추는 공통 세션을 딱 한 번 강하게 한다. 이후는 각자 실무에서 는다.

---

## 준비물 (진행자)
- 팀원 각자 클로드 계정/접근 (`99_우리팀_규칙.md` 참조)
- `00_기초개념.md`를 세션 **전날** 읽어오게 (15분 숙제)
- `10_치트시트.md` 인쇄본 1인 1장
- 화면 공유 가능한 회의실/온라인

---

## 90분 진행표

| 시간 | 세션 | 내용 |
|---|---|---|
| 0–10 | **개념 정렬** | `00_기초개념.md` 핵심 3가지 함께 확인: 접점 3개 / 세션=맥락 / 모델 3줄 규칙. 질문 받기. |
| 10–25 | **실습 1: 첫 대화** | 각자 챗에서 "맥락 먼저" 원칙으로 자기 업무 하나를 요청. 나쁜 요청 → 좋은 요청으로 고쳐보기. |
| 25–40 | **실습 2: 대화로 다듬기** | 방금 결과를 "더 짧게 / 이 톤으로 / 3안만" 왕복으로 다듬기. 예시 붙이기 체험. |
| 40–55 | **실습 3: 모델 바꿔보기** | 같은 요청을 Sonnet → Opus로 올려 차이 체감. "언제 올릴지" 감 잡기. |
| 55–70 | **실습 4: 프로젝트/세션 관리** | 클라이언트별 프로젝트 만들기. 주제 바뀌면 새 세션 여는 습관 체험. |
| 70–85 | **역할별 레시피** | `40_역할별_레시피.md`에서 자기 직군 레시피 1개를 실제로 돌려보기. |
| 85–90 | **마무리 & 다음 단계** | 체크리스트 배부. 2단계(실무 적용) 안내. 주간 공유 일정 공지. |

## 이수 체크리스트 (전원 8/8 = "같은 출발선" 달성)

각자 스스로 체크하고, 안 되는 항목은 진행자/동료가 옆에서 도와 통과시킨다.

- [ ] 챗·프로젝트·코드 3접점의 차이를 한 문장씩 설명할 수 있다
- [ ] "세션 = 맥락"을 이해하고, 주제가 바뀌면 새 세션을 연다
- [ ] 모델 3줄 규칙을 안다 (기본 Sonnet / 어려우면 Opus / 양 끝은 Haiku·Fable)
- [ ] 나쁜 요청을 "맥락 먼저 + 구체적으로" 좋은 요청으로 고칠 수 있다
- [ ] 결과를 대화로 다듬어 본다 (더 짧게/톤 변경/N안 남기기)
- [ ] 클라이언트/건별로 프로젝트를 하나 만들어 봤다
- [ ] 사실·숫자·PII·톤은 사람이 검수 후 발송한다는 원칙을 안다
- [ ] 막혔을 때 어디에 물어보는지 안다 (팀 공유 채널)

> 통과 기준은 "완벽"이 아니라 "혼자 겁 없이 시도할 수 있다"이다.
