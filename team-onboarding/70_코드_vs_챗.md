# 70. Claude Code vs 일반 챗 — 언제 뭘 써야 해?

> **핵심**: 둘 다 클로드인데, 어디로 들어가느냐에 따라 할 수 있는 일이 달라요. 팀 공유 기록이 필요하면 코드, 가볍고 빠르면 챗. 이 페이지 하나면 "뭘 쓸지" 결정돼요.

---

## 빠른 비교표

| | **일반 챗** (claude.ai) | **Claude Code** (Claude Code) |
|---|---|---|
| **뭐예요** | 대화창으로 즉시 시작 | 저장소와 파일을 다루는 작업실 |
| **장점** | 가볍고 빠르고 즉시 가능. 설치 없음. | 팀이 함께 볼 수 있는 **기록**. 파일 자동 생성. 에이전트 자동 로드. |
| **단점** | 대화만 기록됨. 팀 공유 어려움. 에이전트 안 됨. | 저장소 필요. 초기 설정 한 번 더. |
| **비용** | 낮음 | 약간 높음 (파일 작업) |

---

## 이렇게 쓰면 돼요 (결정 플로우)

### 🟢 일반 챗을 쓰세요

**"내 일은 빠르고 혼자 하는 거"** 라면:

- ✅ "제목 5개 뽑아줘"
- ✅ "이 글을 100자로 요약해줘"
- ✅ "배너 톤 다듬어줘"
- ✅ "아이디어 10개 발산해줘"
- ✅ 일회성 질문·자료찾기·검토

**이 일은 챗의 프로젝트 폴더에 정리하면 돼요:**
- 클라이언트/건별 프로젝트 만들기 (폴더처럼 대화를 모음)
- 같은 프로젝트 안에서 관련 대화 이어가기
- 확정된 내용은 메모/문서로 따로 저장

---

### 🔴 Claude Code를 써야 해요

**"팀이 함께 봐야 할 기록"이 필요하면:**

- ✅ **팀 공유 템플릿·규칙** 문서 작성 (e.g., 제안서 표준 골격, 견적 양식)
- ✅ **클라이언트별 프로젝트 노트** 만들기 (`.claude/projects/YYYYMM_클라이언트.md`)
- ✅ **반복 흐름 자동화** (e.g., 월간 리포트 생성, 분석 표 자동 정리)
- ✅ **팀 에이전트/스킬** 사용 (e.g., 마케터 에이전트로 시장조사 → 발산 자동 연결)
- ✅ 매번 물어보는 작업을 **한 번 정리해서 재사용**

**코드에서 나온 산출:**
- 실제 `.md` 파일로 저장소에 남음 = 팀 누구나 볼 수 있음
- 같은 저장소의 팀원이 저장소를 열면 자동으로 우리 에이전트·스킬을 봄
- 에이전트는 이 폴더에서만 자동 작동 (챗에선 수동 붙여넣기 필요)

## 구체적 예시

### 예 1️⃣: 배너 문구 작성

```
요청: "20대 여성 여름 세일 배너, 15자 안으로 5개 뽑아줘"

→ 어디서? 일반 챗 (○)
→ 왜? 일회성. 팀 공유 불필요. 빠르게 5개 나오면 끝.
→ 정리: 챗의 "[클라이언트]_배너" 프로젝트에 결과 메모
```

### 예 2️⃣: 팀 공통 배너 가이드

```
요청: "우리 팀 배너의 톤·규칙을 정리해서 표준화하고, 
       매번 이 가이드를 쓰도록 하고 싶어"

→ 어디서? Claude Code (●)
→ 왜? 팀이 반복해서 쓸 규칙. 파일로 남아야 함. 
      프로젝트별로 다시 설명 안 하려고.
→ 정리: `.claude/library/prompts/배너가이드.md` 생성
       프로젝트 지침에 링크
```

### 예 3️⃣: 경쟁사 분석 (마케터 에이전트)

```
요청: "시장조사 해줘 — [카테고리], 
       경쟁 포지셔닝 맵으로"

→ 어디서? Claude Code (●)
→ 왜? 마케터 에이전트는 Code의 자동 로드만 작동
      결과가 표·맵 → 파일로 남아야 함
      팀 마케터 모두 이 흐름 써야 함
→ 정리: 저장소를 Claude Code로 열고 "시장조사"라고 부르면
       자동으로 마케터 에이전트 시작
```

### 예 4️⃣: 프로젝트 진행 정리

```
요청: "지난 일주일 진행현황과 
      차주 계획을 표로 정리해줘"

→ 어디서? Claude Code (●)
→ 왜? 팀이 매주 봐야 할 기록
      `.claude/projects/진행현황_0721.md` 같은 파일로 만들어둬야 함
      다음주 업데이트할 때도 같은 파일 수정
```

## 팀장(실장) 초기 세팅 체크리스트

### 1단계: 저장소 준비 (Claude Code)
- [ ] `team-onboarding` 폴더를 팀 저장소에 복사
- [ ] `99_우리팀_규칙.md` 작성 (팀 규칙·계정·보안)
- [ ] `.claude/` 폴더 구조 확인
  - `.claude/agents/` (마케터 등 팀 에이전트)
  - `.claude/projects/` (팀 프로젝트 노트)
  - `.claude/library/` (공유 템플릿·자산)

### 2단계: 팀 게이트웨이 (Claude Code)
- [ ] Claude Code로 저장소를 열기 (팀이 쓸 주요 진입점)
- [ ] 팀원 계정 생성 & 저장소 접근 권한 부여
- [ ] `00_기초개념.md` 안내: "에이전트·팀 프로젝트는 Code에서만 작동"

### 3단계: 가벼운 작업 (일반 챗)
- [ ] 계정 이메일 공유 (chatgpt.com/claude.ai 설정)
- [ ] 모델 3줄 규칙 안내 (Sonnet 기본 → Opus 필요시)
- [ ] 프로젝트 폴더 사용법 (클라이언트별 폴더)

### 4단계: 온보딩 진행
- [ ] 90분 공통 온보딩 (챗 + 코드 양쪽 기초)
- [ ] 팀원이 "일회성은 챗, 팀 기록은 코드"를 체화할 때까지 반복 안내

## 헷갈릴 때 ("이건 어디서?")

| 상황 | 결정 |
|---|---|
| "팀이 매번 이 흐름을 쓸 거야" | → Code (한 번 정리 후 재사용) |
| "이건 내 일하나? 팀 일하나?" | → 내 일만 쓰면 Chat, 팀이 봐야 하면 Code |
| "파일로 남겨야 하나?" | → Yes → Code, No → Chat |
| "팀 에이전트/스킬을 써야 하나" | → Code만 가능. Chat에선 수동 붙여넣기 |
| "빨리 한 번만 하면 되나" | → Chat |
| "내년에도 같은 양식을 쓸 거야" | → Code (자산화) |
