# 00. 기초 — 처음이라도 15분이면 이해돼요

> 이 글 하나면 "클로드로 일한다"는 게 무슨 뜻인지, 어디서·어떻게·무슨 모델로 쓰는지 감이 잡혀요. 어려운 말은 최대한 빼고, 쉬운 비유로 설명해요.

---

## 1. 클로드가 뭐예요?

클로드(Claude)는 앤트로픽이 만든 **AI 도우미**예요. 글쓰기·자료 찾기·기획·검토·정리까지, "머리랑 손이 필요한 일"을 같이 해주는 **일 파트너**라고 보면 돼요.

핵심은 딱 하나예요. **말로 시키면 돼요.** 어려운 명령어를 외울 필요 없어요. 옆자리 동료에게 부탁하듯이, 그냥 우리말로 설명하면 돼요.

---

## 2. 어디서 쓰나 — 쓰는 곳 3가지 (여기가 제일 헷갈려요)

같은 클로드인데 **들어가는 문**이 세 개예요. 문마다 잘하는 일이 달라요.

| 쓰는 곳 | 한 줄 설명 | 이럴 때 써요 |
|---|---|---|
| **① 챗 (claude.ai)** | 대화창으로 쓰는 기본 방법 | 아이디어·글쓰기·자료 찾기·요약·시안 봐주기 |
| **② 프로젝트 (챗 안의 기능)** | 한 가지 일에 필요한 대화·파일을 **한곳에 모으는** 폴더 | "이 고객 일은 여기서만" — 얘기를 계속 이어가고 싶을 때 |
| **③ 코드 (Claude Code)** | 파일·자동화를 직접 다루는 개발자용 화면 | 진짜 파일 만들기/고치기, 반복 작업 자동으로, 문서 대량 생성 |

### "챗"과 "코드"가 뭐가 달라요?
- **챗** = 대화창 하나예요. 말로 시키면 클로드가 **글·아이디어·요약**으로 답해줘요. 파일을 "만들어 보여줄" 수는 있지만, 내 컴퓨터 폴더를 직접 건드리진 않아요.
- **코드** = 작업실이에요. 클로드가 **진짜 파일을 만들고 고치고 정리**해요. 이 온보딩 폴더 같은 곳이 코드가 일하는 자리예요.

> **처음엔 이렇게:** 대부분은 **①챗 + ②프로젝트**로 시작하면 돼요. 이것만으로 자료 찾기·기획·글쓰기·검토의 대부분이 돼요. **③코드**는 파일을 많이 만들거나 자동화가 필요한 사람만 나중에 배우면 돼요.

---

## 3. "세션"이 뭐예요 — 제일 중요한 개념 하나

**세션 = 대화 하나 = 기억 하나예요.**

새 대화를 열면 클로드는 **그 대화에서 나눈 얘기만** 기억해요. 옆 대화는 몰라요. 사람으로 치면, **이 회의실에서 한 얘기만 기억하는** 상태예요.

이걸 알면 3가지가 풀려요.

1. **주제가 바뀌면 새 대화를 열어요.** 고객 A 얘기하다 고객 B로 넘어가면 새 대화. 안 그러면 얘기가 섞여서 답이 흐려져요.
2. **한 대화가 너무 길어지면** 클로드가 앞부분을 **요약**해서 이어가요. 그러다 보면 세세한 게 흐려질 수 있어요. 그러니 중요하게 정한 건 중간중간 문서에 적어둬요.
3. **다시 이어서 하려면** 같은 대화로 돌아가거나, 중요한 내용을 붙여넣고 새로 시작하면 돼요.

### 컨텍스트랑 토큰 — 딱 이만큼만
- **컨텍스트**: 클로드가 지금 "머릿속에 들고 있는" 대화 전부예요. 여기 없는 건 몰라요.
- **토큰**: 글자를 세는 단위예요(대략 우리말 1~2글자 = 1토큰). 클로드가 한 번에 들고 있을 수 있는 양엔 한계가 있어요 = **칠판 크기**예요. 꽉 차면 오래된 걸 지우거나 요약해요.
- **팁**: 상관없는 잡담으로 대화를 채우지 마세요. 칠판은 **지금 일에 필요한 얘기**로만 채워요.

---

## 4. 모델 — Sonnet? Opus? Haiku? Fable? (딱 3줄)

클로드에는 **모델(두뇌 등급)** 이 여러 개 있어요. 똑똑함·속도·비용이 조금씩 달라요. 이름을 다 외울 필요 없어요. **아래 3줄만 기억하면 돼요.**

> 1. **보통은 Sonnet.** 매일 하는 일(글·기획·자료찾기·검토)엔 이게 빠르고 충분해요.
> 2. **어려우면 Opus.** 복잡한 생각, 긴 제안서, 까다로운 판단일 때 올려요.
> 3. **빠르고 간단하면 Haiku, 아주 어렵고 오래 걸리면 Fable.**

### 모델 한눈에

| 모델 | 성격 | 언제 써요 |
|---|---|---|
| **Haiku** (4.5) | 가장 빠르고 값쌈 | 간단·반복·빠르게 (분류, 짧은 요약, 초안 여러 개) |
| **Sonnet** (5) | 속도·똑똑함 균형 — **기본** | 매일 하는 일 거의 다: 기획·글쓰기·자료찾기·시안 봐주기·코딩 |
| **Opus** (4.8) | 가장 똑똑한 주력, 혼자서도 척척 | 복잡한 전략, 긴 제안서, 오래 걸리는 자동 작업, 어려운 판단 |
| **Fable** (5) | 최상위 — 아주 어렵고 오래 걸리는 일 | 며칠짜리 난제, 최고 난도. **값이 비싸요 → 꼭 필요할 때만** |

**이렇게 해요**: 헷갈리면 **Sonnet으로 시작**하고, 답이 얕거나 막히면 **"이거 Opus로 다시 해줘"** 하고 올려요. 올리는 건 쉬워요.

> **비유**: Haiku = 인턴(손이 빠름), Sonnet = 실무 담당(대부분의 일), Opus = 선배(어려운 판단), Fable = 특급 전문가(최고 난도, 몸값 비쌈).

*모델 이름은 바뀔 수 있어요. 새 이름이 보이면 "더 좋은 버전이구나" 하고, 이 3줄 규칙은 그대로 쓰면 돼요.*

---

## 5. 클로드 잘 쓰는 5가지 습관 (모두 공통)

1. **상황부터 알려줘요.** "누구를 위한, 무슨 일인지"를 먼저 말하면 답이 확 좋아져요.
   - 나쁨: "배너 문구 뽑아줘"
   - 좋음: "20대 여성 화장품 여름 세일 배너예요. 튀되 과하지 않게, 15자 안으로 5개 뽑아줘."
2. **한 번에 완성 기대하지 말고 대화로 다듬어요.** "더 짧게" "이 느낌으로" "3개만 남겨" — 주고받으며 좁혀가요.
3. **예시를 줘요.** 원하는 느낌의 샘플 하나만 붙여도 잘 맞춰줘요.
4. **주제가 바뀌면 새 대화.** (위 3번 참고)
5. **결과는 확인하고 써요.** 클로드는 빠르고 잘하지만, **사실·숫자·개인정보는 꼭 사람이 확인**해요. 특히 밖으로 나가는 건요.

## 6. 자주 하는 오해 바로잡기

| 오해 | 사실 |
|---|---|
| "명령어를 외워야 한다" | 아니에요. 우리말로 부탁하면 돼요. |
| "한 번 시키면 끝" | 대화로 다듬는 게 정석이에요. 주고받는 게 정상이에요. |
| "옆 대화 내용도 알겠지" | 몰라요. 대화 밖은 못 봐요. |
| "비싼 모델(Opus/Fable)을 항상 써야 잘 나온다" | 대부분 Sonnet이면 충분해요. 필요할 때만 올려요. |
| "AI가 썼으니 그대로 보낸다" | 사실·말투·개인정보는 사람이 확인하고 보내요. |
