# 00. 기초 개념 — 처음이라도 15분이면 이해

> 이 문서 하나만 읽으면 "클로드로 일한다"는 게 무슨 뜻인지, 어디서·어떻게·무슨 모델로 쓰는지 감이 잡힌다. 용어는 최소한만, 비유로 설명한다.

---

## 1. 클로드가 뭐야?

클로드(Claude)는 앤트로픽(Anthropic)이 만든 **AI 어시스턴트**다. 글쓰기·리서치·기획·검토·코드·자료 정리까지, "생각과 손이 필요한 일"을 같이 해주는 **디지털 실무 파트너**라고 생각하면 된다.

핵심은 하나: **말로 시키면 된다.** 명령어를 외울 필요 없이, 동료에게 부탁하듯 한국어로 설명하면 된다.

---

## 2. 어디서 쓰나 — 접점 3가지 (이게 제일 헷갈리는 부분)

같은 클로드지만 **들어가는 문**이 세 개다. 문마다 잘하는 일이 다르다.

| 접점 | 한 줄 정의 | 이럴 때 |
|---|---|---|
| **① 챗 (claude.ai)** | 브라우저/앱에서 대화창으로 쓰는 기본 방식 | 아이데이션, 글쓰기, 리서치, 자료 요약, 이미지 보고 피드백 |
| **② 프로젝트 (claude.ai 안의 기능)** | 관련 대화·파일·지침을 **한 폴더로 묶는** 공간 | "이 클라이언트 건은 여기서만" — 맥락을 계속 유지하고 싶을 때 |
| **③ 코드 (Claude Code)** | 파일·폴더·자동화를 직접 다루는 개발자용 환경 | 실제 파일 생성/수정, 반복 자동화, 저장소(repo) 작업, HTML/문서 대량 생성 |

### "챗 세션"과 "코드 세션"이 뭐가 다른가
- **챗 세션** = 대화창 하나. 말로 시키면 클로드가 **글·아이디어·요약**으로 답한다. 파일을 "만들어 보여줄" 수는 있어도 네 컴퓨터 폴더를 직접 건드리진 않는다.
- **코드 세션** = 작업실. 클로드가 **실제 파일을 만들고 고치고 정리**한다. 이 저장소(`team-onboarding` 폴더 같은)가 코드 세션이 일하는 공간이다.

> **팀 초기 권장**: 대부분의 팀원은 **①챗 + ②프로젝트**로 시작한다. 이것만으로 리서치·기획·글쓰기·검토의 80%가 해결된다. **③코드**는 파일을 대량으로 만들거나 자동화가 필요한 사람만 나중에 배운다.

---

## 3. "세션"이 뭐야 — 가장 중요한 개념 하나

**세션 = 하나의 대화 = 하나의 맥락(context).**

새 대화를 열면 클로드는 **그 대화 안에서 오간 내용만** 기억한다. 옆 대화에서 무슨 얘기를 했는지는 모른다. 사람으로 치면 **"이 회의실 안에서 나눈 이야기만 기억하는" 상태**다.

이걸 알면 3가지가 풀린다:

1. **주제가 바뀌면 새 세션을 열어라.** A클라이언트 얘기하다 B클라이언트로 넘어가면 새 대화. 안 그러면 맥락이 섞여 답이 흐려진다.
2. **한 세션이 너무 길어지면** 클로드가 앞부분을 **요약**해서 이어간다(컨텍스트 관리). 세부 디테일이 흐려질 수 있으니, 중요한 결정은 중간에 문서로 남겨라.
3. **다시 이어서 하려면** 같은 세션(대화)으로 돌아가거나, 핵심 맥락을 붙여넣고 새로 시작한다.

### 컨텍스트(맥락)와 토큰 — 딱 이만큼만
- **컨텍스트**: 클로드가 지금 "머릿속에 들고 있는" 대화 전체. 여기 없는 건 모른다.
- **토큰**: 글자를 세는 단위(대략 한국어 1~2글자 ≈ 1토큰). "컨텍스트 창"에는 담을 수 있는 토큰 양에 한계가 있다 = **회의실 화이트보드 크기**. 다 차면 오래된 걸 지우거나 요약한다.
- **실무 팁**: 관련 없는 잡담으로 세션을 채우지 마라. 화이트보드는 **지금 일에 필요한 것**으로 채운다.

## 4. 모델 — Sonnet? Opus? Haiku? Fable? (3줄 규칙)

클로드에는 여러 **모델(두뇌 등급)** 이 있다. 똑똑함·속도·비용이 트레이드오프다. 이름을 다 외울 필요 없다. **아래 3줄만 기억하면 된다.**

> 1. **기본은 Sonnet.** 대부분의 일상 실무(글·기획·리서치·검토)는 Sonnet이 빠르고 충분하다.
> 2. **어려우면 Opus.** 복잡한 추론, 긴 전략 문서, 까다로운 판단이 필요하면 Opus로 올린다.
> 3. **급하고 단순하면 Haiku, 최고난도 장기전이면 Fable.**

### 모델 한눈에

| 모델 | 성격 | 언제 |
|---|---|---|
| **Haiku** (4.5) | 가장 빠르고 저렴 | 단순·반복·빠른 처리 (분류, 짧은 요약, 초안 대량 생성) |
| **Sonnet** (5) | 속도·지능 균형 — **팀 기본값** | 일상 실무 전반: 기획·글쓰기·리서치·시안 피드백·코딩 |
| **Opus** (4.8) | 가장 똑똑한 주력급, 자율성 높음 | 복잡한 전략, 긴 제안서, 장시간 자동 작업, 어려운 판단 |
| **Fable** (5) | 최상위 — 가장 어려운 추론·초장기 작업 | 며칠짜리 난제, 최고 난도 리서치. **비용 높음 → 꼭 필요할 때만** |

**실전 규칙**: 헷갈리면 **Sonnet으로 시작**해서, 답이 얕거나 막히면 **"이거 Opus로 다시 해줘"** 하고 올린다. 위로 올리는 건 쉽다.

> **비유**: Haiku = 인턴(빠른 손), Sonnet = 실무 담당(대부분의 일), Opus = 시니어(어려운 판단), Fable = 특급 전문가(최고 난도, 몸값 비쌈).

*모델 라인업과 이름은 갱신될 수 있다. 새 이름이 보이면 "더 최신/상위 버전"으로 이해하고, 위 3줄 규칙(기본 Sonnet / 어려우면 Opus / 단순·최고난도는 양 끝)은 그대로 적용하면 된다.*

## 5. 클로드를 잘 부리는 5가지 습관 (전원 공통)

1. **맥락을 먼저 준다.** "누구를 위한, 무슨 목적의 일인지"를 앞에 깔면 결과가 확 좋아진다.
   - 나쁨: "배너 문구 뽑아줘"
   - 좋음: "20대 여성 타깃 화장품 여름 세일 배너다. 톡톡 튀되 과하지 않은 문구 5개, 각 15자 이내."
2. **한 번에 완성 기대하지 말고 대화로 다듬는다.** "더 짧게" "이 톤으로" "3안만 남기고" — 왕복하며 좁혀간다.
3. **예시를 준다.** 원하는 톤/형식의 샘플 하나를 붙이면 적중률이 오른다.
4. **주제 바뀌면 새 세션.** (위 3번 참조)
5. **결과는 검토하고 쓴다.** 클로드는 빠르고 유능하지만 **사실·숫자·인용은 반드시 사람이 확인**한다. 특히 외부 발송물.

## 6. 자주 하는 오해 바로잡기

| 오해 | 사실 |
|---|---|
| "명령어를 외워야 한다" | 아니다. 한국어로 부탁하면 된다. |
| "한 번 시키면 끝" | 대화로 다듬는 게 정석. 왕복이 정상이다. |
| "옆 대화 내용도 알겠지" | 모른다. 세션 밖은 못 본다. |
| "비싼 모델(Opus/Fable)을 항상 써야 잘 나온다" | 대부분 Sonnet이면 충분. 필요할 때만 올린다. |
| "AI가 썼으니 그대로 발송" | 사실·톤·PII는 사람이 검수 후 발송. |
