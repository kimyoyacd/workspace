# 60. 에이전트 팀에 공유하기 — 예: 마케터가 "시장조사 에이전트" 쓰게 하기

> "에이전트를 팀이 쓰게 하려면 md 파일이랑 에이전트를 옮기면 되나?" → **거의 맞아요.** 에이전트는 곧 `.md` 파일이에요. 딱 두 가지만 챙기면 됩니다: ① 그 에이전트가 **딸고 다니는 파일**도 같이, ② 팀이 **어떻게 접근하는지**에 맞춰 위치 정하기.
> 여기서는 **마케터 팀원이 시장조사 에이전트(`market-research`)를 쓰는 경우**로 설명해요. 다른 에이전트도 방법은 똑같아요.

---

## 1. 먼저, 에이전트가 뭐예요?

특정 일만 잘하는 **클로드 분신**이에요. 부르면 그 시선으로 일해요. (자세히는 [`50_용어사전.md`](./50_용어사전.md))

- **`market-research`** = 마케터를 받쳐주는 **시장조사 분신**. 경쟁사를 모아 표로 정리하고, XY 포지셔닝 맵을 그려서 "우리가 노릴 빈자리"를 찾아줘요.

---

## 2. 옮길 때 딱 2개 (이 에이전트가 딸고 다니는 것)

시장조사 에이전트는 혼자 완결되지 않고 **참고 파일 하나**를 같이 봐요. 그래서 **세트로 2개**를 옮겨요.

```
.claude/agents/market-research.md          ← 에이전트 본체
.claude/library/prompts/wide-research.md   ← 에이전트가 참고하는 상세 사양
```

> 도구(웹 검색 등)는 클로드에 기본으로 있어서 따로 챙길 게 없어요.
> **핵심**: 에이전트를 옮길 땐 "본체 md + 그 안에서 언급하는 참고 파일"을 같이 옮긴다. 이게 전부예요.

---

## 3. 마케터는 이렇게 써요 (실제 쓰는 법)

마케터 팀원이 대화창에서 이렇게 부르면 돼요:

```
시장조사 해줘 — [우리 카테고리/제품군], 경쟁 브랜드 위주로.
```
또는
```
경쟁사 분석해줘. [카테고리]에서 최근 1년 활동 중인 브랜드로 포지셔닝 맵 그려줘.
```

**그러면 이런 게 나와요:**
1. 조사 범위 (카테고리·브랜드 수·기준 시점)
2. 브랜드 분석 표 (이름·소재지·타깃·기능·**출처 URL**)
3. 브랜드를 3그룹으로 묶은 공통 관점
4. **XY 포지셔닝 맵** (예: 감성↔기능 / 대중↔프리미엄)
5. **화이트스페이스** = 아무도 없는 빈자리 = 우리가 노릴 곳
6. 이 구도가 캠페인·기획에 주는 한 줄 함의

**마케터가 이걸로 하는 일**: 캠페인 방향 잡기, 우리 포지셔닝 정하기, 경쟁 대응 논리 만들기. "감이 아니라 출처 있는 근거"로 시작할 수 있어요.

> 팁: "빠르게 5개만"이라고 하면 짧게, "제대로 30개로"라고 하면 폭넓게 조사해요. 필요한 만큼만 시켜요.

---

## 4. 공유하는 3가지 방법 (팀이 어떻게 쓰냐에 따라)

### 🔴 핵심: **에이전트는 Claude Code에서만 자동 작동해요**

일반 챗(claude.ai)에는 `.claude/agents/` 자동 로드 기능이 없어요. 대신 수동으로 붙여넣어야 해요.

| 팀이 쓰는 방식 | 해야 할 일 | 에이전트 작동? |
|---|---|---|
| **A. 같은 저장소 + Claude Code** ⭐ **추천** | 팀원이 이 저장소로 Claude Code 세션을 열면, "시장조사"라고 부르는 순간 자동으로 에이전트 시작. 2개 파일도 자동 로드. | ✅ 자동 작동 |
| **B. 다른 팀/워크스페이스로 이식** (Claude Code) | **2개 파일을 같은 폴더 구조 그대로 복사**. `.claude/agents/…`, `.claude/library/prompts/…` 경로만 맞추면 끝. | ✅ 자동 작동 |
| **C. 일반 챗** (claude.ai, Claude Code 아님) | md 에이전트가 **자동으로 안 불려요.** 대신: 프로젝트 지침에 md 내용 붙여넣기 OR 대화 시작 때 프롬프트로 붙여넣기 | ❌ 수동만 가능 |

> **팀 권장**: **A번 (Claude Code + 같은 저장소)** 를 써요. 자동 로드 + 파일 기록 + 팀 공유가 모두 가능.

---

## 5. 옮긴 뒤 확인 & 주의

**확인** (A, B 방식만 적용 — Claude Code)
- 팀원이 Claude Code에서 저장소를 열고 "시장조사 해줘"라고 불렀을 때 출처 있는 표 + 포지셔닝 맵이 나오면 성공.
- 일반 챗에선 자동 작동 안 함. (C 방식 사용 중이면 OK)

**주의 (보안)**: 
- 이 에이전트는 웹에서 자료를 찾아와요. 조사 내용에 **미공개 클라이언트 정보·개인정보**를 넣지 않도록 해요. 
- (보안 규칙 → [`30_그룹_개인_운영.md`](./30_그룹_개인_운영.md), [`99_우리팀_규칙.md`](./99_우리팀_규칙.md))

**중요**: 
- **A·B 방식 (Claude Code)** 쓰세요. 자동화·팀 기록·파일 공유가 모두 가능.
- **C 방식 (일반 챗)** 은 에이전트를 매번 수동으로 붙여넣어야 하니까, 팀의 반복 흐름엔 맞지 않음.

---

## 6. 다른 에이전트도 똑같아요 (일반 규칙)

어떤 에이전트든 공유·이식 방법은 같아요:

1. **본체 찾기** — `.claude/agents/[에이전트이름].md`
2. **딸린 파일 찾기** — 본체 안에서 언급하는 `.claude/library/…` 파일이 있으면 같이
3. **옮기기** — 같은 폴더 구조로 복사 (또는 같은 저장소면 이미 공유됨)
4. **확인** — 트리거 단어로 불러서 잘 뜨는지 테스트

> 이게 온보딩 3단계(팀 표준화)의 핵심이에요: **개인이 잘 쓰는 분신 → 팀 모두의 분신으로.**
