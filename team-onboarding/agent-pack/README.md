> RFP 분석 → 시장조사 → 와이드 리서치 → 디자인 리서치(트렌드·레퍼런스) → 발산까지

## 설치 — 4단계 (Claude Code 저장소 기준)

1. **CLAUDE.md 복사** — `CLAUDE.md` → 새 저장소 루트(`/CLAUDE.md`)
   이게 있어야 Claude가 조직구조(라우팅)를 인식해요.
2. **에이전트·스킬 복사** — 폴더 구조 그대로:
   - `agents/` 내용물 → 새 저장소의 `.claude/agents/`
   - `library/prompts/wide-research.md` → 새 저장소의 `.claude/library/prompts/`
   - `skills/mx-deck-design/` → 새 저장소의 `.claude/skills/mx-deck-design/`
   (폴더가 없으면 만들면 돼요. 경로만 같으면 끝.)
3. **확인** — Claude Code로 그 저장소를 열고 **"시장조사 해줘 — [카테고리]"** 라고 불러보세요.
   출처 URL 붙은 브랜드 표 + 포지셔닝 맵이 나오면 성공.
4. **전체 흐름 테스트** — **"마케터 돌려 — [카테고리]"** (RFP 파일이 있으면 첨부).
   RFP 4분류 → 경쟁 구도 → 트렌드·레퍼런스 → 방향 후보까지 쭉 돌면 완료.

> ⚠️ 에이전트는 **Claude Code에서만 자동 작동**해요. 일반 챗(claude.ai)에서는 md 내용을
> 프로젝트 지침에 직접 붙여넣어야 해요. (자세히: [`../60_에이전트_공유하기.md`](../60_에이전트_공유하기.md))

### 이미 압축 파일로 등록 시도해서 오류가 났다면

ZIP을 Claude에 직접 주면 인식을 못 해요. 대신:
1. ZIP을 풀어서 각 파일을 **위 경로에 맞게 직접 배치**
2. 또는 Claude Code에서 "이 파일들을 `.claude/` 폴더 구조에 맞게 넣어줘" 라고 말하면서 파일 내용을 하나씩 붙여넣기

## 쓰는 법 — 4챕터 자동 체이닝

"마케터 돌려"면 아래 4챕터가 자동으로 순서대로 돌아요. 챕터 하나만 요청해도 돼요.

```
[0. RFP] → CH1. 현황 조사 → CH2. 진단 → CH3. 문제 정의 → CH4. 발산
              팩트 수집        구도·빈자리     문제 한 문장      방향 후보
```

| 이렇게 말하면 | 도는 챕터 |
|---|---|
| "RFP 분석" / "과업 분석" | 0 → CH1 자동 이어감 |
| "시장조사" / "경쟁사 분석해줘" / "현황 조사" | CH1 — 팩트 수집 |
| "브랜드 진단" / "포지셔닝 맵" / "빈자리 찾아줘" | CH2 — 구도·빈자리 |
| "문제 정의" / "진짜 문제가 뭐야" | CH3 — 핵심 문제 한 문장 |
| "트렌드 봐줘" / "레퍼런스 찾아줘" | CH1 중 해당 파트 |
| "브레인스토밍" / "아이디어 줘" | CH4 — 방향 후보 발산 |
| **"리서치 쭉 돌려줘" / "마케터 돌려"** | **전체 (0→CH1→CH2→CH3→CH4)** |

산출물은 마지막에 **한 장 요약**(핵심 문제 + 경쟁 구도 + 빈자리 + 트렌드 + 방향 후보 2~3개)으로 닫혀요.
