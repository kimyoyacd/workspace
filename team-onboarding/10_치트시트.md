# 10. 클로드 치트시트 (1장 요약)

> 인쇄하거나 위키에 붙여두세요. 막히면 여기부터 봐요.

---

## 어디서 쓰나
- **🟢 챗 (claude.ai)** — 아이디어·글·자료찾기·요약·시안 봐주기 (빠르고 가볍게) → **기본이 여기**
- **프로젝트** — 한 가지 일을 폴더로 모아 계속 이어가기 (고객마다 하나씩)
- **🔴 코드 (Claude Code)** — 팀 기록·파일·에이전트·자동화 (팀 공유할 때) → **"팀 규칙"은 여기**

> **핵심**: 일회성은 **챗** | 팀이 계속 쓸 규칙/에이전트/기록은 **코드**
> 자세히는 [`70_코드_vs_챗.md`](./70_코드_vs_챗.md)

## 세션 = 대화 하나 = 기억 하나
- 클로드는 **그 대화 안 얘기만** 기억해요. 옆 대화는 몰라요.
- **주제 바뀌면 → 새 대화.**
- 대화가 길어지면 앞부분이 요약돼요 → **중요하게 정한 건 문서에 적어둬요.**

## 모델 3줄 규칙
1. **보통은 Sonnet** (매일 하는 일)
2. **어려우면 Opus** (복잡한 전략·긴 문서·까다로운 판단)
3. **빠르고 간단하면 Haiku, 아주 어렵고 오래 걸리면 Fable**
> 헷갈리면 Sonnet으로 시작 → 답이 얕으면 "Opus로 다시 해줘"로 올려요.

| 모델 | 비유 | 언제 |
|---|---|---|
| Haiku | 인턴 | 빠르고 간단·반복 |
| **Sonnet** | 실무 담당 | **매일 하는 일 (기본)** |
| Opus | 선배 | 어려운 판단·긴 전략 |
| Fable | 특급 전문가 | 아주 어렵고 오래 걸리는 일 (비쌈) |

## 잘 쓰는 5습관
1. **상황 먼저** — 누구를 위한, 무슨 일인지 앞에 말하기
2. **대화로 다듬기** — "더 짧게" "이 느낌으로" 주고받기
3. **예시 주기** — 원하는 느낌 샘플 하나
4. **주제 바뀌면 새 대화**
5. **사실·숫자·개인정보·말투는 사람이 확인하고 보내기**

## 좋은 요청 vs 나쁜 요청
- ❌ "배너 문구 뽑아줘"
- ✅ "20대 여성 화장품 여름 세일 배너예요. 튀되 과하지 않게, 15자 안으로 5개 뽑아줘."

## 막힐 때
- 답이 얕다 → 모델을 Opus로 올리거나, 상황·예시를 더 준다
- 엉뚱하다 → 새 대화에서 더 자세히 부탁한다
- 그래도 막힌다 → **팀 공유 채널에 요청 그대로 올려 물어보기**
