# 10. 클로드 치트시트 (1장 요약)

> 인쇄하거나 사내 위키에 핀 고정. 막히면 여기부터 본다.

---

## 어디서 쓰나
- **챗 (claude.ai)** — 아이디어·글·리서치·요약·시안 피드백 → **여기서 시작**
- **프로젝트** — 한 건을 폴더로 묶어 맥락 유지 (클라이언트별로 하나씩)
- **코드 (Claude Code)** — 파일 실제 생성/수정·자동화 (필요한 사람만)

## 세션 = 대화 하나 = 맥락 하나
- 클로드는 **그 대화 안 내용만** 기억한다. 옆 대화는 모른다.
- **주제 바뀌면 → 새 세션.**
- 세션이 길어지면 앞부분이 요약된다 → **중요 결정은 문서로 남겨라.**

## 모델 3줄 규칙
1. **기본은 Sonnet** (일상 실무 전반)
2. **어려우면 Opus** (복잡한 전략·긴 문서·까다로운 판단)
3. **단순·급하면 Haiku, 최고난도 장기전이면 Fable**
> 헷갈리면 Sonnet으로 시작 → 얕으면 "Opus로 다시 해줘"로 올린다.

| 모델 | 비유 | 언제 |
|---|---|---|
| Haiku | 인턴 | 빠른·단순·반복 |
| **Sonnet** | 실무 담당 | **대부분의 일 (기본값)** |
| Opus | 시니어 | 어려운 판단·긴 전략 |
| Fable | 특급 전문가 | 최고 난도·초장기 (비쌈) |

## 잘 부리는 5습관
1. **맥락 먼저** — 누구를 위한, 무슨 목적인지 앞에 깔기
2. **대화로 다듬기** — "더 짧게" "이 톤으로" 왕복
3. **예시 주기** — 원하는 톤/형식 샘플 하나
4. **주제 바뀌면 새 세션**
5. **사실·숫자·PII·톤은 사람이 검수 후 발송**

## 좋은 요청 vs 나쁜 요청
- ❌ "배너 문구 뽑아줘"
- ✅ "20대 여성 화장품 여름세일 배너. 톡톡 튀되 과하지 않게, 15자 이내 5개."

## 막힐 때
- 답이 얕다 → 모델을 Opus로 올리거나, 맥락·예시를 더 준다
- 엉뚱하다 → 새 세션에서 요청을 더 구체적으로
- 그래도 막힌다 → **팀 공유 채널에 프롬프트째 올려 물어보기**
