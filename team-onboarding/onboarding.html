클로드 시작 가이드

클로드를 처음 써도, Sonnet·Fable 같은 이름을 몰라도 괜찮아요. 이 가이드만 보면 바로 일에 쓸 수 있어요. 우리 회사에만 맞춘 게 아니라서, 폴더째 복사해 다른 팀에도 그대로 쓸 수 있어요.

01

어디서 쓰나 — 쓰는 곳 3가지

같은 클로드지만 들어가는 문이 세 개예요. 문마다 잘하는 일이 달라요.

① 챗 · claude.ai

대화창

아이디어 · 글쓰기 · 자료 찾기 · 요약 · 시안 봐주기. 거의 모두가 여기서 시작해요.

② 프로젝트

한곳에 모으는 폴더

한 가지 일에 필요한 대화·파일을 한곳에 모아요. "이 고객 일은 여기서만" 해두면 계속 이어서 할 수 있어요.

③ 코드 · Claude Code

작업실

파일을 진짜로 만들고 고치고, 반복 작업을 자동으로. 필요한 사람만 나중에 배우면 돼요.

처음엔: ①챗 + ②프로젝트만으로 자료 찾기·기획·글쓰기·검토의 대부분이 돼요.

02

세션 = 대화 하나 = 기억 하나

이것만 알면 절반은 끝나요.

새 대화를 열면 클로드는 그 대화에서 나눈 얘기만 기억해요. 옆 대화는 몰라요. 사람으로 치면, 이 회의실에서 한 얘기만 기억하는 거예요.

세션 쓰는 3가지 규칙
  1. 주제가 바뀌면 새 대화. 고객 A 얘기하다 고객 B로 넘어가면 새 대화를 열어요. 안 그러면 얘기가 섞여서 답이 흐려져요.
  2. 대화 이름을 규칙대로. [고객]_[일]_[날짜] 이렇게 지으면 나중에 찾기 쉬워요.
  3. 대화가 길면 중요한 건 밖에 적어둬요. 대화가 길어지면 앞부분이 요약돼서 세세한 게 흐려질 수 있어요. 정한 방향·숫자·문구는 그때그때 문서에 옮겨 적어요.

컨텍스트 = 클로드가 지금 들고 있는 대화 전부(칠판 같아요). 토큰 = 글자를 세는 단위. 칠판이 꽉 차면 오래된 걸 지워요. → 지금 일에 필요한 얘기로만 채워요.

03

모델 — 딱 3줄만 기억해요

이름을 다 외울 필요 없어요.

모델 3줄 규칙
  1. 보통은 Sonnet. 매일 하는 일(글·기획·자료찾기·검토)엔 이게 빠르고 충분해요.
  2. 어려우면 Opus. 복잡한 생각, 긴 제안서, 까다로운 판단일 때 올려요.
  3. 빠르고 간단하면 Haiku, 아주 어렵고 오래 걸리면 Fable.

Sonnet

Opus

특급 전문가아주 어렵고 오래 걸리는 일. 꼭 필요할 때만

이렇게 해요: 헷갈리면 Sonnet으로 시작하고, 답이 얕으면 "이거 Opus로 다시 해줘" 하면 돼요. 올리는 건 쉬워요. (이름은 바뀔 수 있어요. 새 이름이 보이면 더 좋은 버전이구나 하고, 이 3줄은 그대로 쓰면 돼요.)

04

잘 쓰는 5가지 습관

프롬프트 전문가가 될 필요 없어요.

  • 01상황부터 알려줘요. 누구를 위한, 무슨 일인지 먼저 말하면 답이 확 좋아져요.
  • 02대화로 다듬어요. "더 짧게 · 이 느낌으로 · 3개만 남겨" — 이렇게 주고받는 게 정상이에요.
  • 03예시를 줘요. 원하는 느낌의 샘플 하나만 붙여도 잘 맞춰줘요.
  • 04주제가 바뀌면 새 대화.
  • 05사실·숫자·개인정보·말투는 사람이 확인하고 보내요. 특히 밖으로 나가는 건요.
나쁨배너 문구 뽑아줘
좋음20대 여성 화장품 여름 세일 배너예요. 튀되 과하지 않게, 15자 안으로 5개 뽑아줘.

06

팀으로 쓰기 vs 혼자 쓰기

뭘 각자 하고, 뭘 같이 관리할까요.

혼자 · 빠르게
  • 각자 일 대화·초안·실험
  • 개인 요청, 한 번 쓰고 마는 것
  • 자유롭게 빨리 해보기
팀 · 똑같이 맞추기
  • 팀 공통 규칙 · 잘 된 요청 모음
  • 고객 프로젝트 메모 · 틀
  • 노하우 쌓기 · 겹치는 일 줄이기

원칙: 혼자 잘 쓴 게 자꾸 반복되면 → 팀 것으로 올려요. 계정은 각자 쓰고(보안 때문), 비밀·개인정보는 넣기 전에 가려요.

프로젝트 두 가지 — 뭘 쓸지 선택

같은 이름이지만 다르게 쓰는 방식이 있어요. 대화 방식으로 하는 "프로젝트"와 파일로 남기는 "프로젝트 노트"예요.

07

용어 사전 — "이 말이 대체 뭐야?"

스킬·마크다운·깃허브가 뭔지 몰라도 일은 시작돼요. 이 사전은 겁먹지 말라고 있는 거예요.

먼저 안심하세요. 지금 꼭 알아야 할 건 거의 없어요. 챗에서 우리말로 부탁하는 것만으로 대부분 돼요. 아래는 "일하다 지나가며 보게 될 간판"일 뿐이에요 — 뜻만 알면 안 무서워요. 알면 편함 / 몰라도 됨.
A · 파일·문서
마크다운 (.md)알면 편함

글을 간단한 기호로 예쁘게 쓰는 가벼운 문서예요. 이 가이드 문서들이 다 이걸로 돼 있어요.비유 — 제목·굵게·목록이 되는 아주 가벼운 메모장

HTML몰라도 됨

웹페이지를 만드는 방식이에요. 결과를 예쁘게 보여줄 때 클로드가 대신 만들어줘요. 이 페이지가 그 예예요.

B · 깃허브 — 팀 공동 창고
대부분은 깃허브를 직접 안 만져도 돼요. "우리 결과물이 여기 보관되고 링크로 공유된다" 정도만 알면 돼요.
깃허브 (GitHub)알면 편함

문서·파일을 팀이 같이 보관하고, 누가 언제 뭘 바꿨는지 다 기록하는 인터넷 창고예요.비유 — 구글 드라이브 + 되돌리기가 아주 센 것

저장소 · 커밋 · 브랜치 · PR원리는 몰라도 됨

저장소 = 프로젝트 폴더 전체 · 커밋 = "저장 + 뭘 바꿨는지 메모"(게임 세이브) · 브랜치 = 원본 안 건드리는 작업용 복사본 · PR = "이거 원본에 합쳐도 돼요?" 하고 올리는 확인 요청.

C · 클로드에서 자주 듣는 말
스킬 (Skill)알면 편함

어떤 일을 "이렇게 해" 하고 미리 정리해 둔 설명서 묶음이에요. 클로드가 필요할 때 알아서 꺼내 써요. 매번 똑같이 길게 설명 안 해도 돼요.비유 — 자주 하는 요리의 레시피 카드

에이전트 (Agent)알면 편함

특정 역할만 잘하는 클로드 분신이에요. "견적 담당" "트렌드 조사 담당"처럼 부르면 그 시선으로 일해요.스킬 = 하는 방법 / 에이전트 = 그 방법 든 담당자

아티팩트 (Artifact)알면 편함

클로드가 만든 결과(웹페이지·문서)를 링크로 바로 볼 수 있게 내보낸 거예요. 이 페이지가 그 예예요.

D · 연결·도구
MCP · 커넥터 (Connector)알면 편함

클로드를 다른 도구(노션·지메일·구글드라이브·피그마 등)에 연결하는 다리예요. 연결하면 그 안 자료를 읽거나 쓸 수 있어요. (연결은 보통 관리자가 해줘요)주의 — 연결된 자료는 비밀·개인정보일 수 있어요. 보안 규칙을 지켜요.

Claude Code · 터미널몰라도 됨

까만 화면에 명령을 쳐서 파일·자동화를 직접 다루는 개발자용 화면이에요. 대부분은 지금 몰라도 돼요.

핵심: 이런 말이 나와도 놀라지 마세요. "아 그거였지" 하고 넘기면 돼요. 진짜 필요할 때 이 사전을 다시 펴면 돼요.

08

다 됐는지 확인 목록

모두 8개 다 통과하면 "같은 출발선" 완성. 완벽이 아니라 "혼자서도 겁 없이 해본다"가 기준이에요.

  • 챗·프로젝트·코드, 세 가지 쓰는 곳의 차이를 한 문장씩 말할 수 있어요
  • "세션 = 기억 하나"를 알고, 주제가 바뀌면 새 대화를 열어요
  • 모델 3줄을 알아요 (보통 Sonnet / 어려우면 Opus / 양 끝은 Haiku·Fable)
  • 대충 한 요청을 "상황부터 + 자세히"로 고칠 수 있어요
  • 답을 대화로 다듬어 봐요 (더 짧게 / 느낌 바꾸기 / 몇 개만 남기기)
  • 고객·일별로 프로젝트를 하나 만들어 봤어요
  • 사실·숫자·개인정보·말투는 사람이 확인하고 보낸다는 걸 알아요
  • 막히면 어디에 물어보는지 알아요 (팀 공유 채널)
09

실제로 자주 묻는 질문

온보딩 후 나올 질문들. 여기 없으면 팀 공유 채널에 물어봐도 돼요.

Q. 대화가 자꾸 흐려져요

언제 새 대화를 열어야 하나요?

주제가 다르면 바로 새 대화. 고객 A → 고객 B, 또는 배너 작업 → 기획 작업으로 넘어가면 새 대화를 열어요. 안 그러면 답이 섞여요.

→ 자세히는 02. 세션 규칙 참고

Q. Sonnet이 이상해요

Opus로 바꿔도 되나요?

맞다, 올려도 돼. "답이 얕으면 Opus로 다시 해줘" 하면 바로 올려요. 모델 올리기는 자유고, 내리는 것도 자유예요.

→ 모델 팁은 03. 모델 참고

Q. 고객 정보를 입력해도 돼요?

고객 이름·가격·계약서는?

발송 전에 사람이 필수 확인. 입력할 때는 구체적인 수치·이름은 최소화하고, 특히 "외부 반출 금지" 자료는 넣지 않아요. 마스킹해서 입력해요.

→ 보안은 30_그룹_개인_운영.md 참고

Q. 프로젝트는 언제 닫아요?

계속 열어두면 안 되나요?

고객 일이 끝나면 닫는다. 프로젝트는 "한 가지 일" 단위니까, 그 일이 끝나면 정리해요. 나중에 같은 고객이 오면 새로 만들면 돼요.

→ 프로젝트 관리는 README · 30_그룹_개인_운영.md 참고

Q. 한 달 전 대화를 다시

이어서 할 수 있나요?

새 대화로 시작하되, 핵심만 붙여넣기. 세션은 각각 따로 기억하니까, 이전 대화를 다시 열면 그때 맥락을 다시 읽을 수 있어요. 하지만 새 대화에서 계속하려면 핵심을 옮겨 붙여요.

→ 세션 규칙 3번 참고

Q. 같은 질문을 여러 번

다시 해도 되나요?

되지만, 새 대화에서 해야 돼요. 같은 대화에서 같은 질문을 반복하면 "아까 했는데?" 할 수 있어요. 새 대화에서 다시 부탁하면 새로 풀어줘요.

→ 세션 개념 이해하기

Q. 막혔어요

누구에게 물어봐?

팀 공유 채널에 올려. "클로드가 이렇게 말하는데 뭘 의미하나요?" 또는 "이런 일에는 뭐가 먼저인가요?" 하고 물어봐도 돼요. 팀이 함께 풀어요.

→ 자세히는 README의 "진행자 체크리스트" 참고

Q. 계속 틀린 답을 줘요

클로드가 이상한 건가요?

아니다. 요청이 불명확했을 수 있어요. 답이 이상하면: ① 예시를 하나 붙여주기 ② 상황을 더 자세히 설명하기 ③ Opus로 올려보기. 보통 이 중 하나로 해결돼요.

→ 잘 쓰는 5가지 습관 참고

Q. 내 요청이 반복돼요

자동화할 수 있나요?

넹, 할 수 있어요. 같은 질문을 월 2회 이상 하면, 팀 공유 채널에 "이 요청 자동화하면 좋겠어요" 하면 돼요. 에이전트/스킬로 만들어줄 수 있어요.

→ 자세히는 60_에이전트_공유하기.md 참고

Q. 코드는 꼭 배워야 하나요?

지금 배워야 할까요?

아니다. 나중에 필요하면 배워. 지금은 챗으로 충분해요. 팀이 자동화가 필요하면 그때 Code를 쓰는 사람들이 챙길 거예요. 우선은 챗을 잘 쓰는 게 먼저예요.

→ 코드가 뭔지는 07. 용어 사전 참고

10

한 걸음 더 — 심화 학습

기초를 다진 후, 자기 속도대로. 한 번씩 따라하며 배우는 7단계 마스터 가이드.

Day 1
Chat 메모리 & 설정 · 약 40분

Claude가 나를 기억하게 하기

메모리 토글 켜기: 설정 → 기능(Capabilities) → "대화 기록으로 메모리 생성" + "지난 대화 검색·참조" 토글 ON. Claude가 일주일만 지나도 당신의 프로젝트·말투·우선순위를 기억하게 됩니다.

사용자 환경설정: 설정 → 사용자 환경설정에 당신의 직책·브랜드·말투·진행 중인 프로젝트를 한 번 써 두세요. 모든 대화에 적용돼요.

→ 다음부터 맥락 설명 없이 바로 일하기 시작할 수 있어요
Day 2
프로젝트·도구·모델 · 약 50분

상황별로 다르게 쓰는 법

프로젝트 4가지 만들기: ① 콘텐츠 스튜디오 (글쓰기·릴스) ② 비즈니스 운영 (고객·제안) ③ 제품/프로젝트 (기술·로드맵) ④ 커리어 (이력서·면접).

도구 켜고 끄기: 채팅 입력창 왼쪽 아래 검색·도구로 필요한 것만 ON. 웹 검색(리서치), 확장 사고(전략·어려운 문제), 딥 리서치(시장조사), 코드 실행(데이터 정리). 안 쓸 땐 끠서 토큰 아끼기.

→ 일의 종류에 따라 맞춤 설정
Day 3
Claude Code 실전 · 약 60분

파일·자동화 다루기

설치: 터미널에서 npm install -g @anthropic-ai/claude-code → 프로젝트 폴더에서 claude 입력.

첫 명령어: /init (프로젝트 메모리 생성) → /goal [작업] (Claude가 끝날 때까지 자율 수행).

CLAUDE.md 작성: 프로젝트명·기술스택·코드규칙·명령어·건드리지 말 것을 한 번 써 두면, 매번 반복 설명할 필요가 없어요.

→ 터미널에서 "Claude, CSV 정리하고 리포트 만들어줘" 하면 알아서 해줌
Day 4
MCP 커넥터 · 약 40분

내 도구들과 연결하기

연결할 수 있는 것: Gmail (메일 요약·초안), Google Drive (파일 검색·데이터 추출), Google Calendar (일정 관리·블록 생성), Slack, Notion, GitHub, n8n (내 자동화 실행).

연결 방법: 설정 → 커넥터 → 찾아보기 → 추가. 이후 Chat나 Cowork에서 바로 접근 가능.

주의: 필요 없을 땐 커넥터를 꺼두세요. 토큰을 소모하고 컨텍스트 한도에 더 빨리 도달합니다.

→ "어제 마케팅팀이 보낸 메일 찾아줘. 요청이 뭔지 요약해"라고 하면 Gmail에서 자동으로 찾아 정리해줌
Day 5
스킬 vs 플러그인 · 약 50분

Claude를 전문가로 만들기

스킬: 특정 일을 가르치는 마크다운 파일. 관련 맥락이 나오면 자동으로 발동. 예) 릴스 대본 작성기, 브랜드 보이스 체커.

플러그인: 스킬 + 슬래시 명령어 + 커넥터를 묶은 완제품. Cowork에서 설치 후 / 입력하면 자동 완성. 예) Productivity, Marketing, Sales.

스킬 만들기: .claude/skills/[스킬명]/SKILL.md 파일 만들고, 목적·규칙·출력 형식 3가지만 정의하면 끝.

→ "매번 같은 요청을 하고 있다" 싶으면 스킬로 만들어 자동화
Day 6
Cowork 위임 · 약 50분

결과물만 받기

Cowork는 Chat이 아닙니다. 작업을 설명하고 자리를 뜨면, Claude가 로컬 파일·여러 앱을 넘나들며 자율 수행합니다.

예약 작업: /schedule 입력해서 주간 반복 업무 자동화. 예) 매주 월요일 오전 8:30에 Gmail·Slack 확인 + 요약 저장.

모바일 디스패치: 휴대폰에서 "이번 주 협업 메일 정리해줘"라고 지시 → 데스크톱 Claude가 작업 → 완료되면 푸시 알림.

→ 데스크톱 켜고만 있으면, 휴대폰에서 일 지시 가능
Day 7
Claude Design + 안전 · 약 60분

마무리와 완성

Claude Design: 시각 작업 전용 캔버스. 피치덱·랜딩페이지·프로토타입·마케팅 자료. 먼저 "브랜드 킷" (GitHub 레포/Figma/웹사이트)을 가리키면 색·타이포·컴포넌트를 자동 학습.

Tweaks 패널: 채팅 없이 슬라이더로 디자인 조정 (색온도·간격·타이포스케일). 구조 변경만 채팅으로, 나머지는 Tweaks로.

안전 규칙: 낮은 위험부터 시작 (요약·초안·정리) → 높은 위험은 사람 승인 거친다 (금융 실행·고객데이터·자동 메일).

→ 완성된 디자인은 Claude Code로 "구현해줘"라고 하면 프로덕션 코드가 나옴
