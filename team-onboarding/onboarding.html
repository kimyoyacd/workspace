01

어디서 쓰나 — 쓰는 곳 3가지

같은 클로드지만 들어가는 문이 세 개예요. 문마다 잘하는 일이 달라요.

① 챗 · claude.ai

대화창

아이디어 · 글쓰기 · 자료 찾기 · 요약 · 시안 봐주기. 거의 모두가 여기서 시작해요.

② 프로젝트

한곳에 모으는 폴더

한 가지 일에 필요한 대화·파일을 한곳에 모아요. 계속 이어서 할 수 있어요.

③ 코드 · Claude Code

작업실

파일을 진짜로 만들고 고치고, 반복 작업을 자동으로. 필요한 사람만 나중에 배우면 돼요.

처음엔: ①챗 + ②프로젝트만으로 충분해요.

02

세션 = 대화 하나 = 기억 하나

이것만 알면 절반은 끝나요.

새 대화를 열면 클로드는 그 대화에서만 기억해요. 옆 대화는 몰라요.

세션 쓰는 3가지 규칙
  1. 주제가 바뀌면 새 대화. 고객 A → 고객 B로 넘어가면 새 대화를 열어요.
  2. 대화 이름을 규칙대로. [고객]_[일]_[날짜] 이렇게 지으면 찾기 쉬워요.
  3. 대화가 길면 중요한 건 밖에 적어둬요. 정한 방향·숫자는 그때그때 문서에 옮겨 적어요.
03

모델 — 딱 3줄만 기억해요

이름을 다 외울 필요 없어요.

모델 3줄 규칙
  1. 보통은 Sonnet. 매일 하는 일엔 이게 충분해요.
  2. 어려우면 Opus. 복잡한 생각·긴 제안서일 때 올려요.
  3. 빠르고 간단하면 Haiku, 아주 어렵고 오래 걸리면 Fable.
모델 비유 언제 써요
Haiku 인턴 (손이 빠름) 간단·반복·빠르게 — 분류, 짧은 요약, 초안 여러 개
Sonnet 실무 담당 매일 하는 일 거의 다 — 기획·글쓰기·자료찾기·시안
Opus 선배 복잡한 전략·긴 제안서·어려운 판단
Fable 특급 전문가 아주 어렵고 오래 걸리는 일. 꼭 필요할 때만
04

잘 쓰는 5가지 습관

프롬프트 전문가가 될 필요 없어요.

  • 01상황부터 알려줘요. 누구를 위한, 무슨 일인지 먼저 말하면 답이 확 좋아져요.
  • 02대화로 다듬어요. "더 짧게 · 이 느낌으로 · 3개만" — 주고받는 게 정상이에요.
  • 03예시를 줘요. 원하는 느낌의 샘플 하나만 붙여도 잘 맞춰줘요.
  • 04주제가 바뀌면 새 대화.
  • 05사실·숫자·개인정보는 사람이 확인하고 보내요. 특히 밖으로 나가는 건요.
05

3단계로 시작해요

한 번의 교육이 아니라, 같은 출발선 → 실제 업무 → 팀 방식으로 굳히기 순서예요.

1
1주차 · 같은 출발선 만들기

모두가 개념을 알고 혼자 시작할 수 있어요

기초 개념 읽기 → 다 같이 90분 배우기 → 확인 목록 8개 통과.

→ 모두 통과하면 완성
2
2–4주차 · 실제 업무에 붙이기

각자 자기 일에 써봐요

역할별 예시를 실제 일에 써보기 → 매주 15분 나누기 → 잘 된 요청 모으기.

→ 한 사람당 잘 되는 방법 3개 이상
3
2개월차~ · 팀 방식으로 굳히기

잘 쓰는 법을 팀 것으로 만들어요

자주 쓰는 요청 → 공유 틀로 만들기 → 자주 하는 일은 스킬/에이전트로 묶기.

→ "개인이 잘 쓰는 팀"에서 "팀이 잘 쓰는 팀"으로
06

팀으로 쓰기 vs 혼자 쓰기

뭘 각자 하고, 뭘 같이 관리할까요.

혼자 · 빠르게
  • 각자 일 대화·초안·실험
  • 개인 요청, 한 번 쓰고 마는 것
  • 자유롭게 빨리 해보기
팀 · 똑같이 맞추기
  • 팀 공통 규칙 · 잘 된 요청 모음
  • 고객 프로젝트 메모 · 틀
  • 노하우 쌓기 · 겹치는 일 줄이기

원칙: 혼자 잘 쓴 게 자꾸 반복되면 → 팀 것으로 올려요.

프로젝트 두 가지 — 뭘 쓸지 선택

구분 일반 챗 프로젝트 (가볍게) Claude Code 프로젝트 노트 (팀 기록)
뭐예요 챗 안의 폴더. 관련 대화를 한곳에 모음 .claude/projects/ 폴더의 .md 파일. 팀이 봄
기록 대화만 남음. 파일 없음 실제 파일로 저장소에 남음
장점 빠르고 즉시 시작. 가벼움 팀이 함께 봄. 재사용 가능. 오래 보관
네이밍 [고객]_[일]_[날짜] YYYYMM_고객_일
07

용어 사전

"이 말이 대체 뭐야?"

먼저 안심하세요. 지금 꼭 알아야 할 건 거의 없어요. 낯선 말이 나와도 "아 그거였지" 하고 넘기면 돼요.
A · 파일·문서
마크다운 (.md)알면 편함

글을 간단한 기호로 예쁘게 쓰는 가벼운 문서예요. 비유 — 제목·굵게·목록이 되는 아주 가벼운 메모장

HTML몰라도 됨

웹페이지를 만드는 방식이에요. 결과를 예쁘게 보여줄 때 클로드가 대신 만들어줘요.

B · 깃허브 — 팀 공동 창고
깃허브 (GitHub)알면 편함

문서·파일을 팀이 같이 보관하고, 누가 언제 뭘 바꿨는지 다 기록하는 인터넷 창고예요. 비유 — 구글 드라이브 + 되돌리기가 센 것

C · 클로드에서 자주 듣는 말
에이전트 (Agent)알면 편함

특정 역할만 잘하는 클로드 분신이에요. "견적 담당" "트렌드 조사 담당"처럼 부르면 그 시선으로 일해요.

08

다 됐는지 확인 목록

모두 8개 다 통과하면 "같은 출발선" 완성.

  • 챗·프로젝트·코드, 세 가지의 차이를 말할 수 있어요
  • "세션 = 기억 하나"를 알고, 주제가 바뀌면 새 대화를 열어요
  • 모델 3줄을 알아요 (보통 Sonnet / 어려우면 Opus / 양 끝은 Haiku·Fable)
  • 대충 한 요청을 "상황부터 + 자세히"로 고칠 수 있어요
  • 답을 대화로 다듬어 봐요 (더 짧게 / 느낌 바꾸기 / 몇 개만)
  • 고객·일별로 프로젝트를 하나 만들어 봤어요
  • 사실·숫자·개인정보는 사람이 확인하고 보낸다는 걸 알아요
  • 막히면 어디에 물어보는지 알아요
09

자주 묻는 질문

온보딩 후 나올 질문들.

Q. 대화가 자꾸 흐려져요

언제 새 대화를 열어야 하나요?

주제가 다르면 바로 새 대화. 고객 A → 고객 B로 넘어가면 새 대화를 열어요.

Q. Sonnet이 이상해요

Opus로 바꿔도 되나요?

맞다, 올려도 돼. "답이 얕으면 Opus로 다시 해줘" 하면 바로 올려요.

Q. 고객 정보를 입력해도 돼요?

고객 이름·가격·계약서는?

발송 전에 사람이 필수 확인. 개인정보는 마스킹해서 입력해요.

Q. 같은 질문을 여러 번

다시 해도 되나요?

되지만, 새 대화에서 해야 돼요. 같은 대화에서 반복하면 답이 흐릿할 수 있어요.

Q. 막혔어요

누구에게 물어봐?

팀 공유 채널에 올려. 팀이 함께 풀어요.

Q. 계속 틀린 답을 줘요

클로드가 이상한 건가요?

아니다. 요청이 불명확했을 수 있어요. 예시를 붙여주거나 상황을 더 자세히 설명하세요.

10

한 걸음 더 — 심화 학습

기초를 다진 후, 자기 속도대로. 한 번씩 따라하며 배우는 7단계 마스터 가이드.

Chat 메모리 & 설정 · 약 40분

Claude가 나를 기억하게 하기

메모리 토글 켜기: 설정 → 기능 → "대화 기록으로 메모리 생성" + "지난 대화 검색" 토글 ON. Claude가 당신의 프로젝트·말투를 기억하게 됩니다.

사용자 환경설정: 설정 → 사용자 환경설정에 당신의 직책·브랜드·말투를 한 번 써 두세요. 모든 대화에 적용돼요.

→ 다음부터 맥락 설명 없이 바로 일하기 시작할 수 있어요
프로젝트·도구·모델 · 약 50분

상황별로 다르게 쓰는 법

프로젝트 4가지 만들기: ① 콘텐츠 스튜디오 (글쓰기·릴스) ② 비즈니스 운영 (고객·제안) ③ 제품/프로젝트 (기술·로드맵) ④ 커리어 (이력서·면접).

도구 켜고 끄기: 필요한 것만 ON. 웹 검색, 확장 사고, 딥 리서치, 코드 실행. 안 쓸 땐 끠서 토큰 아끼기.

→ 일의 종류에 따라 맞춤 설정
Claude Code 실전 · 약 60분

파일·자동화 다루기

설치: 터미널에서 npm install -g @anthropic-ai/claude-code → 프로젝트 폴더에서 claude 입력.

첫 명령어: /init (프로젝트 메모리 생성) → /goal [작업] (자율 수행).

CLAUDE.md 작성: 프로젝트명·기술스택·코드규칙을 한 번 써 두면, 매번 반복 설명할 필요가 없어요.

→ 터미널에서 "Claude, CSV 정리하고 리포트 만들어줘" 하면 알아서 해줌
MCP 커넥터 · 약 40분

내 도구들과 연결하기

연결할 수 있는 것: Gmail (메일 요약·초안), Google Drive (파일 검색), Google Calendar (일정 관리), Slack, Notion, GitHub, n8n (자동화 실행).

연결 방법: 설정 → 커넥터 → 추가. 이후 Chat이나 Cowork에서 바로 접근 가능.

주의: 필요 없을 땐 커넥터를 꺼두세요. 토큰을 소모합니다.

→ "어제 마케팅팀이 보낸 메일 찾아줘. 요청이 뭔지 요약해"라고 하면 자동으로 찾아 정리해줌
스킬 vs 플러그인 · 약 50분

Claude를 전문가로 만들기

스킬: 특정 일을 가르치는 마크다운 파일. 관련 맥락이 나오면 자동으로 발동. 예) 릴스 대본 작성기, 브랜드 보이스 체커.

플러그인: 스킬 + 슬래시 명령어를 묶은 완제품. Cowork에서 설치 후 / 입력하면 자동 완성.

스킬 만들기: .claude/skills/[스킬명]/SKILL.md 파일 만들고, 목적·규칙·출력 형식 3가지만 정의하면 끝.

→ "매번 같은 요청을 하고 있다" 싶으면 스킬로 만들어 자동화
Cowork 위임 · 약 50분

결과물만 받기

Cowork는 Chat이 아닙니다. 작업을 설명하고 자리를 뜨면, Claude가 로컬 파일·여러 앱을 넘나들며 자율 수행합니다.

예약 작업: /schedule 입력해서 주간 반복 업무 자동화. 예) 매주 월요일 오전 8:30에 Gmail·Slack 확인 + 요약 저장.

모바일 디스패치: 휴대폰에서 지시 → 데스크톱 Claude가 작업 → 완료되면 푸시 알림.

→ 데스크톱 켜고만 있으면, 휴대폰에서 일 지시 가능
Claude Design + 안전 · 약 60분

마무리와 완성

Claude Design: 시각 작업 전용 캔버스. 피치덱·랜딩페이지·프로토타입·마케팅 자료. 먼저 "브랜드 킷"을 가리키면 색·타이포·컴포넌트를 자동 학습.

Tweaks 패널: 채팅 없이 슬라이더로 디자인 조정 (색온도·간격·타이포스케일). 구조 변경만 채팅으로, 나머지는 Tweaks로.

안전 규칙: 낮은 위험부터 시작 (요약·초안·정리) → 높은 위험은 사람 승인 거친다 (금융·개인데이터·자동 메일).

→ 완성된 디자인은 Claude Code로 "구현해줘"라고 하면 프로덕션 코드가 나옴
