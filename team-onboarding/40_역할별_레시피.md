# 40. 역할별 레시피 — 바로 붙여 쓰는 프롬프트

> 자기 직군에서 3개만 골라 실제 업무에 붙여보라. `[ ]` 부분만 자기 상황으로 바꾼다.
> 모든 레시피는 **맥락 먼저 → 구체적 요청 → 대화로 다듬기**를 전제로 한다.

---

## 공통 (누구나)

**A. 긴 자료 요약**
```
아래는 [자료 종류]다. 핵심을 3가지 소제목으로 정리하고, 실무에서 우리가
결정해야 할 것 2가지를 뽑아줘. 그 다음 원문을 붙일게.
[원문]
```

**B. 초안 톤 다듬기**
```
아래 초안을 [정중하고 신뢰감 있는 / 캐주얼하고 친근한] 톤으로 다듬어줘.
길이는 비슷하게, 과장·군더더기는 빼고.
[초안]
```

**C. 회신/메일 작성**
```
[상황: 누가·무엇을 요청]에 대한 회신을 써줘. 목적은 [무엇], 톤은 [무엇],
분량은 [길이]. 상대가 바로 이해하도록 결론부터.
```

---

## 디자인 실무 (시안·비주얼)

**D. 아이데이션 발산**
```
[제품/캠페인]의 [매체: 배너/상세페이지/키비주얼] 방향을 발산해줘.
서로 다른 컨셉 5개, 각 컨셉마다 한 줄 설명 + 무드 키워드 3개.
타깃은 [누구], 톤은 [무엇], 피해야 할 것은 [무엇].
```

**E. 시안 피드백 받기 (이미지 첨부)**
```
이 시안을 냉정하게 검수해줘. 잘된 점 말고 고칠 점 위주로,
[가독성 / 위계 / 여백 / 톤 일관성] 관점에서 각각 지적하고
우선순위를 매겨줘.
```

**F. 레퍼런스 리서치**
```
[카테고리/스타일]의 최근 레퍼런스를 찾아 정리해줘. 각 사례마다
출처 URL, 왜 좋은지 한 줄, 우리가 훔칠 포인트 한 줄. 5개.
```

---

## 기획·전략 (제안·문서)

**G. RFP/과업 분석**
```
아래 과업지시서를 분석해줘: (1)명시적 요구 (2)숨은 니즈 (3)미정의/모호한 것
(4)리스크. 마지막에 클라이언트에 확인할 질문 3개.
[과업 원문]
```

**H. 제안 논리 점검**
```
아래 제안 방향을 5관점(기획/크리에이티브/사업성/마케팅/고객)에서
짧게 검토하고, 가장 약한 고리 1개와 보완책을 알려줘.
[제안 요약]
```

---

## 운영·매출 (견적·관리)

**I. 견적 초안 잡기** *(최종 금액·마진은 반드시 사람이 결정)*
```
[업무 범위]에 대한 공수 산정 초안을 잡아줘. 직군별 투입 일수를 표로,
가정은 명시적으로. 최종 견적가와 마진은 내가 정할 테니 원가 구조만.
```

**J. 진행 현황 정리**
```
아래 프로젝트 목록을 상태별로 그룹핑하고, 마감 임박·리스크 항목을
맨 위로 올려 표로 정리해줘.
[목록/데이터]
```

---

## 글쓰기·콘텐츠

**K. 카피 N안**
```
[제품/상황]의 [매체] 카피를 [N]개 써줘. 각 [글자수] 이내,
서로 톤이 겹치지 않게. 타깃은 [누구].
```

**L. AI 티 빼기 (자연스럽게)**
```
아래 글에서 AI가 쓴 듯한 어색함(과한 수식어, 뻔한 구조, 상투구)을 걷어내고
사람이 쓴 것처럼 자연스럽게 다듬어줘. 의미는 유지.
[글]
```

---

## 레시피를 팀 자산으로

- 여기서 잘 먹힌 레시피는 **자기 버전으로 개량**해 공유 저장소에 적립한다(→ `30_그룹_개인_운영.md`).
- 3단계(팀 표준화)에서 반복 레시피는 **템플릿/에이전트**로 승격한다.
