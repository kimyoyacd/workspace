# AI 시대, 제작사는 어떻게 살아남는가
> Deep Research Report — 2026.06 | HIVELAB 내부 전략 참고용
> 5개 각도 × 50+ 출처 교차 검증 기반

---

## TL;DR — 3문장 요약

1. **제작 단가는 0에 수렴한다.** AI가 모든 에이전시에 동일하게 제공되는 순간, "잘 만든다"는 더 이상 가치가 아니다.
2. **살아남는 에이전시의 공통점은 3가지:** 자체 데이터 해자, 성과 기반 과금, "취향(Taste)" 기반 포지셔닝.
3. **HIVELAB의 6,700개 소재 데이터는 이 세 가지를 동시에 가능하게 하는 희귀 자산이다.**

## 2. 살아남은 에이전시 — 무엇이 달랐나

### 2-1. 승자 vs 패자를 가른 5가지 요인

| 요인 | 실패/축소 에이전시 | 성공/성장 에이전시 |
|------|-------------------|-------------------|
| **AI 접근법** | 기존 구조에 AI를 "얹는" 방식 | AI를 비즈니스 모델 코어에 내재화 |
| **데이터 자산** | 클라이언트 데이터에 접근 제한 | 자체 데이터 플랫폼 보유 |
| **과금 모델** | 시간 단위(hourly) 유지 | 구독/성과 기반으로 전환 |
| **조직 구조** | 레거시 대규모, 사일로 | AI 네이티브, 소수 정예 |
| **판매 대상** | 실행(Implementation) | 전략 + AI 통합 솔루션 |

### 2-2. 성공 사례 데이터

| 에이전시 | 매출/성과 | 핵심 전략 |
|---------|----------|----------|
| **Publicis** | 2024년 €139.6억, **+6.6%** 성장 | Epsilon(2.5억+ 소비자 프로필) + CoreAI → 데이터 해자 |
| **Accenture Song** | 2025년 **$200억** 돌파 | 크리에이티브 → 전략·컨설팅·기술 통합 (상류 진출) |
| **Brandtech (Pencil)** | **$40억** 밸류에이션 | AI-first 크리에이티브, ROAS **+79%**, CPA **-48%** |
| **Superside** | 2024년 $44.9M, **+46%** YoY | 글로벌 분산 인력 846명 + AI 도구 결합 |

## 3. 제작사 경영자를 위한 5가지 사고 프레임워크

### 프레임워크 ① — "언번들링과 리번들링"

> "7명이 3개월 걸리던 일을 2명이 3주에 끝내는 시대." — Dries Buytaert

AI가 에이전시의 전통적 번들(코드+카피+디자인+설정)을 **해체(unbundle)**하고 있다. 클라이언트는 각각을 AI로 직접 처리할 수 있게 됨. 생존하려면 **AI가 못하는 것 — 전략, 거버넌스, 성과 책임, 장기 운영 — 을 중심으로 리번들링(rebundle)**해야 한다.

**데이터 근거:** Big Five 광고 그룹의 인수합병 점유율: 2016년 **88%** → 2024년 **19%**로 급락. 대형이 유리하던 구조가 해체되고 있다.

---

### 프레임워크 ② — "테이스트 레이어(Taste Layer)"

> "콘텐츠 생성 비용이 0에 수렴할수록, 개별 결과물 하나의 가치는 떨어진다. 부상하는 것은 Taste." — MindStudio

**Taste의 3가지 구성 요소:**

1. **편집적 판단력** — 무엇을 만들고, 무엇을 버릴지 결정
2. **디자인 감각** — 위계, 톤, 시각 언어 설계
3. **큐레이션** — 신호와 소음을 구별

**AI가 Taste를 복제할 수 없는 구조적 한계:**
- 책임 부재 — AI는 성패에 책임지지 않는다
- 맥락 맹점 — 특정 시장·오디언스의 문화적 층위를 놓친다
- 통계적 평균화 — 훈련 데이터의 중심부를 반영한 제네릭 결과
- 전체 관점(POV) 부재 — 문장은 좋지만 전체적 방향이 없다

**HIVELAB 맥락:** 6,700개 넷마블 소재를 만든 경험 = 넷마블 IP에 대한 Taste 축적. 다른 에이전시가 AI를 써도 이 Taste는 복제 불가.

---

### 프레임워크 ③ — "AI 네이티브 vs AI 볼트온(Bolt-on)"

| 차원 | 전통 에이전시 (Bolt-on) | AI 네이티브 에이전시 |
|------|----------------------|-------------------|
| 팀 구조 | 전문 역할별 분업 (순차적) | 2-3명 AI 강화 제너럴리스트 (병렬) |
| 수익 모델 | 시간당 과금 | 산출물 기반 고정 가격 |
| 크리에이티브 | 수동으로 3-5개 컨셉 | AI로 수십 개 변형 → 큐레이션 |
| 인간의 역할 | 실행(Execution) | 전략 + 큐레이션 |

**비용 비교 데이터:**
- 웹사이트 리디자인: $30K-$80K (10-15주) → **$8K-$25K (3-4주)** = 60-75% 절감
- 브랜드 아이덴티티: $25K-$100K → **$6K-$20K** = 75-80% 절감

**핵심:** "기존 프로세스에 AI 도구를 얹는 것"이 아니라 "운영 체제 자체를 AI 네이티브로 재설계"해야 한다.

---

### 프레임워크 ④ — "데이터 플라이휠 = 유일한 해자"

> "모든 마케터가 같은 AI 도구를 쓸 수 있게 되면, 차별화는 도구가 아니라 그 도구에 투입되는 고유 데이터의 질에서 나온다." — Sparkle Insights

**검증된 데이터:**
- 1st party 데이터 활용 기업: 경쟁사 대비 매출 성장률 **2.9배**, CAC **50% 절감**
- 마케터 **78%**가 1st party 데이터를 가장 가치 있는 소스로 평가 (2025)
- 1st party 데이터 공유 요청: 2024년 연 5건 → 2025년 **월 10건** (Dun & Bradstreet)

**대형 에이전시 그룹의 데이터 인수 경쟁:**

| 그룹 | 인수 | 금액 |
|------|------|------|
| Publicis | Epsilon (2019) + LiveRamp (2026) | $44억 + $22억 |
| Omnicom | Acxiom (2025) | — |
| WPP | InfoSum (2025) | — |

**플라이휠 메커니즘:**
소재 제작 → 성과 데이터 수집 → AI 학습 → 더 나은 소재 → 더 많은 성과 데이터 → ...
경쟁사가 복제하려면 동일한 시간 × 동일한 클라이언트 관계가 필요. **시간이 지날수록 격차는 벌어진다.**

---

### 프레임워크 ⑤ — "조율자(Orchestrator)로의 전환"

APR Co.의 3-Track 제작 모델:

| 트랙 | 성격 | 인간 개입도 | AI 역할 |
|------|------|-----------|---------|
| **Craft** | 히어로 콘텐츠, 장편 브랜드 | 높음 | 보조 |
| **Maker** | 소셜 퍼스트, 빠른 턴어라운드 | 중간 | 초안 + 변형 |
| **Content Engine** | 글로벌 버전화, 로컬라이징 | 낮음 | 주도 |

에이전시의 미래 역할은 세 트랙을 **동시에 운영하고 조율하는 오케스트레이터(Orchestrator)**.

> 포춘코리아가 정리한 한국 에이전시 6대 생존 전략:
> 1. 균형 (AI 효율 + 인간 통찰)
> 2. AI 문해력 내재화 (도구 → 문화)
> 3. 통합 (데이터+크리에이티브+미디어)
> 4. 협업 (Tech+Creative+Planning → ROAS **+48%**)
> 5. 과학적 성과 검증
> 6. 플랫폼 전문성

## 4. 과금 모델의 전환 — "시간을 파는 시대의 종말"

### 현재 압박

- AI가 제작 시간을 **3-4배 압축** → 시간 기반 과금 시 매출 자동 하락
- Bain 추정: 에이전시 수수료 적정 인하 폭 약 **20%**
- AI로 마케팅 노동 시간 **24% 감소** 가능 (생산성 **30%** 향상)

### 새로운 모델들

| 모델 | 설명 | 사례 |
|------|------|------|
| **성과 연동** | 매출·ROAS·브랜드 지표에 보수 연동 | WPP × Jaguar Land Rover |
| **구독/SaaS형** | AI 도구·대시보드·워크플로우 라이선스 | WPP 기술 라이선싱 |
| **산출물 기반 고정가** | 시간이 아닌 결과물 단위 과금 | Superside 구독 모델 |
| **3-Tier 패키지** | AI 보조 / AI+전략 / 풀 컨설팅 | 독립 에이전시 도입 중 |
| **월간 소재 팩 번들** | 소재 N개 + 인사이트 리포트 정액제 | HIVELAB 제안 가능 |

## 5. 사고 전환 요약 — 기존 vs 필요

| 기존 사고방식 | 필요한 사고방식 |
|-------------|---------------|
| "AI 도구를 도입하자" | "운영 체제를 AI 네이티브로 재설계하자" |
| "효율성을 높이자" (비용 절감) | "레버리지를 극대화하자" (소수 정예 + AI) |
| "우리의 실행력이 가치다" | "우리의 판단력(Taste)이 가치다" |
| "시간을 판다" | "결과를 판다" |
| "규모가 경쟁력" | "민첩성이 경쟁력" (소규모 AI 네이티브 > 대형 레거시) |

## 6. HIVELAB에 대한 시사점

위 리서치를 HIVELAB 맥락에 적용하면:

### 이미 가진 것 (현재 자산)

| 자산 | 가치 |
|------|------|
| 넷마블 6,700개 소재 데이터 | 1st party 데이터 해자 — 경쟁사 복제 불가 |
| 넷마블 IP별 성과 패턴 지식 | Taste Layer — AI가 대체할 수 없는 맥락적 판단력 |
| 제작 프로세스·승인 기준 내재화 | 전환 비용(Switching Cost) — 깊은 고객 관계 |

### 전환해야 할 것

| From | To | 구체적 액션 |
|------|----|-----------|
| 배너 단건 납품 | **월간 소재 팩 구독** | 소재 N개 + AI 변형 + 인사이트 리포트 정액제 |
| "잘 만든다" 어필 | **"데이터 근거로 결정한다"** 어필 | CTR 예측, 피로도 감지, 다음 방향 제안 포함 납품 |
| 시간 기반 견적 | **산출물 기반 고정가** | AI로 3배 빠르게 만들어도 매출 하락 없는 구조 |
| 제작사 포지션 | **AI 오케스트레이터** 포지션 | Craft + Maker + Content Engine 3트랙 통합 운영 |
| 단건 프로젝트 | **플라이휠 구조** | 납품 → 성과 데이터 → AI 학습 → 다음 납품 개선 → 반복 |

### 경고 신호 — 안 하면 일어날 일

- Klarna처럼 넷마블이 자체 AI 도구를 도입하면, HIVELAB 의뢰량은 **25-60% 감소** 가능
- 경쟁 에이전시가 "AI 할인"으로 진입하면, 단가 경쟁 시작
- 데이터가 쌓이지 않으면, 시간이 지날수록 AI 네이티브 에이전시와의 **격차만 벌어짐**
