# MAX실 Claude 워크스페이스

MAX실(디자인)의 업무용 Claude Code 워크스페이스입니다. 에이전트·스킬·프로젝트 노트·견적/운영 자산이 한곳에 모여 있고, GitHub으로 버전 관리·공유합니다.

## 팀원이라면 여기부터

처음이라면 아래 순서대로 읽으세요. (5분이면 시작할 수 있습니다)

1. **[docs/team/00_시작하기.md](docs/team/00_시작하기.md)** — 이게 뭔지, 첫 세션 3분 시작
2. **[docs/team/01_사용설명서.md](docs/team/01_사용설명서.md)** — 에이전트·스킬·프로젝트 노트 실전 사용법
3. **[docs/team/02_GitHub_가이드.md](docs/team/02_GitHub_가이드.md)** — GitHub 처음이어도 따라 하는 협업법
4. **[docs/team/03_실장_롤아웃_체크리스트.md](docs/team/03_실장_롤아웃_체크리스트.md)** — (실장용) 팀 배포 전 준비·공지 사항

## 폴더 지도

```
CLAUDE.md              # Claude 운영 지침 (매 세션 자동으로 읽힘 — 규칙의 본체)
docs/team/             # 👈 팀 온보딩 문서 (이 README가 가리키는 곳)
docs/index.html        # 업무 로그 대시보드
.claude/
├── agents/            # 24종 에이전트 (수주·디자인·검수·운영·조직)
├── skills/            # 스킬 (리포트 생성, 글쓰기 필터 등)
├── library/           # 재사용 자산 (견적 단가표·QC 체크리스트·자동화 맵·데이터 연결·덱 스타일 가이드)
└── projects/          # 프로젝트별 노트 (YYYYMM_프로젝트명)
```

## 핵심 규칙 3줄 요약

- 모든 규칙의 본체는 **`CLAUDE.md`** — 매 세션 자동으로 읽히니, 바꾸려면 이 파일을 고친다.
- 업무는 **프로젝트 단위**로 관리하고, 새 프로젝트는 `.claude/projects/YYYYMM_프로젝트명.md`에 노트를 만든다.
- **최종 견적가·마진율·계약 날인·외부 발송 OK는 사람(실장)이 직접** 결정한다. Claude에 위임하지 않는다.
