# 01. 사용설명서 — 에이전트·스킬·프로젝트

> 실전 사용법. 필요할 때 목차에서 찾아 쓰세요.

## 1. 에이전트 (역할별 전문가 24종)

특정 역할을 부르면 그 관점으로 일합니다. **트리거 단어**로 부르거나, 그냥 상황을 설명해도 Claude가 알아서 맞는 에이전트를 씁니다.

### ① 수주·전략
| 에이전트 | 하는 일 | 부를 때 |
|---|---|---|
| `rfp-analyst` | RFP → 명시·숨은·미정의·위험 4분류 | "RFP 분석해줘" |
| `market-research` | 경쟁사 → XY 포지셔닝 맵 + 빈자리 | "경쟁사 분석", "포지셔닝 맵" |
| `concept-director` | ★모든 디자인 전에 관점 한 문장 고정 | "관점 잡아줘", "컨셉 정의" |
| `brainstormer` | 아이디어 발산 | "브레인스토밍", "방향 뽑아줘" |
| `critic` | 비용·기간·역량 기준 냉정한 점수 | "현실적으로 돼?", "점수 매겨줘" |

### ② 디자인 인텔리전스
`design-trend-radar`(트렌드) · `reference-curator`(레퍼런스) · `design-system-guardian`(시스템 준수) · `design-critique`(시안 검수 🟢🟡🔴)

### ③ 디자인 생성
`visual-generator`(키비주얼·목업) · `moodboard-builder`(무드보드) · `figma-bridge`(코드↔Figma) · `storyboard-maker`(영상 콘티)

### ④ 검수 게이트 (외부 발송 전)
`fact-checker`(사실·숫자 검증) · `review-panel`(5관점 검토) · `delivery-gate`(발송 직전 린트) · `tone-guardian`(톤 교정)

### ⑤ 운영·매출
`project-manager`(현황·가동률, 읽기전용) · `quote-accountant`(견적 원가·마진) · `account-radar`(클라 히스토리)

### ⑥ 조직·채용·법무
`creative-director`(톤 최종·온보딩) · `hiring-radar`(채용) · `vendor-radar`(외주 풀) · `legal-compliance`(계약·NDA 플래그)

> 각 에이전트 상세는 `.claude/agents/<이름>.md`에 있습니다.

## 3. 프로젝트 노트 (제일 중요한 습관)

새 프로젝트 시작 = **노트부터 만든다.**

- 위치: `.claude/projects/`
- 파일명: `YYYYMM_프로젝트명.md` (예: `202607_코웨이_리뉴얼.md`)
- 필수 항목: **클라이언트 · 업무 범위 · 일정 · 견적 · 진행 메모**

Claude에게 *"코웨이 리뉴얼 프로젝트 노트 만들어줘"* 라고 하면 형식 맞춰 생성해 줍니다. 진행하면서 *"이 내용 프로젝트 노트에 기록해줘"* 로 계속 업데이트하세요.

> 이게 쌓이면 견적·클라 히스토리·회고의 원천이 됩니다. **노트를 안 쓰면 히스토리가 안 남습니다.**

## 4. 검수·발송 순서 (실수 방지 규칙)

외부로 나가는 것(제안서·메일·시안)은 **티어**에 따라 검수 게이트를 통과시킵니다. (`.claude/library/automation.md`)

- **Tier 0** 내부 단순 조회 → 게이트 없음
- **Tier 1** 내부 산출·시안 초안 → `design-critique` → `delivery-gate`
- **Tier 2** 외부 발신·제안·계약·브랜드 → `fact-checker` → `review-panel` → `delivery-gate` → `tone-guardian`
- **애매하면 무조건 상향. 외부 수신자가 있으면 자동 Tier 2.**

## 5. 절대 사람이 직접 (위임 금지)

최종 **견적가 · 마진율 · 계약 날인 · 단가 협상 · 채용/해고 · 외부 발송 최종 OK**. Claude는 초안·검토까지만.

## 6. 자주 하는 실수

- ❌ 프로젝트 노트 없이 진행 → 히스토리 유실
- ❌ 검수 게이트 건너뛰고 외부 발송 → Tier 2는 반드시 통과
- ❌ 견적 최종가를 Claude가 정하게 둠 → 원가·시나리오까지만, 결정은 실장
- ❌ GitHub에 저장 안 하고 세션 종료 → 다음 [GitHub 가이드](02_GitHub_가이드.md) 참고
