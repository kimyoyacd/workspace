## D. 킥오프 공지 (팀에게 전달할 것)

아래를 팀 채널/미팅에서 한 번 정리해 공지하세요:

1. **왜 도입하나** — 반복 업무(RFP 분석·견적 초안·시안 1차 검수·리서치) 속도. 최종 판단은 여전히 사람.
2. **읽을 문서** — `README.md` → `docs/team/00~02`. 신입은 00부터.
3. **황금률 3가지**
   - main 직접 손대지 않기 (항상 브랜치 → PR)
   - 민감정보(단가·키·PII·계약) 올리지 않기
   - 돈·계약·외부 발송 최종 결정은 실장
4. **첫 과제** — 각자 담당 프로젝트의 노트를 `.claude/projects/`에 하나씩 만들어 보기 (실습 겸)
5. **질문 창구** — 막히면 Claude에게 먼저 물어보고, 안 되면 실장
