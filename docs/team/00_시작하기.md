# 00. 시작하기 — 3분이면 됩니다

> 대상: MAX실 전원. 디자인 툴은 잘 다루지만 코딩·GitHub은 처음인 분 기준으로 씁니다.

## 이게 뭔가요?

우리 실이 쓰는 **AI 업무 비서(Claude) 세팅 한 벌**입니다. 그냥 챗봇이 아니라, 우리 실 규칙·견적 단가표·프로젝트 히스토리·검수 기준을 미리 학습시켜 둔 "우리 회사 버전 AI"라고 보면 됩니다.

- **에이전트** = 역할별 전문가 (RFP 분석가, 견적 회계사, 시안 비평가 등 24명)
- **스킬** = 특정 작업 자동화 (리포트 HTML 만들기, 글 다듬기 등)
- **프로젝트 노트** = 프로젝트별 기록 (클라이언트·범위·일정·견적·메모)
- **GitHub** = 이 모든 걸 팀이 함께 보고, 버전으로 안전하게 관리하는 창고

## 무엇을 시킬 수 있나요? (예시)

| 하고 싶은 것 | 이렇게 말하면 됨 |
|---|---|
| 받은 RFP에서 진짜 니즈 뽑기 | "이 RFP 분석해줘" (파일 첨부) |
| 견적 원가표·마진 시나리오 | "이 건 견적 뽑자" |
| 올린 시안 냉정하게 검수 | "이 시안 검수해줘" |
| 경쟁사 포지셔닝 맵 | "경쟁사 분석해서 포지셔닝 맵 그려줘" |
| 캡션·상세페이지 카피 다듬기 | "이 글 AI 티 빼줘" |
| 이번 주 팀 현황·가동률 | "이번 주 현황 정리해줘" |

전체 목록은 **[01_사용설명서](01_사용설명서.md)** 참고.

## 첫 세션 (3분)

1. Claude Code(웹 또는 앱)에서 이 워크스페이스(`workspace` 저장소)를 연다.
2. 세션이 시작되면 Claude가 **`CLAUDE.md`를 자동으로 읽어** 우리 실 규칙을 파악합니다. 따로 설명 안 해도 됨.
3. 바로 평소 말투로 시킵니다. 예: *"넷마블 상황 브리핑해줘"*, *"이 상세페이지 시안 검수해줘"*.
4. 결과가 마음에 들면, 남길 만한 산출물은 **프로젝트 노트에 기록**하거나 **GitHub에 저장(커밋)** 합니다. → [02_GitHub_가이드](02_GitHub_가이드.md)

## 딱 3가지만 기억하세요

1. **한국어로, 평소 실무 말투로** 시키면 됩니다. 명령어 외울 필요 없음.
2. **애매하면 되물어봐 달라고** 하세요. Claude가 멋대로 판단하기보다 확인하게 하는 게 안전합니다.
3. **돈·계약·외부 발송의 최종 결정은 사람이** 합니다. Claude는 초안·검토까지. ([CLAUDE.md 규칙](../../CLAUDE.md))

- GitHub 조작이 무서움 → [02_GitHub_가이드](02_GitHub_가이드.md) (겁먹지 않아도 됩니다)
- 그래도 모르겠으면 → Claude에게 그냥 물어보세요. "이거 어떻게 해?"
