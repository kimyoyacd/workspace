## 0. 표지 / 목차
- `〔클라이언트〕 〔프로젝트명〕 제안서` + 한 줄 슬로건(예: "경험을 지능으로 확장하는")
- 목차: 배경·목적 / 제안사항 / 리서치 / 전략·설계 / 수행범위 / 방법론 / 일정 / 운영방안 / 제안사 현황
- 🔖 원본: KT 표지·목차, 42dot Renewal "제안서 목차" 페이지

## 1. 제안 배경 및 목적
세 단으로 전개하는 검증된 구조 (🔖 KT "제안 배경 및 목적"):
1. **시장·트렌드 배경** — 〔도메인〕의 거시 변화 2~3가지 (예: AI-First 전환, Zero-Click UX, 초개인화)
2. **전략적 필요성** — 경쟁사 동향 → 〔클라이언트〕가 지금 움직여야 하는 이유
3. **제안 목적** — `고객경험 / 플랫폼 / 브랜드` 3축으로 목적 분해

> Tip: 트렌드 주장에는 출처 인용을 붙인다 (🔖 42dot Research가 Adobe·Microsoft·Verizon CX 인용 방식 사용).

## 2. 제안 목표 & 핵심 제안사항
- 한 문장 목표 선언 ("단순 화면 고도화가 아니라 〔핵심 전환〕")
- **3대 제안사항(PROPOSAL 01/02/03)** 카드 — 각 카드에 `제안 → 기대효과` (🔖 KT "3대 제안사항")

## 3. 리서치 / Discovery
가장 재사용도 높은 전략 골격 (🔖 42dot Design Strategy 전체 흐름):
```
Introduction → Research(UX·AI·Visual) → Audit(자사 진단) → Competitive Audit(경쟁사)
→ Challenges & Opportunities → Approach → Next Step
```
- **Research**: UX/AI/Visual 3개 트랙으로 트렌드 + 인용구
- **Audit**: 현재 〔서비스〕의 Features(잘된 점) / Problems(개선점) 대비표
- **Competitive**: 경쟁사 3~4곳을 "메시지 전달 방식 / 기술 표현 방식"으로 비교
- **C&O**: 위 분석을 디자인 과제 + 기회로 수렴

## 4. 사용자 분석
- **사용자별 관점 진단** — 이해관계자를 2~4 그룹으로 나눠 각자의 니즈 정의 (🔖 Clay SDIJ: 학생/강사·운영/방문객 3분할)
- **정성 데이터 인사이트** — 설문·User Feedback 인용 + 언급 빈도수로 이슈 우선순위화 (🔖 Clay "소음 31회·공기 21회 언급" 방식)
- 각 이슈마다 `ISSUE → 근거(피드백) → SUGGESTION → REFERENCE` 4단 구성

## 5. UX/디자인 전략
- **N축 전략 프레임** — 핵심 가치를 3개 키워드로 (🔖 42dot Recruit: `Exploration · Personalization · Communication`)
- 각 축마다 `정의 → 적용 기능 → 레퍼런스 링크`
- AI 프로젝트라면: AIX 프레임워크 + AI UX 라이팅·톤앤매너 가이드 추가 (🔖 KT)

## 6. 수행 범위 (Scope)
- **3-Layer 수직 통합** 구조로 제시 (🔖 KT·42dot):
  - Phase 1. Insight & Strategy — 전략/방향 정의
  - Phase 2. Experience Design — 설계·시각화
  - Phase 3. Realization & Operation — 구현 연계·운영
- 각 Phase별 **Deliverable** 명시 (전략서 / 디자인 시스템 / 프로토타입 / 가이드북 / 운영 매뉴얼)

## 7. 추진 방법론
- **PDCA × Agile 하이브리드** (🔖 KT): 품질확보·이슈 통합관리 요구 RFP에 강력
  - Agile 루프(1~3주 스프린트, 데일리 스크럼, 회고)로 실험적 AI/UX 과업 대응
  - PDCA로 품질·장애 예방 강조

## 8. 시행 일정
- 총 기간 → 단계(분석/UX/UI/가이드/이관) → 주차별 WBS 표 (🔖 KT "24주 시행일정")
- 마일스톤: 착수보고 / 디자인방향성 보고 / 중간보고 / 완료보고

## 9. 운영 및 관리 방안
- 기술 이전·교육 / 운영 프로세스 / 운영 품질관리 / 리스크 대응 (🔖 42dot Renewal "5. 운영 및 관리 방안")
- CMS 운영 효율화, 컴포넌트 라이브러리 문서화 (🔖 42dot·KT)

## 10. 제안사 현황 / 수행 실적
- 핵심 메시지: "테스트가 아닌 실전 경험" — 연도별 대표 프로젝트 리스트업 (🔖 KT "수행 장점" 페이지)
- 동종업계 신뢰 카드 배치 ([활용계획] 2-2 도메인 매칭 참고)
